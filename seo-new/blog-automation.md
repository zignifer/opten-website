# Блог Opten — свободный контент-бриф для SEO/GEO статей

Этот бриф задает направление для создания блог-постов Opten: темы, семантика,
SEO-параметры, структура, визуальные референсы и критерии качества. Он не
описывает процессы, инфраструктуру или технические шаги.

## 0. Как пользоваться файлами

При создании статьи используй этот набор как обязательный контекст:

- `blog-automation.md` — главный бриф. Если есть конфликт между краткой задачей и
  этим файлом, ориентируйся на этот файл как на источник правил по статье.
- `blog-clusters.md` — топовые ключи для блога: кластеры `BL-news`,
  `BL-technique`, `BL-usecase`, `BL-prompting`, объемы, RU/EN формулировки и
  идеи тем.
- `homepage-clusters.md` — топовые ключи для главной страницы и общего
  позиционирования Opten: prompt generator, prompt enhancer, prompt scorer,
  prompt extension и близкие brand-level формулировки.
- `Reference/` — прошлые генерации, которые нравятся по композиции,
  visual-first подаче, крупным коротким надписям и общему уровню картинки.

Для статьи сначала выбери тему и основной интент по `blog-clusters.md`, затем
сверь структуру и качество с этим брифом, а продуктовые формулировки Opten
калибруй по `homepage-clusters.md`. Для изображений `Reference/` используй как
референс вкуса, но текущий бриф важнее: текст на картинках и палитра должны
соответствовать ТЗ конкретной статьи, даже если на старых референсах другие
надписи или немного другие цвета.

## 1. Контентная роль блога

Блог Opten объясняет, как получать лучший результат в генеративных AI-моделях за
счет хорошего промпта, правильного выбора модели и грамотной итерации. Основные
темы:

- промптинг для изображений и видео;
- сравнение и разбор AI-моделей;
- практические сценарии: логотипы, аватары, UGC, реклама, headshots, постеры;
- типичные ошибки генерации и способы их исправления;
- новости моделей, если они дают понятный пользовательский интент.

Статья должна быть самостоятельной и полезной без продажи в каждом абзаце. Opten
уместен как мягкий контекст: когда результат зависит от промпта, расширение
помогает оценить и улучшить запрос под конкретную модель.

## 2. Семантические кластеры

Используй четыре editorial-кластера.

| Кластер | Направление | Роль |
| --- | --- | --- |
| `BL-news` | релизы, новости, сравнения | быстрый спрос и актуальные инфоповоды |
| `BL-technique` | how-to, техники, исправление ошибок | вечнозеленый обучающий трафик |
| `BL-usecase` | практические применения | коммерческий интент и понятный результат |
| `BL-prompting` | промптинг как тема | ядро продукта и самый близкий интент |

Примеры углов:

- "Nano Banana Pro: как писать промпты для стабильного результата";
- "Sora 2 vs Veo 3.1: что выбрать для коротких роликов";
- "Как оживить фото нейросетью и не получить лишние пальцы";
- "Что такое негативный промпт и когда он действительно помогает";
- "AI logo generator: как подготовить промпт для бренда";
- "Consistent character AI: как удержать лицо и стиль в серии кадров".

## 3. Выбор ключей

Одна статья = одна основная фраза и 2-4 смежные фразы. Ключи должны отражать
реальный интент, а не просто высокий объем.

Карта семантики:

| Элемент | Что использовать |
| --- | --- |
| Title / H1 | основная фраза ближе к началу |
| Excerpt | основная фраза + польза для читателя |
| Meta description | основная фраза + смежная фраза в естественной формулировке |
| Intro | основная фраза и прямой answer-block |
| H2 | смежные how/what/why/best/vs формулировки |
| Body | long-tail, синонимы, названия моделей, практические уточнения |
| FAQ | реальные вопросы пользователей |
| Alt | описание сцены + один релевантный ключ, если звучит естественно |

RU и EN семантику подбирай отдельно. Русский спрос часто формулируется как
"нейросеть для видео", "как составить промпт", "негативный промпт"; английский —
"image to video ai", "prompt examples", "ai face swap", "what is a prompt".

## 4. SEO-параметры

Ориентиры для каждой локали:

| Параметр | Значение |
| --- | --- |
| Slug | единый для RU/EN, латиница, kebab-case |
| Title | до 60 символов, читабельно, с основной фразой |
| Excerpt | 140-180 символов |
| Description | 150-160 символов |
| Category | `guide`, `deep-dive`, `comparison`, `news` |
| Tags | 2-4: `ai-image-gen`, `ai-video-gen`, `prompt-engineering`, `model-deep-dive`, `workflow`, `release-notes` |
| Intro | 40-60 слов, прямой ответ в начале |
| Main body | 4-6 steps или 3-6 sections |
| FAQ | 3-5 Q&A |
| Reading time | реалистично по объему |
| Images | обложка + 4-5 поясняющих изображений, если тема визуальная |

Структурированные данные должны быть возможны без расхождений с текстом:
BlogPosting/Article, HowTo для пошаговых материалов, FAQPage для вопросов,
BreadcrumbList, canonical, hreflang, Open Graph, image metadata, speakable/AI
summary при необходимости.

## 5. Answer-block

Первые 40-60 слов — главный GEO-блок. Он должен отвечать на запрос так, чтобы
его можно было процитировать отдельно.

Формула:

1. Назвать сущность или задачу.
2. Дать прямой ответ.
3. Уточнить, от чего зависит результат.
4. Мягко связать с промптом или выбором модели.

Пример направления:

> Image to video AI превращает статичное изображение в короткий ролик, но
> качество движения зависит не только от модели. Самый стабильный результат
> дает промпт, где отдельно описаны объект, камера, движение, ограничения и
> нежелательные артефакты.

## 6. Структура материала

Хороший пост обычно содержит:

- короткий answer-block;
- контекст: кому и когда это нужно;
- named practical case с конкретной моделью;
- пошаговый процесс или сравнение подходов;
- блок типичных ошибок;
- чек-лист промпта;
- FAQ.

Шаги не должны быть одинаковыми по форме. Чередуй:

- диагностику проблемы;
- разбор конкретного кейса;
- сравнение моделей;
- до/после;
- чек-лист;
- предупреждение о типичных ошибках.

## 7. Practical case

В каждом материале нужен хотя бы один конкретный кейс:

- модель;
- исходная задача;
- первая ошибка результата;
- точная правка промпта;
- что изменилось после правки.

Пример формы:

> В Kling 3.0 первый рендер дал резкий скачок камеры и лишние пальцы. После
> добавления "locked camera, natural hand anatomy, preserve five fingers, no
> sudden zoom" движение стало спокойнее, а руки перестали ломать кадр.

Кейс должен быть частью статьи, а не только подписью к картинке.

## 8. Фото и визуальные референсы

Обложка:

- 16:9;
- без текста;
- один понятный объект или сцена;
- темный cinematic / modern tech стиль;
- deep teal-black `#011417`, lime `#9cfb51`, white, muted green;
- мягкое свечение, чистая композиция, без перегруза.

Внутристатейные изображения:

- visual-first, не сухой интерфейсный слайд;
- результат генерации крупнее UI-элементов;
- аккуратные lime-линии, стрелки, play bar, timeline, before/after;
- короткий крупный текст: 1-4 слова или одна короткая строка;
- генерируй внутристатейные RU/EN изображения сразу с нужным коротким текстом в
  кадре; не создавай "чистую" базу без текста и не накладывай надпись локально;
- если генератор исказил текст, сократи текст и перегенерируй изображение, а не
  исправляй его отдельным локальным оверлеем;
- RU и EN пары совпадают по композиции;
- смысл изображения повторяется в тексте и alt;
- без мелких абзацев, таблиц и плотных карточек.

Удачный референс: кинематографичный кадр результата, поверх которого есть
несколько понятных UI-подсказок. Неудачный референс: темный экран с множеством
карточек, мелким текстом и без сильного визуального объекта.

Папка `Reference/` содержит прошлые генерации, которые нравятся по композиции и
качеству подачи. Ориентируйся на них по уровню визуала, крупности текста,
visual-first сцене и аккуратным оверлеям. Не копируй старые надписи и не
повторяй старую палитру, если она отличается от текущего ТЗ: для новых кадров
главные источники — текст, смысл и цвета из этого брифа и задачи статьи.

## 9. Бренд-стиль

Визуальный язык:

- темная тема;
- `#011417` как основной фон;
- `#9cfb51` как акцент;
- белый текст с высоким контрастом;
- стеклянные панели, тонкие линии, мягкое лаймовое свечение;
- технологичный, но не кричащий SaaS-вид;
- без декоративных градиентных шаров и абстрактных фонов без смысла.

Шрифтовое направление:

- заголовки в духе Unbounded;
- подписи и UI-текст в духе PT Root UI;
- крупные слова вместо длинных предложений на изображении.

## 10. Тон и E-E-A-T

Пиши как практик, который сам проверяет промпты и понимает ограничения моделей.
Важны конкретика, проверяемые формулировки и честные ограничения.

Хорошие признаки:

- реальные названия моделей;
- конкретные ошибки генерации;
- точные prompt-fixes;
- сравнение применимости, а не абстрактный рейтинг;
- объяснение, почему один промпт работает лучше другого;
- короткие FAQ-ответы, пригодные для AI Overview.

Слабые признаки:

- "лучший инструмент для всего";
- обещания безлимита, бесплатности или гарантированного результата;
- SEO-повторы одной фразы;
- пустые определения;
- устаревшие версии моделей;
- изображения, которые не объясняют мысль статьи.

## 12. Финальная проверка качества

Материал готов, если:

- тема не повторяет уже сделанный пост;
- RU и EN написаны под свои поисковые формулировки;
- в первых 40-60 словах есть прямой ответ;
- title, description, H2, FAQ и alt используют семантику естественно;
- есть практический кейс с моделью, ошибкой и исправлением;
- FAQ отвечает на реальные вопросы;
- изображения помогают понять материал;
- внутристатейные кадры с текстом сгенерированы сразу как финальные RU/EN
  изображения, без локального наложения надписей;
- Opten упомянут по делу и не превращает статью в рекламную страницу.
