## Directory Purposes

**`api/`** — Vercel serverless functions. Currently one file (`download-skill.ts`). Anything in `api/_assets/**` bundled via `vercel.json:includeFiles`.

**`public/`** — Served verbatim from `/`. Static assets, fonts, OG images (RU + EN), blog covers, sitemap, robots. Read by `vite-imagetools` imports too.

**`scripts/`** — Build-time Node tooling (8 files). `entry-server.tsx` + `seo-routes.ts` SSR-compiled into `.ssr-cache/` + `.ssr-meta/`; others are pure-Node consumers running postbuild. ESM only.

**`src/main.tsx`** — Single entry. Owns hydrate-vs-render decision (`__PRERENDER_PATH` check). Declares 18 Routes + catch-all. Hydration guard prevents React #418/#423 on SPA-fallback hits.

**`src/app/App.tsx`** — Landing; serves `/` and `/en/` (same component, language read from context). Only component prerendered for both landing variants.

**`src/app/components/`** — Shared UI primitives. Logical groups:
- Top level: LangSwitcher, LocalizedLink, SiteHeader, SiteFooter, BlogPostCard, FaqBlock, OptenHeroAnimation, Picture, InstallButton, RouteLoading
- `components/figma/` — Figma-generated helpers (brittle interface)
- `components/layout/` — Page layouts (LegalLayout)
- `components/ui/` — shadcn-style Radix wrappers

**`src/app/pages/`** — One component per page (or dual RU/EN). Same components reused for RU + EN siblings — language from LangProvider context. Pages touching `window`/`localStorage`/`chrome.runtime` at mount are lazy-loaded.

**`src/content/`** — Human-editable content data (not React components). Three files:
- `about.tsx` — AboutPage body structure
- `landingFaq.ts` — Landing FAQ (mirrored 1:1 into FAQPage schema with parity enforcement)
- `blog/` — BlogPost files (one per post, one barrel)

**`src/i18n/`** — Custom React context i18n:
- `LangContext.tsx` — Provider, `useT()`, `useLang()`, `useOnEnPath()`
- `paths.ts` — EN_SIBLINGS allow-list + path rewriters (single source of truth for redirects)
- `ru.json`, `en.json` — String dicts (68 KB RU, ~41 KB EN lazily loaded on client)

**`src/imports/`** — ⚠ **Auto-generated by Figma Make.** Each subfolder bundles `.tsx`, SVG paths, raster assets. Treat as black-box; edit only if designer regenerating. Used by PayPage, AccountPage, DownloadSkillPage.

**`src/lib/`** — Plain helper modules. Currently only `paddle.ts`.

**`src/styles/`** — All CSS. Tailwind 4 via `@tailwindcss/vite`; theme tokens in `theme.css`; `@font-face` in `fonts.css`.

**`src/types/`** — Ambient TS declarations (Paddle global, imagetools query types).

**`docs/`** — Stable human-written reference:
- `INTEGRATION-CONTRACT.md` — BINDING interface with extension (read before payment/auth/route changes)
- `CONTENT-AUTHORING.md` — GEO/SEO playbook for new pages/posts (read before content changes)
- `ARCHITECTURE.md`, `TECH.md`, `SEO-AUDIT.md` — Reference docs

**`.planning/`** — Live GSD state. Obsidian vault root; prefer `[[wikilinks]]` inside.

## Naming Conventions

**Files:**
- Components: `PascalCase.tsx`, one per file
- Hooks / helpers: `camelCase.ts`
- Pure helpers: `camelCase.ts` (`paths.ts`, `paddle.ts`)
- CSS: lowercase (`index.css`, `theme.css`)
- Dicts: `ru.json`, `en.json`

**Directories:**
- lowercase, kebab-case (`api/_assets`, `src/imports/`)
- `src/app/pages/`, `src/app/components/`, `src/content/blog/` always plural

**Constants:**
- Module-level: `UPPER_SNAKE_CASE` (`SUPABASE_URL`, `EXTENSION_IDS`, `EN_SIBLINGS`)

**Storage keys:**
- Site-owned: `opten_` prefix (`opten_lang_v3`, `opten_pay_currency`)
- Extension-owned (read-only): `ps_*` (never touched by site)

**i18n keys:** dot-namespaced (`nav.blog`, `blog.intro`, `faq.item1.question`)

**Comments:**
- Phase refs: `// Phase 5 B-04:` (match extension repo numbering)
- Design decisions: `// D-03:`, `// D-06:`
- Bugs: `// BG-67-01:`

## Where to Add New Code

**New prerendered page (full, both languages):**
1. Component: `src/app/pages/FooPage.tsx` (same component for RU + EN)
2. **Two** Routes in `src/main.tsx`: `<Route path="/foo">` + `<Route path="/en/foo">`
3. **Two** Routes in `scripts/entry-server.tsx`
4. **Two** entries in `scripts/seo-routes.ts` with `prerender: "full"`, reciprocal hreflangAlternates
5. Add `/foo` to `EN_SIBLINGS` in `src/i18n/paths.ts`
6. Add **both** `src/i18n/ru.json` and `src/i18n/en.json` keys

**New SPA-only page (no EN sibling):**
1. Component: `src/app/pages/FooPage.tsx`
2. `const FooPage = lazy(() => import(...))` in `src/main.tsx`
3. **One** `<Route>` (no `/en/foo` sibling)
4. Do NOT add to `entry-server.tsx`
5. Do NOT add to `EN_SIBLINGS`
6. Optionally add to `seo-routes.ts` for meta (like `/pay`)

**New blog post:**
1. Create `src/content/blog/<slug>.ts` with `{ ru, en }` locales
2. Import + add to `blogPostsBySlug` in `src/content/blog/index.ts`
3. Create `public/blog/<slug>/cover.jpg` (≥1600×900, no text)
4. Add **two** RouteMeta entries to `scripts/seo-routes.ts` (RU + EN)
5. Add `/blog/<slug>` to `EN_SIBLINGS` in `src/i18n/paths.ts`
6. Update `scripts/sitemap.mjs` PATH_TO_SOURCE + floor check
7. Update `scripts/llms.mjs` floor check (or section already matches)
8. Run `npm run build` locally (floor checks catch missing registrations)
9. Optionally run `node scripts/smoke-blog.mjs` (Playwright test)

**New shared component:** `src/app/components/Foo.tsx` (top level) or `components/layout/` if a page wrapper.

**New internal navigation:** Always use `<LocalizedLink to="...">` — preserves `/en/` prefix. Use bare `<Link>` only when intentionally crossing language.

**New imperative routing:** Import `toEnTarget`, `toRuTarget`, `localizeHref` from `src/i18n/paths.ts`. Never hand-roll `/en` concatenation.

**New helper module:** `src/lib/foo.ts`.

**New static asset:** `public/assets/<category>/...`. Reference via `/assets/...` or `?as=picture` import.

**New env var (public):** Add `VITE_FOO` to `.env.example` + Vercel project settings.

**New serverless function:** `api/<name>.ts`. Add asset bundling to `vercel.json` if needed. CORS must whitelist `https://opten.space`.

**`opten-design/`** — Figma design assets. Generated: yes. Committed: yes.

**`font/`** — Raw font sources (pre-WOFF2). Generated: no. Committed: yes. Runtime fonts in `public/fonts/`.

**`icons/`** — Source icons. Generated: no. Committed: yes. Runtime icons from Lucide / MUI / inline SVG, not this folder.

**`guidelines/`** — Brand/design guidelines. Generated: no. Committed: yes.

**`.obsidian/`** — Obsidian vault config. Committed selectively.

**`.ssr-cache/`, `.ssr-meta/`** — Vite SSR build outputs. Gitignored. Separate dirs because Vite empties outDir on each build.
