## Directory Purposes

**`api/`** — Vercel serverless functions only. Currently one file (`download-skill.ts`). Anything in `api/_assets/**` is bundled with the function via `vercel.json:16-18`.

**`public/`** — Served verbatim from `/`. Static assets, fonts, OG images, sitemap, robots. Read by `vite-imagetools` static imports too (`?as=picture` query) because `vite.config.ts:17` sets `imagetools({ exclude: '' })`.

**`scripts/`** — Build-time Node tooling. Runs after `vite build` to produce per-route prerendered HTML and sitemap. ESM only.

**`src/main.tsx`** — Single entry point. Owns the hydrate-vs-render decision; without that guard, React #418/#423 fires on every extension deep-link load.

**`src/app/App.tsx`** — Landing page; only component prerendered for `/`. Imports landing imagery as `?w=…&format=webp;png&as=picture` queries.

**`src/app/components/`** — Shared UI primitives. Three subgroups:
- `components/figma/` — Figma-generated helpers (brittle interface — see notes).
- `components/layout/` — Page chrome / layouts.
- `components/ui/` — shadcn-style Radix wrappers (full kit copied in; only a subset is actually used by the 8 routes — see notes).

**`src/app/pages/`** — One component per route. Pages that touch `window`/`localStorage`/`chrome.runtime` at mount are lazy-loaded from `main.tsx`.

**`src/i18n/`** — Custom React context i18n. JSON dicts, no `i18next`. Strings keyed by dot-namespaced paths (`success.next.step1` etc.).

**`src/imports/`** — ⚠ **Auto-generated by Figma Make.** Each subfolder bundles a generated `.tsx`, raw SVG path strings (e.g. `svg-bvy0jfb1g6.ts`), and raster assets used inline. Treat as black-box: edit only if a designer is regenerating. Used by PayPage, AccountPage, DownloadSkillPage (for Chrome/Yandex icons + landing-style background paths).

**`src/lib/`** — Plain helper modules. Currently only `paddle.ts`. Anything not React-bound and not a hook goes here.

**`src/styles/`** — All CSS. Tailwind 4 via `@tailwindcss/vite` plugin; theme tokens in `theme.css`; `@font-face` in `fonts.css`.

**`src/types/`** — Ambient TS declarations (Paddle global, imagetools query types).

**`docs/`** — Stable human-written reference. `INTEGRATION-CONTRACT.md` is the binding interface with the extension repo — read before any payment/auth/route change.

**`.planning/`** — Live GSD state. Obsidian vault root is the repo root, so prefer `[[wikilinks]]` inside `.planning/`.

**`supabase/`** — CLI link directory only (project `opten-seo`). Real Supabase project (`opten`) is linked from the extension repo.

| Path | Status | Notes |
|------|--------|-------|
| `src/imports/**` | **Fragile, auto-generated** | Figma-Make output. Includes raw SVG path-data strings, base64 assets, layered JSX. Re-generation overwrites entire subfolder. Do not hand-edit unless the Figma source is gone. |
| `src/app/components/figma/ImageWithFallback.tsx` | Figma-generated helper | Stable but not project-style; do not extend with new features — write a fresh component in `components/` instead. |

| `public/sitemap.xml` | Generated | Output of `scripts/sitemap.mjs` after build. |

**Configuration:**
- `vite.config.ts`: Vite + Tailwind + imagetools + manualChunks

- `postcss.config.mjs`: PostCSS (Tailwind 4)

**Core Logic:**
- `src/app/App.tsx`: Landing

- `src/lib/paddle.ts`: Paddle SDK bootstrap
- `src/i18n/LangContext.tsx`: i18n provider

- `SUPABASE_URL` / `SUPABASE_ANON_KEY` — same three pages + `api/download-skill.ts:14-16`

## Naming Conventions

**Files:**
- Components: `PascalCase.tsx`, one per file (`PayPage.tsx`, `OptenHeroAnimation.tsx`)
- Hooks / helpers: `camelCase.ts` or `kebab-case.ts` for shadcn copies (`use-mobile.ts`, `utils.ts`)
- CSS: lowercase (`index.css`, `theme.css`)
- Dicts: `ru.json`, `en.json`

**Directories:**
- lowercase, kebab-case (`api/_assets`, `dashboard/download-skill` route segment)
- `src/app/pages/` always plural

**Constants:**
- Module-level: `UPPER_SNAKE_CASE` (`SUPABASE_URL`, `EXTENSION_IDS`, `CHROME_STORE_URL`, `CURRENCY_STORAGE_KEY`)

**Storage keys:**
- Site-owned: `opten_` prefix (`opten_lang`, `opten_pay_currency`)
- Extension-owned (read indirectly): `ps_*` (never touched by site code)

**i18n keys:** dot-namespaced (`success.next.step1`, `account.payment.cardDefault`)

**Comments:**
- Phase refs: `// Phase 2.2:`, `// Phase 66:` — match extension repo phase numbering
- Decisions: `// D-04:` — design decisions
- Bugs: `// BG-67-01:` — bug refs

## Where to Add New Code

**New marketing route (full prerender):**
1. Component: `src/app/pages/FooPage.tsx`
2. Eager import + `<Route>` in `src/main.tsx`
3. Eager import + `<Route>` in `scripts/entry-server.tsx`
4. Entry in `scripts/seo-routes.ts` with `prerender: "full"`
5. Update sitemap will regenerate automatically via `scripts/sitemap.mjs`

**New user-state route (SPA-only):**
1. Component: `src/app/pages/FooPage.tsx`
2. `const FooPage = lazy(() => import(...))` in `src/main.tsx`
3. `<Route>` inside the `<Suspense>` block
4. Do NOT add to `entry-server.tsx`
5. Optionally add `prerender: "head"` entry to `seo-routes.ts` for meta

**New shared component:** `src/app/components/Foo.tsx` (top level) or under `components/layout/` if it's a layout wrapper.

**New UI primitive:** Prefer extending `src/app/components/ui/*` (shadcn pattern) only if it follows that codegen style; otherwise put a hand-written component at `src/app/components/`.

**New helper module:** `src/lib/foo.ts`.

**New i18n strings:** Add to both `src/i18n/ru.json` and `src/i18n/en.json`. If user-facing on a prerendered route, also update the duplicated copy in `scripts/seo-routes.ts`.

**New static asset:** `public/assets/<category>/...`. Reference via `/assets/...` or as `?as=picture` import.

**New env var (public):** Add `VITE_FOO` to `.env.example` and Vercel project settings. Bundled into client at build.

**New serverless function:** `api/<name>.ts`. Add asset bundling to `vercel.json` `functions` block if needed. CORS must whitelist `https://opten.space`.

**New planning artifact:** `.planning/phases/<N>-<name>/<N>-SPEC.md|PLAN.md|CONTEXT.md`. Use GSD slash commands rather than hand-creating.

## Special Directories

**`opten-design/`** — Purpose: Figma design assets. Generated: yes (from Figma). Committed: yes.

**`font/`** — Purpose: Raw font sources before WOFF2 conversion. Generated: no. Committed: yes. Runtime fonts live in `public/fonts/`.

**`icons/`** — Purpose: Source icons. Generated: no. Committed: yes. Runtime icons come from Lucide / MUI / inline SVG in `src/imports/`, not from this folder.

**`guidelines/`** — Purpose: Brand / design guidelines. Generated: no. Committed: yes.

**`.obsidian/`** — Obsidian vault config (vault root = repo root). Committed selectively.

**`.vercel/`** — Vercel CLI link state. Mostly gitignored.

**`.playwright-mcp/`** — Playwright MCP server cache. Gitignored.
