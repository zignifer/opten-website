## Directory Purposes

**`api/`** вЂ” Vercel serverless functions only. Currently one file (`download-skill.ts`). Anything in `api/_assets/**` is bundled with the function via `vercel.json:16-18`.

**`public/`** вЂ” Served verbatim from `/`. Static assets, fonts, OG images (RU + EN), sitemap, robots. Read by `vite-imagetools` static imports too (`?as=picture` query) because `vite.config.ts:17` sets `imagetools({ exclude: '' })`.

**`scripts/`** вЂ” Build-time Node tooling. Five files. `entry-server.tsx` + `seo-routes.ts` are SSR-compiled into separate outDirs (`.ssr-cache/`, `.ssr-meta/`); `prerender.mjs` and `sitemap.mjs` are pure-Node consumers that run after `vite build`. ESM only.

**`src/main.tsx`** вЂ” Single entry point. Owns the hydrate-vs-render decision (`__PRERENDER_PATH` marker check); declares all 15 Routes (9 RU + 6 EN). Without the hydration guard, React #418/#423 fires on every SPA-fallback hit.

**`src/app/App.tsx`** вЂ” Landing page; serves `/` and `/en/` (same component вЂ” language read from context). Only component prerendered for both landing variants. Imports landing imagery as `?w=вЂ¦&format=webp;png&as=picture` queries.

**`src/app/components/`** вЂ” Shared UI primitives. Four logical groups:
- Top level вЂ” `LangSwitcher`, `LocalizedLink`, `OptenHeroAnimation`, `Picture`, `RouteLoading`. `LangSwitcher` + `LocalizedLink` are the i18n routing primitives (added in Phase 3 follow-up).
- `components/figma/` вЂ” Figma-generated helpers (brittle interface вЂ” see notes).
- `components/layout/` вЂ” Page chrome / layouts.
- `components/ui/` вЂ” shadcn-style Radix wrappers (full kit copied in; only a subset is actually used by the route components вЂ” see notes).

**`src/app/pages/`** вЂ” One component per page. Same page components are reused for RU + EN siblings вЂ” language comes from `LangProvider` context, not from the component itself. Pages that touch `window`/`localStorage`/`chrome.runtime` at mount are lazy-loaded from `main.tsx`.

**`src/i18n/`** вЂ” Custom React context i18n. Four files:
- `LangContext.tsx` вЂ” provider, `useT()`, `useLang()`, `useOnEnPath()`. Detection order: URL prefix в†’ `localStorage.opten_lang_v3` в†’ legacy `localStorage.opten_lang === "en"` (one-shot migration) в†’ `navigator.language`.
- `paths.ts` вЂ” `EN_SIBLINGS` allow-list + `toEnTarget` / `toRuTarget` / `localizeHref`. **Single source of truth for path rewrites.** Open-redirect mitigation lives here.
- `ru.json`, `en.json` вЂ” string dicts. RU is statically imported; EN is statically imported on the SSR path only (synchronous SSR EN render) and lazily imported on the client.

**`src/imports/`** вЂ” вљ  **Auto-generated by Figma Make.** Each subfolder bundles a generated `.tsx`, raw SVG path strings, and raster assets used inline. Treat as black-box: edit only if a designer is regenerating. Used by PayPage, AccountPage, DownloadSkillPage (for Chrome/Yandex icons + landing-style background paths).

**`src/lib/`** вЂ” Plain helper modules. Currently only `paddle.ts`. Anything not React-bound and not a hook goes here.

**`src/styles/`** вЂ” All CSS. Tailwind 4 via `@tailwindcss/vite` plugin; theme tokens in `theme.css`; `@font-face` in `fonts.css`.

**`src/types/`** вЂ” Ambient TS declarations (Paddle global, imagetools query types).

**`docs/`** вЂ” Stable human-written reference. `INTEGRATION-CONTRACT.md` is the binding interface with the extension repo вЂ” read before any payment/auth/route change.

**`.planning/`** вЂ” Live GSD state. Obsidian vault root is the repo root, so prefer `[[wikilinks]]` inside `.planning/`.

**`supabase/`** вЂ” CLI link directory only (project `opten-seo`). Real Supabase project (`opten`) is linked from the extension repo.

## Naming Conventions

**Files:**
- Components: `PascalCase.tsx`, one per file (`PayPage.tsx`, `LangSwitcher.tsx`, `LocalizedLink.tsx`)
- Hooks / helpers: `camelCase.ts` or `kebab-case.ts` for shadcn copies (`use-mobile.ts`, `utils.ts`)
- Pure helper modules: `camelCase.ts` (`paths.ts`, `paddle.ts`)
- CSS: lowercase (`index.css`, `theme.css`)
- Dicts: `ru.json`, `en.json`

**Directories:**
- lowercase, kebab-case (`api/_assets`, `dashboard/download-skill` route segment)
- `src/app/pages/` always plural

**Constants:**
- Module-level: `UPPER_SNAKE_CASE` (`SUPABASE_URL`, `EXTENSION_IDS`, `CHROME_STORE_URL`, `CURRENCY_STORAGE_KEY`, `EN_SIBLINGS`, `STORAGE_KEY`)

**Storage keys:**
- Site-owned: `opten_` prefix (`opten_lang_v3` вЂ” current; `opten_lang` вЂ” legacy one-shot; `opten_pay_currency`)
- Extension-owned (read indirectly): `ps_*` (never touched by site code)

**i18n keys:** dot-namespaced (`success.next.step1`, `account.payment.cardDefault`, `legal.footer.privacy`)

**Comments:**
- Phase refs: `// Phase 2.2:`, `// Phase 3 D-05:` вЂ” match extension repo phase numbering
- Decisions: `// D-04:`, `// D-07:` вЂ” design decisions
- Bugs: `// BG-67-01:` вЂ” bug refs

## Where to Add New Code

**New marketing route (full prerender, both languages):**
1. Component: `src/app/pages/FooPage.tsx` вЂ” same component serves RU and EN, read lang from `useT()`.
2. Add **two** Routes in `src/main.tsx`: `<Route path="/foo">` and `<Route path="/en/foo">`.
3. Add **two** Routes in `scripts/entry-server.tsx` mirror.
4. Add **two** entries to `scripts/seo-routes.ts` with `prerender: "full"`, fully reciprocal `hreflangAlternates`.
5. Add `/foo` to `EN_SIBLINGS` in `src/i18n/paths.ts:9-16`.
6. Sitemap will regenerate; the `<12` floor check still passes.
7. Add string keys to **both** `src/i18n/ru.json` and `src/i18n/en.json`.

**New user-state route (SPA-only, no EN sibling):**
1. Component: `src/app/pages/FooPage.tsx`.
2. `const FooPage = lazy(() => import(...))` in `src/main.tsx`.
3. **One** `<Route>` inside the `<Suspense>` block (no `/en/foo` sibling).
4. Do NOT add to `entry-server.tsx`.
5. Do NOT add to `EN_SIBLINGS`. `LangSwitcher` will correctly skip navigation on this route.
6. Optionally add a `prerender: "head"` entry to `seo-routes.ts` for meta (matches the `/pay` pattern).

**New shared component:** `src/app/components/Foo.tsx` (top level) or under `components/layout/` if it's a layout wrapper.

**New internal navigation:** Always use `<LocalizedLink to="...">` instead of `<Link>` for anything that should follow the user's lang prefix. Use raw `<Link>` only when intentionally crossing language (rare).

**New imperative routing decision:** Import from `src/i18n/paths.ts` вЂ” `toEnTarget`, `toRuTarget`, `localizeHref`. Never hand-roll `"/en" + path` concatenation.

**New UI primitive:** Prefer extending `src/app/components/ui/*` (shadcn pattern) only if it follows that codegen style; otherwise put a hand-written component at `src/app/components/`.

**New helper module:** `src/lib/foo.ts`.

**New i18n strings:** Add to both `src/i18n/ru.json` and `src/i18n/en.json`. If user-facing on a prerendered route, also update the duplicated copy in `scripts/seo-routes.ts` (per-locale title/description).

**New static asset:** `public/assets/<category>/...`. Reference via `/assets/...` or as `?as=picture` import.

**New env var (public):** Add `VITE_FOO` to `.env.example` and Vercel project settings. Bundled into client at build.

**New serverless function:** `api/<name>.ts`. Add asset bundling to `vercel.json` `functions` block if needed. CORS must whitelist `https://opten.space`.

**New planning artifact:** `.planning/phases/<N>-<name>/<N>-SPEC.md|PLAN.md|CONTEXT.md`. Use GSD slash commands rather than hand-creating.

## Special Directories

**`opten-design/`** вЂ” Purpose: Figma design assets. Generated: yes (from Figma). Committed: yes.

**`font/`** вЂ” Purpose: Raw font sources before WOFF2 conversion. Generated: no. Committed: yes. Runtime fonts live in `public/fonts/`.

**`icons/`** вЂ” Purpose: Source icons. Generated: no. Committed: yes. Runtime icons come from Lucide / MUI / inline SVG in `src/imports/`, not from this folder.

**`guidelines/`** вЂ” Purpose: Brand / design guidelines. Generated: no. Committed: yes.

**`.obsidian/`** вЂ” Obsidian vault config (vault root = repo root). Committed selectively.

**`.vercel/`** вЂ” Vercel CLI link state. Mostly gitignored.

**`.playwright-mcp/`** вЂ” Playwright MCP server cache. Gitignored.

**`.ssr-cache/`, `.ssr-meta/`** вЂ” Vite SSR build outputs. Gitignored. Two separate dirs because Vite 6 empties outDir on every SSR build вЂ” keeping `entry-server` and `seo-routes` outputs in separate dirs avoids the second `vite build --ssr` wiping the first.
