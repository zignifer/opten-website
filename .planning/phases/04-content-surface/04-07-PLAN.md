    - "Welcome": `["/welcome", "/en/welcome"]`
    - "About": `["/about"]` (RU only — D-02)
    - "Guides": all routes whose path matches `/^\/(en\/)?guides\//`
    - "Legal": `["/privacy", "/terms", "/refund", "/en/privacy", "/en/terms", "/en/refund"]`

    Each route's TOC line: `- [<title>](<canonical>)`. The `title` comes from `manifest.title` and the `canonical` from `manifest.canonical`.

    (d) Build llms.txt markdown body:
    ```
    # Opten

    > Opten — AI prompt scorer Chrome extension that evaluates and improves prompts for 43+ image-generation models. This site sells, services, and onboards extension users.

    ## Marketing
    - [<title>](<url>)
    - ...

    ## Pricing
    ...

    ## About
    - [<title>](<url>)

    ## Guides
    ...

    ## Legal
    ...
    ```

    The blockquote summary string comes from a constant at the top of the file (the same factual summary used in the Organization schema description IF it exists, otherwise hand-authored once at the top of llms.mjs).

    (e) Build llms-full.txt: iterate the prerendered routes, for each `dist/<route>/index.html`:
    - readFile the HTML.
    - Strip `<script>`, `<style>`, and HTML tags via regex (per `04-PATTERNS.md` `extractText` function). NO cheerio, NO jsdom — keep dependency footprint zero (per RESEARCH §"Don't Hand-Roll" recommendation: regex is sufficient for our prerendered HTML).
    - Collapse whitespace to single spaces.
    - Prefix the body with a `# <title>` line (from manifest) and `URL: <canonical>` line.
    - Concatenate with `\n\n---\n\n` separator.

    (f) Size cap: if `Buffer.byteLength(llmsFullTxt) > 50_000`, drop all "Legal" section bodies from `llms-full.txt`. Prefix the truncated file with `# Opten (truncated)\n\n# Note: legal pages omitted from llms-full.txt to keep the file under 50 KB. Full legal copy is available at the URLs listed in llms.txt.\n\n`.

    (g) `writeFile(resolve(DIST, "llms.txt"), llmsTxt)`; `writeFile(resolve(DIST, "llms-full.txt"), llmsFullTxt)`.

    (h) Console output: `console.log("✓ llms.txt → " + sectionCount + " routes, " + (llmsTxt.length/1024).toFixed(1) + " KB");` and `console.log("✓ llms-full.txt → " + (llmsFullTxt.length/1024).toFixed(1) + " KB" + (truncated ? " (legal truncated)" : ""));`. Pattern matches `sitemap.mjs:49`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build script reads dist HTML | `llms.mjs` parses prerendered HTML with regex; bug in the regex strips legitimate content or leaks `<script>` bodies into the text file. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-15 | Information disclosure | llms-full.txt accidentally including `<script>` content (e.g. JSON-LD blocks) | mitigate | Tag-strip regex per `04-PATTERNS.md` `extractText` removes `<script>` and `<style>` ENTIRE blocks first, then strips remaining tags. Verified by grep `dist/llms-full.txt` for `application/ld+json` — must return 0 matches. |
| T-04-16 | Tampering (stale TOC) | llms.txt drifts from actual route set | mitigate | TOC is generated from the same manifest as the sitemap — single source of truth. Floor check at 15 fails the build if manifest count drops. |
| T-04-17 | DoS (file size) | llms-full.txt unbounded growth as guides expand | mitigate | 50 KB size cap with truncation rule documented; if exceeded, legal pages drop first. |
</threat_model>
