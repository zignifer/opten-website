## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build-time string templating in `prerender.mjs` | JSON.stringify-emitted JSON-LD reaches every prerendered HTML response; injection bug here pollutes every crawler view. |
| `/pay` Paddle script preservation | The path-gated apply chain is the only thing keeping `cdn.paddle.com` in `/pay`'s `<head>`. Refactor risk: reordering helpers without keeping the `</head>` consumption order intact silently drops the Paddle script. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | `applyJsonLd` JSON.stringify output | mitigate | The schema constants are authored in TS (typechecked at SSR build via `vite build --ssr scripts/seo-routes.ts`). String content reaches `JSON.stringify` which escapes user-controlled chars; the values come from the manifest, not from request input — no XSS surface. |
| T-04-02 | DoS | Schema bundle size on every route | accept | Schema blocks add ~3 KB compressed per route (12 routes × 3 KB ≈ 36 KB total uncompressed; gzip ~10 KB). Negligible vs current dist size. |
| T-04-03 | Information disclosure | Founder photo URL on every page via `Person.image` | accept | Photo is public marketing material; no PII. Image is omitted in this plan (lands in Plan 04-05). |
| T-04-04 | Integrity (Paddle invariant) | `applyPaddleScript` apply-chain position | mitigate | Verification command in Task 2 grep-checks `cdn.paddle.com` in `dist/pay/index.html` AND `dist/en/pay/index.html` post-build. Per INTEGRATION-CONTRACT §6, this is a blocking gate. |
