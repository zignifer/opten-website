## Gray areas analyzed

The phase domain (from ROADMAP.md) is "close the hydration gap that Phase 2 exposed". From the locked scope (code-split, modulepreload, `<img>` dims, PNG→WebP) and the Phase 2 lessons-learned (Suspense + renderToString = React #419 → rollback), I generated these gray areas:

1. **Code-split scope** — which routes to lazy-load, given Phase 2's prerender constraint
2. **Suspense fallback design** — what shows during the lazy chunk fetch
3. **modulepreload injection point** — Vite default vs. prerender.mjs explicit
4. **`<img>` dimension declaration** — HTML attributes vs. CSS aspect-ratio
5. **PNG → WebP tooling** — `vite-imagetools` plugin vs. postbuild `sharp` script
6. **Image format strategy** — WebP-only, WebP+PNG fallback, or AVIF+WebP+PNG
7. **Build pipeline shape** — minimal addition vs. broader refactor
8. **Hydration-mismatch protection** — re-affirm `__PRERENDER_PATH` discriminator, or modify it
9. **Measurement gate** — what counts as "done" for an unmeasurable goal like "feels faster"

## Auto-selected decisions (recommended option per area)

| Gray area | Auto-selected | Why recommended |
|---|---|---|
| **Code-split scope** (D-01) | Lazy ONLY SPA-fallback routes (`/pay`, `/account`, `/success`, `/dashboard/download-skill`); keep prerendered routes static | Phase 2 hotfix `80b16be` proved `<Suspense>` breaks `renderToString`. SPA-fallback routes use `createRoot` per Phase 2 D-06, so their lazy boundaries never enter SSR. |
| **Suspense fallback** (D-02) | Single shared `<RouteLoading>` — full-viewport dark spinner | Per-route skeletons triple the code-split benefit (skeletons would ship in main chunk). Dark spinner consistent with existing extension-coupled UX. |
| **modulepreload** (D-03) | `prerender.mjs` injects `<link rel="modulepreload">` into every emitted HTML's `<head>` | Vite's default behavior is unreliable across SSR-prerendered output paths; explicit injection in our own script is auditable. Save 1-2 RTT on mobile. |
| **`<img>` dimensions** (D-04) | `width="N" height="N"` HTML attributes, derived from native PNG size + Tailwind size classes | HTML attributes work BEFORE CSS parses; `aspect-ratio` CSS works only AFTER. CLS prevention requires the former. |
| **PNG→WebP tool** (D-05) | `vite-imagetools` plugin with `?w=N&format=webp;png&as=picture` syntax | Integrates with Vite's module graph; outputs ready-to-render `<picture>` JSX. Alternatives (postbuild sharp, vite-plugin-image-optimizer) require more bespoke code. |
| **Image format** (D-06) | WebP + PNG fallback via `<picture>`; NO AVIF | WebP = 98%+ support, ~70-80% size reduction. AVIF adds another 10-15% but iOS Safari < 16 decoder is shaky and encoder is 3-5× slower. Defer AVIF. |
| **Build pipeline** (D-07) | Keep current chain; add `vite-imagetools` plugin + extend `prerender.mjs` | Atomic-commit pattern from Phase 1/2 stays viable. No new top-level scripts. |
| **Hydration protection** (D-08) | Lock `__PRERENDER_PATH` discriminator unchanged | Phase 2 hotfix invariant — cost a Vercel rollback when broken. Re-locking as hard constraint. |
| **Measurement** (D-09) | Lighthouse mobile ≥ 90 + CLS = 0 on production, plus manual tap-to-handler test ≤ 500ms on Slow 4G | Vercel exposes Lighthouse; "buttons don't respond" maps to Total Blocking Time. |
