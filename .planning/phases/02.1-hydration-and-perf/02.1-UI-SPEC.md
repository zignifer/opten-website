## New Component: RouteLoading.tsx

**File:** `src/app/components/RouteLoading.tsx`

**Purpose:** Suspense fallback shown for ~100-500ms while lazy-loaded chunks
(PayPage, AccountPage, SuccessPage, DownloadSkillPage) download on first
navigation to those routes. Only activates on the `createRoot` code path
(SPA-fallback routes). Never shown on prerendered routes (/, /welcome, /privacy,
/terms, /refund).

### Visual Specification

| Property | Value | Source |
|----------|-------|--------|
| Background | `#011417` (brand dark) | D-02; matches html/body background in theme.css |
| Layout | Full viewport, centered | `min-h-screen flex items-center justify-center` |
| Spinner color | `white/20` border + `white` top border | Matches white text convention in Phase 2 landing |
| Spinner size | 32px × 32px | `h-8 w-8` |
| Spinner shape | Circle, 2px border | `rounded-full border-2` |
| Spinner animation | CSS `animate-spin` (Tailwind built-in) | No custom keyframes needed |
| Implementation | CSS-only, no SVG, no library | Per D-02; ~200 bytes total |

### Exact Markup Contract

```tsx
// src/app/components/RouteLoading.tsx
export function RouteLoading() {
  return (
    <div
      className="min-h-screen flex items-center justify-center bg-[#011417]"
      role="status"
      aria-label="Loading"
    >
      <div className="h-8 w-8 rounded-full border-2 border-white/20 border-t-white animate-spin" />
    </div>
  );
}
```

### Accessibility

| Property | Value |
|----------|-------|
| Role | `role="status"` on outer div |
| ARIA label | `aria-label="Loading"` |
| Reduced motion | Tailwind's `animate-spin` respects `prefers-reduced-motion: reduce` via the default Tailwind config — spinner stops when motion is reduced. No additional work required. |
| Screen reader | `role="status"` is a live region; screen reader announces the label. |

### Behavioral Contract

| Scenario | Behavior |
|----------|----------|
| User navigates to `/pay` cold | Spinner shows for ~100-500ms, then PayPage renders |
| User navigates to `/pay` with warm cache | Spinner may not be visible (chunk already cached) |
| User navigates to `/` | Spinner NEVER shows (App is a static import, not lazy) |
| User navigates to `/welcome` | Spinner NEVER shows (WelcomePage is a static import) |
| Spinner duration | Determined by network speed; no artificial minimum delay |
| Spinner dismissed | React automatically replaces Suspense fallback when chunk resolves |

## Picture Element Pattern

**Purpose:** Wrap every PNG `<img>` in App.tsx in a `<picture>` element so the
browser receives WebP (smaller) with PNG as fallback. Visual output is
pixel-identical to the current `<img>` — the wrapping is a structural change
only.

### HTML Structure Contract (D-06)

```html
<picture>
  <source srcset="...image.webp" type="image/webp" />
  <img
    src="...image.png"
    width="N"
    height="N"
    alt="..."
    class="..."
    loading="eager|lazy"
  />
</picture>
```

Rules:
- Always WebP `<source>` first, PNG `<img>` fallback.
- No AVIF source in this phase (deferred per D-06).
- The `<img>` element retains all existing Tailwind `className` values unchanged.
- `width` and `height` attributes are always present on the `<img>` (see
  Width/Height Attribute Policy below).
- `<picture>` element itself carries no className — it is a structural wrapper.
- `<source>` element carries no width, height, or className.

### Picture Wrapper Component Contract

**File:** `src/app/components/Picture.tsx`

```tsx
// Thin wrapper over vite-imagetools@6.2.9 PictureData output
import type { Picture as PictureData } from 'vite-imagetools'

interface PictureProps {
  data: PictureData;
  width: number;
  height: number;
  alt: string;
  className?: string;
  loading?: "eager" | "lazy";
  fetchPriority?: "high" | "low" | "auto";
}
```

The `Picture` component renders `<picture><source>...<img>...</picture>`. It
is the only call site for `<picture>` elements in this phase.

### vite-imagetools Import Pattern

## Width/Height Attribute Policy (D-04)

**Rule:** Every `<img>` in `src/app/` must carry explicit `width` and `height`
HTML attributes. Values are derived from either:

1. The native PNG dimensions in `scripts/img-dims.json` (for PNG images), or
2. The Tailwind size class on the element (for fixed-size SVG icons), or
3. The SVG viewBox (for variable-size SVGs).

The attributes give the browser the aspect ratio before CSS loads, eliminating
Cumulative Layout Shift. The CSS Tailwind classes still control rendered size;
the attributes are the CLS hint only.

### Loading Policy Summary

| Zone | Policy | Elements |
|------|--------|----------|
| Above the fold (navbar, hero) | `loading="eager"` | Logo, chrome-lg, yandex-lg, chrome-sm, yandex-sm, highlight.svg |
| High-priority hero images | `loading="eager" fetchPriority="high"` | chrome-lg, yandex-lg (most visible images in first viewport) |
| Borderline (Partners, immediately below hero) | `loading="eager"` | Partner logos ×4 |
| Below the fold | `loading="lazy"` | Feature card images, steps image, cross.svg, optenicon.svg, stars.svg |
