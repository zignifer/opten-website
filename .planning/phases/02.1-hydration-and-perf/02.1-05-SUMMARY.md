## Wave 0 Probe Findings

**A1 Assumption (RESEARCH.md) — WRONG:** vite-imagetools@6.2.9 `?as=picture` does NOT return `{ sources: [{ srcset, type }] }` (array). It returns:
```js
{
  sources: { webp: "/assets/img.webp 1100w" },  // object keyed by format
  img: { src: "/assets/img.png", w: 1100, h: 424 }
}
```
The `sources` field is a `Record<string, string>` mapping format names to srcset strings.

**A2 Assumption (RESEARCH.md) — PARTIALLY CORRECT:** SSR build did pass, but required an additional config fix (see deviations). With `exclude: ''` in imagetools(), the SSR build processes imports and generates the same data shape as the client build.

**Path resolution finding:** Import paths using `/assets/...` URL-style (as specified in PLAN.md interfaces) fail — vite-imagetools resolves from the filesystem, not Vite's URL layer. Files in `public/` must be imported with relative paths: `../../public/assets/...` from `src/app/App.tsx`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] imagetools default exclude blocks public/ directory**
- **Found during:** Wave 0 probe + Task 2 first build
- **Issue:** vite-imagetools has `exclude: 'public/**/*'` as default. Imports from `../../public/assets/` matched this exclusion, so the plugin passed files through as raw URL strings (no WebP generated).
- **Fix:** Changed `imagetools()` to `imagetools({ exclude: '' })` in `vite.config.ts`. Empty string overrides the default exclusion pattern.
- **Files modified:** `vite.config.ts`
- **Commit:** 5e6bdd0

**2. [Rule 1 - Bug] RESEARCH.md A1 assumption wrong — sources shape is object not array**
- **Found during:** Task 2 first successful build — SSR prerender crashed with `data.sources.map is not a function`
- **Issue:** RESEARCH.md assumed `as=picture` returns `{ sources: [{ srcset, type }] }` (array). Actual vite-imagetools@6.2.9 returns `{ sources: { webp: "srcset_string" }, img: { src, w, h } }` (object keyed by format name).
- **Fix:** Updated `Picture.tsx` to treat `sources` as `Record<string, string>`, extract `webpSrcset = sourcesObj['webp']`, and render a single `<source srcSet={webpSrcset} type="image/webp">` tag.
- **Files modified:** `src/app/components/Picture.tsx`
- **Commit:** 5e6bdd0

**3. [Rule 3 - Blocking] Import path must be relative filesystem path, not /assets/ URL**
- **Found during:** Wave 0 probe
- **Issue:** The PLAN.md interfaces section specified import paths like `'/assets/landing-design/feature-models.png?w=...'`. vite-imagetools resolves imports via the filesystem, not Vite's URL-to-public/ mapping. The `/assets/...` path was missing the `public/` prefix.
- **Fix:** All 12 imports use `'../../public/assets/...'` relative to `src/app/App.tsx`.
- **Files modified:** `src/app/App.tsx`
- **Commit:** 5e6bdd0
