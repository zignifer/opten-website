interface PictureProps {
  data: PictureData;
  width: number;
  height: number;
  alt: string;
  className?: string;
  loading?: "eager" | "lazy";
  fetchPriority?: "high" | "low" | "auto";

        src={data.img.src}
        width={width}
        height={height}
        alt={alt}

### `scripts/prerender.mjs` (MODIFIED — add `applyModulePreload` function)

**Role:** Postbuild HTML orchestrator. Reads `dist/index.html`, splices per-route head/body, writes per-route HTML files. Change: add a new `applyModulePreload(html)` function called BEFORE `applyMarker()` in the route loop.

**Existing function pattern** (the three existing splice functions — `src/prerender.mjs` lines 34–71):

`applyMeta` pattern (lines 34–51) — regex replace with fallback guard:
```js
function applyMeta(html, meta) {
  const before = html;
  html = html.replace(/<title>[^<]*<\/title>/, `<title>${escapeHtml(meta.title)}</title>`);
  // ... more replaces ...
  if (html === before) {
    throw new Error(`prerender(${meta.path}): no <head> tags matched. ...`);
  }
  return html;
}
```

`applyBody` pattern (lines 54–60) — single replace with existence guard:
```js
function applyBody(html, rendered) {
  const target = '<div id="root"></div>';
  if (!html.includes(target)) {
    throw new Error('prerender: empty <div id="root"></div> not found ...');
  }
  return html.replace(target, `<div id="root">${rendered}</div>`);
}
```

`applyMarker` pattern (lines 63–71) — replace `</head>` with marker + closing tag:
```js
function applyMarker(html, path) {
  const marker = `<script>window.__PRERENDER_PATH=${JSON.stringify(path)};</script>`;
  if (!html.includes("</head>")) {
    throw new Error("prerender: </head> not found — cannot inject route marker");
  }
  return html.replace("</head>", `    ${marker}\n  </head>`);
}
```

**New `applyModulePreload` function** — mirrors the `applyBody` guard pattern (from RESEARCH.md lines 534–553):
```js
function applyModulePreload(html) {
  // Extract main chunk href from Vite-emitted <script type="module" crossorigin src="...">
  const match = html.match(/src="(\/assets\/index-[^"]+\.js)"/);
  if (!match) return html; // dev build or no entry script — skip silently
  const chunkHref = match[1];
  const tag = `    <link rel="modulepreload" href="${chunkHref}">`;
  return html.replace("</head>", `${tag}\n  </head>`);
}
```

**Route loop call-order pattern** (CRITICAL — from RESEARCH.md Pitfall 5, lines 465–470):

Current loop (`scripts/prerender.mjs` lines 74–95):
```js
for (const meta of routes) {
  let html = applyMeta(template, meta);
  html = applyMarker(html, meta.path);    // ← replaces </head> → removes the string
  if (meta.prerender === "full") {
    html = applyBody(html, renderRoute(meta.path));
  }
  // ...
}
```

New loop order:
```js
for (const meta of routes) {
  let html = applyMeta(template, meta);
  html = applyModulePreload(html);        // ← NEW: must come BEFORE applyMarker
  html = applyMarker(html, meta.path);    // ← existing: consumes </head>
  if (meta.prerender === "full") {
    html = applyBody(html, renderRoute(meta.path));
  }
  // ...
}
```

**Risk callouts (CRITICAL):**
- `applyModulePreload` MUST be called BEFORE `applyMarker`. `applyMarker` replaces the `</head>` string. After `applyMarker` runs, the string `</head>` no longer exists in `html`. A subsequent call to `applyModulePreload` using `html.replace("</head>", ...)` would match nothing and silently fail.
- The RESEARCH.md confirms (line 138) that `<link rel="modulepreload">` must be placed AFTER the Paddle `<script>` tag. The `replace("</head>", ...)` pattern appends before `</head>`, which is after the existing Paddle `<script>` tag (which is inline in the source `index.html`). This ordering is automatically correct.
- The `__PRERENDER_PATH` script marker injected by `applyMarker` must remain — do not remove or reorder after the new function is added.
- The regex `/src="(\/assets\/index-[^"]+\.js)"/` matches the Vite-emitted `<script type="module" crossorigin src="/assets/index-HASH.js">`. Verify against actual `dist/index.html` output from `npm run build` in Wave 0.
