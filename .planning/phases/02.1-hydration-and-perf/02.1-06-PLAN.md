<objective>
Extend `scripts/prerender.mjs` with a new `applyModulePreload(html)` function that injects `<link rel="modulepreload" href="/assets/index-HASH.js">` into the `<head>` of every emitted HTML file. The function synthesizes the tag by extracting the entry-chunk href from the Vite-emitted `<script type="module" crossorigin src="...">` (Vite 6 does NOT emit `modulepreload` by default â€” confirmed in RESEARCH.md line 122â€“130).

Effect: the browser begins fetching the main JS chunk during HTML parse instead of after parsing the `<script>` tag at end-of-body, saving 1â€“2 RTT on mobile networks (RESEARCH.md line 393â€“394).

**Ordering contract (Pitfall 5, RESEARCH.md lines 465â€“470):**
`applyMarker` replaces the `</head>` literal with `marker + </head>`. After it runs, the string `</head>` no longer exists in the buffer. Therefore `applyModulePreload` MUST run BEFORE `applyMarker` â€” both use `html.replace("</head>", ...)`.

**Paddle ordering (Integration Contract Â§6, CONTEXT.md):**
The Paddle CDN `<script>` tag is INSIDE the source `index.html` template and persists through all splice operations. The `<link rel="modulepreload">` is inserted at the very end of `<head>` (immediately before `</head>`), which is AFTER Paddle's `<script>` tag (which is in the upper portion of `<head>`). `modulepreload` only hints the browser; it does NOT change script execution order. Paddle still loads synchronously before `main.tsx` runs.

**SC mapping:** SC-6 (no regression â€” build still green, prerender still emits same 6 HTML files). Enables faster hydration on `/` (perceived TBT win, not a hard SC).
</objective>
