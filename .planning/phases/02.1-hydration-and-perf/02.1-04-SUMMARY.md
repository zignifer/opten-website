# Phase 2.1 Plan 04: Add width/height attrs to all <img> tags (CLS=0) Summary

**One-liner:** Adds explicit `width`, `height`, `loading`, and selective `fetchPriority` HTML attributes to every `<img>` in `src/app/` so the browser reserves the correct box via the aspect-ratio-from-attributes algorithm before CSS loads, eliminating Cumulative Layout Shift.

## Implementation Notes

**BrowserIcons:** Added `const px` and `const fp` derivations inside the component body so both `<img>` tags in the component get correct attrs from the single `size` prop. No prop API changes needed.

**FeatureCard:** Extended prop types with `imgWidth: number` and `imgHeight: number`. Both `<img>` nodes inside the component (imageFirst and !imageFirst branches) updated. Four call sites updated with explicit dimensions — card4 uses 1100×372, cards 1-3 use 1100×424.

**OptenHeroAnimation.tsx:** NOT touched — its `<img>` tags already carry inline `style={{ width: N, height: N }}` (Phase 2 hotfix invariant per UI-SPEC line 247).

**Extension-coupled constants:** `EXTENSION_IDS`, `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `CHROME_STORE_URL` in PayPage.tsx and AccountPage.tsx were not touched.
