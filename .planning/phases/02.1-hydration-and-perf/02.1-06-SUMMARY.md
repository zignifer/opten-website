# Phase 2.1 Plan 06: modulepreload injection into prerendered HTML

Adds `applyModulePreload()` to `scripts/prerender.mjs`. Vite 6 does not emit `<link rel="modulepreload">` for the entry chunk under default configuration. This function synthesizes the tag from the Vite-emitted `<script type="module" crossorigin src="/assets/index-HASH.js">` tag and injects it before `</head>` in all 6 prerendered HTML files, allowing the browser to begin fetching the main JS chunk during HTML parse — saving 1–2 RTT on mobile networks.

## Route Loop: Before / After

**Before (plan 05 baseline):**
```js
for (const meta of routes) {
  const ogImage = meta.ogImage ?? DEFAULT_OG_IMAGE;
  let html = applyMeta(template, meta);
  html = applyMarker(html, meta.path);       // consumed </head>
  if (meta.prerender === "full") {
```

**After (plan 06):**
```js
for (const meta of routes) {
  const ogImage = meta.ogImage ?? DEFAULT_OG_IMAGE;
  let html = applyMeta(template, meta);
  html = applyModulePreload(html);            // Phase 2.1 D-03: must precede applyMarker
  html = applyMarker(html, meta.path);        // consumes </head>
  if (meta.prerender === "full") {
```
