<objective>
Add explicit `width` and `height` HTML attributes to every `<img>` tag in `src/app/`, plus appropriate `loading` and (selectively) `fetchPriority` attributes per the patch table in UI-SPEC.md lines 222–245. Mechanical patch — no logic changes, no new imports, no behavior changes beyond the browser now reserving the correct box for each image before its bytes arrive.

Attributes are read by Chrome 79+, Safari 14+, Firefox 71+ as a hint for the `aspect-ratio-from-attributes` algorithm (CONTEXT.md D-04 rationale) and eliminate Cumulative Layout Shift from image streaming.

- `src/app/App.tsx` — ~15 `<img>` tags.

- `src/app/pages/AccountPage.tsx` — 1 `<img>` (logo).

- `src/app/components/OptenHeroAnimation.tsx` — its `<img>` tags already use inline `style={{ width: N, height: N }}` (UI-SPEC line 247; PATTERNS.md Critical Invariant #1). DO NOT touch.
- PNG → WebP `<picture>` wrapping — plan 02.1-05.

**SC mapping:** SC-4 (CLS = 0) primary; SC-6 (no regression) gate.
</objective>
