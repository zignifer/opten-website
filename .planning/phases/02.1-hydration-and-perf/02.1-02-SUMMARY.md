**One-liner:** Net-new `Picture` wrapper component that consumes vite-imagetools `?as=picture` PictureData and renders `<picture><source type="image/webp"><img>` with full width/height/loading/fetchPriority pass-through.

## Component API — Single Source of Truth for Plan 05

### Props Interface

```ts
import type { Picture as PictureData } from 'vite-imagetools'

interface PictureProps {
  data: PictureData;        // vite-imagetools ?as=picture output
  width: number;            // native image width (CLS hint for browser)
  height: number;           // native image height (CLS hint for browser)
  alt: string;              // img alt text
  className?: string;       // Tailwind classes — passed to <img>, NOT <picture>
  loading?: "eager" | "lazy";
  fetchPriority?: "high" | "low" | "auto";
}
```

### Call-site syntax (plan 05 usage)

```tsx
import { Picture } from "./components/Picture";
import featureModelsSrc from '/assets/landing-design/feature-models.png?w=1100&format=webp;png&as=picture'

<Picture
  data={featureModelsSrc}
  width={1100}
  height={424}
  alt=""
  className="mx-8 h-auto w-[calc(100%-64px)] rounded-[4px] object-contain"
  loading="lazy"
/>
```

### Rendered HTML output

```html
<picture>
  <source srcset="...image.webp" type="image/webp" />
  <img
    src="...image.png"
    width="1100"
    height="424"
    alt=""
    class="mx-8 h-auto ..."
    loading="lazy"
  />
</picture>
```

### Hard contracts (do not violate in plan 05)

- `<picture>` carries NO `className` (UI-SPEC.md line 143)
- `<source>` carries NO `width`, `height`, or `className` (UI-SPEC.md line 143)
- `fetchPriority` is camelCase in JSX (React maps it to `fetchpriority` HTML attribute)
- `key` on `<source>` is `source.type` — the MIME type string (e.g. `"image/webp"`)
