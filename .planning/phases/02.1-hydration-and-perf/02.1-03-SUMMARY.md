# Phase 2.1 Plan 03: RouteLoading.tsx Suspense Fallback Summary

**One-liner:** CSS-only full-viewport dark spinner on `bg-[#011417]` with `role="status"` accessibility, ~200 bytes, unused until plan 07 wires it as Suspense fallback.

## What Was Built

Created `src/app/components/RouteLoading.tsx` — the Suspense fallback component for lazy-loaded routes (`/pay`, `/account`, `/success`, `/dashboard/download-skill`). The component renders when React suspends while downloading a lazy chunk, for approximately 100–500ms on first navigation to those routes.

**Shipped markup (exact — as plan 07's executor can import without re-reading UI-SPEC):**

```tsx
// Phase 2.1 D-02: Full-viewport dark spinner — Suspense fallback for lazy-loaded routes.
// Net-new file. Not yet imported anywhere; plan 02.1-07 wires it into main.tsx.
export function RouteLoading() {
  return (
    <div
      className="min-h-screen flex items-center justify-center bg-[#011417]"
      role="status"
      aria-label="Loading"
    >
      <div className="h-8 w-8 rounded-full border-2 border-white/20 border-t-white animate-spin" />
    </div>
  );
}
```

**Plan 07 import path (for the executor's reference):**

```tsx

// Usage:

  <Routes>...</Routes>
</Suspense>
```
