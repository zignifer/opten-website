# Phase 4.2 / Plan 03 — Catch-all 404 + locale-aware NotFound + meta noindex

<objective>
Replace today's soft-404 behaviour (Vercel rewrite serves the prerendered
landing for any unknown path → 200 OK + landing-like content) with a real
404 surface: a `<Route path="*">` catch-all that renders a minimal
locale-aware `NotFound` component and injects `<meta name="robots"
content="noindex">` at runtime so search engines stop indexing typo'd or
deprecated URLs as duplicates of the landing.

Purpose: close A-5 from the synthesis (no catch-all 404). Avoids the
"every random URL ranks for landing keywords" SEO trap and gives users a
real "this page does not exist" signal instead of confusion.

Output: 1 new component, 2 router edits, 3 i18n keys × 2 locales,
1 atomic commit.

Hard constraint: this does NOT change the Vercel rewrite rule
(`/((?!api/).*) → /index.html`). The catch-all is React-Router-level only;
Vercel still serves `dist/index.html` (HTTP 200) for unknown paths — the
catch-all then re-renders client-side as the 404 view + injects the
noindex meta. This is the standard SPA-with-prerender pattern; status
code remains 200 (changing it would require a Vercel edge function, out
of scope per CONTEXT "no architectural refactor").

The 404 surface is NOT added to `scripts/seo-routes.ts` (no per-route
prerender for `*`). It is purely client-rendered post-hydration — which is
why the noindex meta must be JS-injected, not baked.
</objective>
