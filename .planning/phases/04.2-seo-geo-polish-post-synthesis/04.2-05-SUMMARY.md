### 2. `public/36ced30e188f58b51d26c30e483124d5.txt` — IndexNow ownership key

### 3. `scripts/indexnow.mjs` — Postbuild ping (95 lines)
Best-effort by design:
- Reads `dist/sitemap.xml`, extracts every `<loc>` URL via regex.
- POSTs `{host, key, keyLocation, urlList}` JSON to `https://api.indexnow.org/indexnow` with a 5-second `AbortController` timeout.
- Logs `✓ indexnow → pinged N URLs (status 200/202)` on success.
- Logs `⚠ indexnow → ping failed (...)` on any network error and **continues** — the build still exits 0. IndexNow is a notification, not a precondition; search engines will recrawl on their own schedule eventually.
- Throws (and fails the build) ONLY on programmer error: missing key file in `public/`, or sitemap with < 16 `<loc>` entries.
- Honours `INDEXNOW_SKIP=1` env var to bypass the ping entirely — for local iteration when you don't want to spam api.indexnow.org from your dev machine.

Key discovery: scans `public/` for `^[a-f0-9]{8,128}\.txt$` and picks the first match. Single source of truth is the file system — the script and the key file cannot drift because the script reads the key from the filename.

### 4. `package.json` — Wires indexnow.mjs as LAST postbuild step
Build chain now ends with `... && node scripts/verify-faq-mainentity.mjs && node scripts/indexnow.mjs`. The FAQ parity gate still runs second-to-last, so the build artifact is complete before the optional ping fires. Local builds and CI builds without network access work fine — the script degrades to a warning.
