# Phase 4.2 / Plan 04 — Welcome carousel: unique alts + lazy non-active slides

<objective>
Replace the 4 identical `alt="Opten extension"` strings on the welcome
carousel with unique descriptive alts (one per slide, sourced from i18n),
and switch all non-active slides from `loading="eager"` to `loading="lazy"`.

Purpose: closes A-7 from the synthesis. Identical alts give AI crawlers
zero differential signal between the 4 onboarding screenshots and create
an accessibility regression (4× same alt = screen reader spam). Eager-
loading all 4 slides also pushes LCP/network outliers — only slide 1 is
visible on first paint; the other 3 should defer until the carousel
rotates (every 3 seconds per the `setInterval` in lines 26-31).

Output: 1 file modified (`WelcomePage.tsx`), 8 i18n keys added (4 × 2
locales), 1 atomic commit.

Hard constraint: `/welcome` is a locked route (extension first-install
target — see CLAUDE.md "Locked routes"). This change is CONTENT-ONLY —
no DOM structure change, no animation change, no slide-source change. Only
the `alt` strings and `loading` attribute change. Per CONTEXT "Welcome
Carousel Hygiene": no `decoding="async"` (called out as optional in
CONTEXT — skip it to keep the diff minimal).

This wave INTENTIONALLY ships BEFORE Wave 6 (the `<h1>` addition) so that
the approval-gated Wave 6 diff is small and isolated to the `<h1>` line.
</objective>
