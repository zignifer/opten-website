# Phase 4.2 Plan 03: Catch-all 404 + Locale-aware NotFound + Meta Noindex Summary

Closes synthesis item A-5 / requirement P1-1: replaces the soft-404 fallback (Vercel SPA rewrite served `dist/index.html` = landing content for any unknown path → 200 OK with landing-duplicate text indexed by Google) with a real React-Router-level `<Route path="*">` catch-all that renders a minimal locale-aware NotFound view and injects `<meta name="robots" content="noindex,nofollow">` at runtime.

## What Changed

### Task 1 — `NotFound.tsx` + i18n keys

New component at `src/app/pages/NotFound.tsx` (75 lines). Imports `useEffect` from React, `useT` from `../../i18n/LangContext`, and the default-exported `LocalizedLink` from `../components/LocalizedLink`.

**Markup:** single `<main>` container, `max-w-[640px]` centered, `py-[96px]`, matching the legal-page minimal-page convention. Renders:
- `404` superscript (uppercase, tracked, 40% opacity black) as the visual code marker
- H1 = `t("notFound.title")`
- Body paragraph = `t("notFound.body")`
- `<LocalizedLink to="/">` button = `t("notFound.home")`

`LocalizedLink to="/"` resolves to `/` on RU paths and `/en/` on EN paths via the `EN_SIBLINGS` lookup in `src/i18n/paths.ts` — EN visitors who land on a typo'd `/en/bogus-path` get sent back to `/en/`, not `/`.

**`useEffect` body:**

1. Snapshot the prior `document.title` and set it to `${t("notFound.title")} — Opten`.
2. Query for an existing `meta[name="robots"]`:
   - If present: snapshot its `content` and set to `"noindex,nofollow"`.
   - If absent: `createElement("meta")` → `setAttribute("name", "robots")` → `appendChild` to `document.head`, then set content to `"noindex,nofollow"`. Track via a `createdMeta` boolean so cleanup only removes our own injection.
3. Cleanup function:
   - Restore the prior `document.title`.
   - If we created the meta: remove it from `<head>`.
   - If we reused an existing meta: restore its prior content.

Dependency array is `[t]`, so the effect re-runs when `t` identity changes (LangProvider re-renders on language flip) — title and meta update on RU↔EN toggle without a remount.

**i18n keys** added to `src/i18n/ru.json` and `src/i18n/en.json` between `footer.copyright` (line 109) and `pay.title` (line 110):

```jsonc
// ru.json
"notFound.title": "Страница не найдена",
"notFound.body": "Извините, такой страницы нет. Возможно, вы перешли по устаревшей ссылке.",
"notFound.home": "Вернуться на главную",

// en.json
"notFound.title": "Page not found",
"notFound.body": "Sorry, this page does not exist. The link may be outdated.",
"notFound.home": "Back to home",
```

Distinct from the existing `guide.notFound.*` namespace used by `GuidePage.tsx` (those are scoped to a specific guide-slug miss, not a generic 404).

### Task 2 — Catch-all wiring in both routers

**`src/main.tsx`:**
- Added eager import: `import NotFound from "./app/pages/NotFound.tsx";` next to the other top-level page imports.
- Added `<Route path="*" element={<NotFound />} />` as the LAST entry inside `<Routes>`, after `<Route path="/en/guides/:slug" .../>`. React Router 7 matches in declaration order, so the `*` only fires when every specific route has missed.

Eager (not `React.lazy`) per plan rationale: the component is ~75 lines, hydration must fire the noindex `useEffect` immediately, and a lazy chunk would flash a `<RouteLoading />` fallback for every unknown path.

**`scripts/entry-server.tsx`:**
- Added the same eager import (path adjusted to `../src/app/pages/NotFound.tsx`).
- Added `<Route path="*" element={<NotFound />} />` as the LAST entry inside the SSR `<Routes>`.

The SSR mirror is defense in depth: `scripts/prerender.mjs` only iterates the manifest in `scripts/seo-routes.ts`, so NotFound is never SSR-rendered today. The mirror means a future plan that adds a new SSR-rendered path can mistype the `Route path=` value without silently rendering empty HTML.

## What's Explicitly Out of Scope

- **HTTP 200 → 404 status code change.** Vercel's SPA rewrite rule (`/((?!api/).*) → /index.html`) is unchanged, so unknown paths still return 200 OK with the prerendered landing HTML — the catch-all then re-renders client-side as the 404 view post-hydration. Converting to a true HTTP 404 requires a Vercel edge function or static error page; deferred per CONTEXT D-3 to the Phase 6 architectural refactor. The `<meta name="robots" content="noindex,nofollow">` injection mitigates the indexing risk in the interim.
- **`/404` entry in `scripts/seo-routes.ts`.** Catch-all is client-rendered only; no prerender, no canonical, no schema, no manifest entry, no sitemap presence.
- **`EN_SIBLINGS` mutation in `src/i18n/paths.ts`.** There is no 404 EN sibling — `LocalizedLink to="/"` already routes EN visitors back to `/en/`.
