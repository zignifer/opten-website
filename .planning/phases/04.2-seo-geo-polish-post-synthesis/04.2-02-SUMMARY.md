## What Changed

### Task 1 — `public/sitemap.xml` deletion (P0-2 / synthesis S-2)

`public/sitemap.xml` was a 6-URL fallback that had been overwritten on every build for months by `scripts/sitemap.mjs`, which writes the canonical 16-route sitemap directly to `dist/sitemap.xml` with reciprocal `xhtml:link` hreflang and per-route git-mtime `lastmod`. The fallback was a trap: a partial postbuild failure would let Vite copy the stale file into `dist/` and onward to production. Removing it makes the dist/ artifact single-sourced.

`scripts/sitemap.mjs` was already independent — it `writeFile`s to `dist/sitemap.xml` directly and never read `public/sitemap.xml`. No code changes were needed.

### Task 2 — `Disallow: /api/` across all 16 robots blocks (P0-4a / synthesis A-2)

Added two lines after the existing `Disallow: /en/dashboard/` in every block:

```
Disallow: /api/
Disallow: /en/api/
```

Applied to:
- 1× wildcard `User-agent: *`
- 15× named AI crawlers: GPTBot, ChatGPT-User, OAI-SearchBot, ClaudeBot, anthropic-ai, PerplexityBot, Google-Extended, Bingbot, YandexBot, Applebot-Extended, cohere-ai, Meta-ExternalAgent, Amazonbot, CCBot, Bytespider

The auth-gated `/api/download-skill` endpoint stops being a free crawl target. The `Disallow: /en/api/` line is defense-in-depth — no EN-prefixed API exists today, but the catch-all rewrite plus future-proofing justifies the cheap second line. Pairs with the `X-Robots-Tag` header work scheduled in Wave 7 (approval-gated, separate commit).

Preserved unchanged: per-vendor comments, block ordering, blank-line separators, the `Allow: /` line at the top of each block (robots.txt longest-match precedence means the more-specific `/api/` Disallow wins), the existing `Disallow: /account` / `/success` / `/dashboard/` set from Phase 4 + the morning batch, the `Sitemap: https://opten.space/sitemap.xml` footer, and the `Content-Signal: search=yes, ai-train=yes, ai-input=yes` directive (D-14).
