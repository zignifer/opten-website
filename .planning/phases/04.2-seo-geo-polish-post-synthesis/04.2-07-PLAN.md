# Phase 4.2 / Plan 07 â€” X-Robots-Tag headers for service routes (vercel.json)

<objective>
Add `X-Robots-Tag: noindex, nofollow` HTTP response headers in `vercel.json`
for the SPA-only / extension-coupled / API surfaces that should never appear
in search results: `/account`, `/success`, `/dashboard/*`, `/api/*`, and
the EN counterparts where applicable.

Purpose: closes the second half of A-3 (P0-4b in synthesis). The robots.txt
Disallow set (delivered in Wave 2) handles AI crawlers, but a non-JS
generic crawler that ignores robots can still hit a Vercel response for
these paths. The `X-Robots-Tag` HTTP header is a belt-and-suspenders
backstop â€” it is what Google specifically recommends for SPA / serverless
routes that cannot reliably ship a `<meta name="robots">` in their initial
HTML.
