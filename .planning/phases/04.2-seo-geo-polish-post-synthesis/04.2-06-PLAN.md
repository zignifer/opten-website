# Phase 4.2 / Plan 06 — Visible `<h1>` on /pay + /welcome (RU + EN)

<objective>
Add a single semantic `<h1>` element to each of the four routes (/pay,
/en/pay, /welcome, /en/welcome — same components serve both locales). All
four routes are full-prerender + indexable yet currently have NO visible
`<h1>` in the body, which is a classical SEO miss flagged by synthesis A-1.

Purpose: heading hierarchy + answer-block-near-top extraction (both for
Google + AI citation engines). A single visible `<h1>` per page is the
strongest signal a route can carry, and the absence is conspicuous given
that landing/about/guides all have proper H1s.

Output: 2 components touched (h1 markup only — NO logic, schema, CTA, or
routing change), 4 new i18n keys (2 keys × 2 locales), 1 atomic commit
AFTER user approval.

Hard constraints (LOCKED — see CONTEXT "Visible H1 on Locked Routes" + the
04.1 baseline preservation rules carried forward):

- Locked route paths `/pay`, `/en/pay`, `/welcome`, `/en/welcome` MUST
  remain unchanged.
- `/pay` and `/en/pay` MUST keep their synchronous Paddle CDN `<script>`
  injection (`scripts/prerender.mjs#applyPaddleScript` runs unchanged).
- `/pay` and `/en/pay` MUST keep both pricing cards rendering SSR with
  visible `199₽` and `299₽` (Product schema visible-content gate from
  `f9dfdb1`).
- `/pay` runtime-conditional CTA pattern stays: SSR Chrome Store anchor
  branch, hydration swap to Pay button when extension detected.
- `EXTENSION_IDS`, `SUPABASE_URL`, `SUPABASE_ANON_KEY` constants MUST NOT
  be touched (INTEGRATION-CONTRACT).
- `/welcome` carousel + step list (4 cards) + blinking arrow MUST NOT
  change visually. The h1 lands ABOVE the carousel (which is already the
  natural "first visible thing" — h1 sits above it in the flex column).
- No new schema entries, no new prerender helpers, no new routes.

The h1 markup itself is plain semantic HTML — `<h1 className="…">{t("pay.h1")}</h1>`
inside the existing top-of-page section. NOT a Heading component, NOT
wrapped in motion/animation, NOT styled to be invisible (`sr-only` is
forbidden — defeats the SEO purpose).
</objective>
