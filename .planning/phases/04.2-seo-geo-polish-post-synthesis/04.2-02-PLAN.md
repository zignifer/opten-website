# Phase 4.2 / Plan 02 — Sitemap + robots.txt hygiene

<objective>
Two low-risk hygiene fixes: (1) delete the stale `public/sitemap.xml`
fallback that ships 6 URLs instead of the 16 emitted at build time, and (2)
add `Disallow: /api/` to every per-bot block in `public/robots.txt` so the
auth-gated `/api/download-skill` endpoint stops being a free crawl target.

Purpose: eliminate the "fallback can leak to production if postbuild
breaks" trap (S-2 in synthesis) and harden the API surface against AI
crawler indexing (A-2 in synthesis, paired with the vercel.json
`X-Robots-Tag` work in Wave 7).

Output: 1 file deleted, 1 file modified, 1 atomic commit.
</objective>

<task type="auto">
  <name>Task 1: Delete public/sitemap.xml (P0-2)</name>
  <files>public/sitemap.xml</files>
  <action>
    Delete `public/sitemap.xml` entirely. The build pipeline already emits
    `dist/sitemap.xml` via `scripts/sitemap.mjs` (16 routes, reciprocal
    `xhtml:link` hreflang, per-route git-mtime `lastmod`) — the fallback
    has been overwritten on every build for months, but it is still a
    maintenance trap: a partial postbuild failure could let Vite's plain
    copy of the stale 6-URL fallback into `dist/` and onward to production.
    Deletion is the clean fix (Vite copies what is in `public/` —
    nothing-to-copy means nothing-to-leak).

    Use `git rm public/sitemap.xml` (or `Bash rm` followed by stage in the
    commit). Do NOT replace it with a stub or comment file. The robots.txt
    `Sitemap:` directive stays as-is and continues to point at
    `https://opten.space/sitemap.xml` — Vercel serves whatever file lives
    at `dist/sitemap.xml`, which `scripts/sitemap.mjs` writes on every build.

    No code changes anywhere else are needed. `scripts/sitemap.mjs` already
    writes to `dist/sitemap.xml` directly via `writeFile(resolve(DIST, "sitemap.xml"), …)`
    — it does not read or care whether `public/sitemap.xml` exists.
  </action>
  <verify>
    <automated>npm run build &amp;&amp; ls dist/sitemap.xml &amp;&amp; grep -c "&lt;loc&gt;" dist/sitemap.xml</automated>
    Expected: build exits 0; `dist/sitemap.xml` exists; `<loc>` count = 16 (one per prerendered route).
    Also: `ls public/sitemap.xml` MUST return "No such file or directory" (file deleted).
  </verify>
  <done>`public/sitemap.xml` deleted; `dist/sitemap.xml` still emitted with 16 routes; build clean.</done>
