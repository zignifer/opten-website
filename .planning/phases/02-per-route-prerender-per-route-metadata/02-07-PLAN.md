<objective>
HIGHEST-RISK PLAN OF THE PHASE. Modify `src/main.tsx` to detect whether the route was prerendered and use `hydrateRoot` (for full-tier prerendered routes) or `createRoot` (for empty SPA roots). Ship LAST. Rollback = `git revert <sha>`.

Purpose: Without this change, React's `createRoot` blows away the SSR-rendered DOM on the first client render — the GEO win remains in the initial HTTP response but the post-paint DOM is client-rendered (defeats Pitfall 3 hydration story). With this change, hydration preserves the prerendered tree across the client boundary, eliminates any FOUC, and ensures crawlers that DO run JavaScript (Googlebot post-2019) see a consistent DOM before and after JS execution.

Risk profile: This is the only Phase 2 plan that touches `src/main.tsx`, which owns the Paddle bootstrap. The Paddle init order (BEFORE render call) MUST be preserved (DEC-integration-contract-paddle-init / BG-67-01). The 9 client routes MUST stay mounted (the SSR subset in Plan 02-02 was 5 routes; the client tree must keep ALL 9 including the locked extension-coupled routes).

Output: Two-region surgical edit to `src/main.tsx`:
1. Line 2 (single line) — add `hydrateRoot` to the `react-dom/client` named import.
2. Lines 40-56 (the current single `createRoot(...).render(...)` expression) — restructure to extract the JSX tree into a `const tree = (...)` and branch on `document.getElementById("root")!.hasChildNodes()`.

Lines 1, 3-39, and the rest of the file remain byte-identical.
</objective>
