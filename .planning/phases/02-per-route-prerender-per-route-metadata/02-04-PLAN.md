<objective>
Create `scripts/prerender.mjs` — the Node postbuild orchestrator that reads `dist/index.html` (the Vite SPA shell), imports the SSR-compiled bundle and manifest from `.ssr-cache/`, and emits per-route HTML files into `dist/`.

Purpose: The single executable that turns Phase 2's design (per-route metadata + body content) into actual files on disk. This plan WRITES the script but does NOT yet wire it into `npm run build` — that's Plan 02-06. To verify this plan in isolation, the executor runs `vite build` manually, then `vite build --ssr scripts/entry-server.tsx --outDir .ssr-cache --emptyOutDir scripts/seo-routes.ts` (to compile both the SSR entry AND the manifest), then `node scripts/prerender.mjs`.

Output: One new `.mjs` file in `scripts/`, plus 6 emitted HTML files in `dist/` after the script runs (the `/` route overwrites `dist/index.html`; `/pay`, `/welcome`, `/privacy`, `/terms`, `/refund` create `dist/{route}/index.html`).

NOTE: The sitemap.xml emission is split into a separate file (`scripts/sitemap.mjs` in Plan 02-05). THIS plan's prerender.mjs does NOT emit sitemap.xml — it only emits HTML files. Plan 02-06 chains both scripts in `npm run build`.

NOTE on TS import: `scripts/seo-routes.ts` is a TypeScript file. Node cannot import `.ts` directly without flags. The chosen strategy (RESEARCH.md Open Question 1) is to include `scripts/seo-routes.ts` in the SSR Vite build alongside `scripts/entry-server.tsx`, producing `.ssr-cache/seo-routes.mjs`. The prerender script imports `.ssr-cache/seo-routes.mjs` (not `scripts/seo-routes.ts`).
</objective>

Inputs the script reads:
- `dist/index.html` — Vite SPA shell template
- `.ssr-cache/entry-server.mjs` — exports `renderRoute(path: string): string`
- `.ssr-cache/seo-routes.mjs` — exports `routes: RouteMeta[]`, `SITE_ORIGIN`, `DEFAULT_OG_IMAGE`

Outputs the script writes:
- For each route with `prerender === "full"`: `dist/{path}/index.html` (or `dist/index.html` for `/`) — head overridden + React-rendered body spliced into `<div id="root">...</div>`.
- For each route with `prerender === "head"`: `dist/{path}/index.html` — head overridden + body kept empty (`<div id="root"></div>`).
- For each route with `prerender === "none"`: skipped (no file written).

Module-level constants (Pattern S-3 UPPER_SNAKE_CASE):
- `__dirname` recovered from `import.meta.url` (Pattern S-1)
- `ROOT` = `resolve(__dirname, "..")`
- `DIST` = `resolve(ROOT, "dist")`
- `SSR_BUNDLE` = `resolve(ROOT, ".ssr-cache", "entry-server.mjs")`
- `MANIFEST_BUNDLE` = `resolve(ROOT, ".ssr-cache", "seo-routes.mjs")`

Head substitution targets (exactly 7 regex `.replace()` calls + 1 canonical injection — RESEARCH.md Example B):
1. `<title>` → per-route title
2. `<meta name="description">` → per-route description
3. `<meta property="og:url">` → per-route canonical
4. `<meta property="og:title">` → per-route ogTitle
5. `<meta property="og:description">` → per-route ogDescription
6. `<meta name="twitter:title">` → per-route ogTitle (mirrors og:title)
7. `<meta name="twitter:description">` → per-route ogDescription (mirrors og:description)
8. Canonical injection: insert `<link rel="canonical" href="...">` immediately after `<meta name="theme-color">` (line 11 of `index.html`)

Body substitution target (Pattern 5): exact string `<div id="root"></div>` → `<div id="root">${renderedHtml}</div>` (no inner whitespace).

Tags that MUST NOT be touched by ANY `.replace()` call (Phase 1 + integration contract invariants):
- The two `<script type="application/ld+json">` blocks (lines 51-67 + 70-82 of source `index.html`)
- `<link rel="preconnect" href="https://cdn.paddle.com" />` (line 85)
- `<script src="https://cdn.paddle.com/paddle/v2/paddle.js"></script>` (line 86 — sync, no async/defer — DEC-integration-contract-paddle-init)
- `<meta property="og:image">` and `<meta name="twitter:image">` (Phase 2 keeps og-card-ru.png site-wide per Open Question #2)
- `<html lang="ru">` opening tag (D-05 — RU only in Phase 2)
- The favicon `<link>` block (lines 32-48)
- The `<script type="module" src="/src/main.tsx">` body tag (Vite rewrites this to a hashed bundle path)
