<objective>
Create `scripts/entry-server.tsx` — the build-time SSR React tree entry. NEW file. Exports a single function `renderRoute(path: string): string` that wraps the app in `<StaticRouter location={path}>` and returns the result of `renderToString`. The file is later compiled to `.ssr-cache/entry-server.mjs` by `vite build --ssr` (wired in Plan 02-06).

Purpose: Decouple the SSR tree from `src/main.tsx` (which calls `createRoot` at module load — would crash in Node). Only the 5 full-prerender routes are mounted; PayPage/SuccessPage/AccountPage/DownloadSkillPage are deliberately omitted to keep `EXTENSION_IDS` / `SUPABASE_URL` / `SUPABASE_ANON_KEY` out of the SSR build (RESEARCH.md Pitfall 5).

Output: One new TypeScript-with-JSX file at `scripts/entry-server.tsx`.

NOTE: This plan creates the file but does NOT compile it. The SSR Vite build is wired in Plan 02-06 (package.json edit). At the end of this plan, `npm run build` still works (file is unimported by the SPA tree). The LangContext SSR-safety patch in Plan 02-03 is a prerequisite for the SSR compilation in Plan 02-04/06 to succeed — but is not a prerequisite for THIS plan to land the file.
</objective>
