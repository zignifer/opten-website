<objective>
Create `scripts/sitemap.mjs` — a small Node CLI that reads the per-route manifest from `.ssr-cache/seo-routes.mjs` and emits `dist/sitemap.xml` with one `<url>` entry per non-"none"-tier route, each carrying a build-time `<lastmod>` (GEO-B-3 / D-04).

Purpose: Closes GEO-B-3. Phase 1's `public/sitemap.xml` deliberately omitted `<lastmod>` ("Phase 1 milestone-level decision: 'lastmod' omitted; add in Phase 2 when prerender produces per-route build metadata"). This plan delivers that.

Output: One new `.mjs` file at `scripts/sitemap.mjs`. When invoked, it overwrites `dist/sitemap.xml` (which Vite copied from `public/sitemap.xml`) with a new file that has 6 `<lastmod>` entries.

Separation of concerns: Plan 02-04 owns HTML emission; this plan owns XML emission. Both scripts will be chained into `npm run build` by Plan 02-06.

NOTE: `public/sitemap.xml` (the Phase 1 file) is NOT modified. It stays as-is for graceful degradation if the postbuild step ever fails. The shadowing rule (RESEARCH.md Pitfall 7) is that Vite copies `public/sitemap.xml` → `dist/sitemap.xml` first, then this script overwrites the `dist/` copy with the lastmod-enabled version.
</objective>

    - `const BUILD_DATE = new Date().toISOString().split("T")[0];` — Pattern: computed exactly ONCE at script start so all `<lastmod>` entries are identical (D-04 rationale: uniform date because "every push to main is a batch release").

    Construct the XML using a template literal (RESEARCH.md Example B sitemap block — verbatim shape):
    ```
    const xml = `<?xml version="1.0" encoding="UTF-8"?>
    <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
    ${sitemapRoutes.map(r => `  <url>
        <loc>${r.canonical}</loc>
        <lastmod>${BUILD_DATE}</lastmod>
        <changefreq>${r.changefreq}</changefreq>
        <priority>${r.priority.toFixed(1)}</priority>
      </url>`).join("\n")}
    </urlset>
    `;
    ```

    Note `priority.toFixed(1)` produces e.g. `1.0`, `0.8`, `0.6`, `0.3` (matches the Phase 1 `public/sitemap.xml` priority style).

    Write the file: `await writeFile(resolve(DIST, "sitemap.xml"), xml, "utf-8");`.

    Log progress: `console.log(\`✓ sitemap.xml → ${sitemapRoutes.length} routes (lastmod ${BUILD_DATE})\`);`.

    Safety: if `sitemapRoutes.length === 0`, throw `new Error("sitemap.mjs: no routes with prerender != 'none' — manifest mis-loaded?")` to fail loudly on a future manifest regression.

    Do NOT:
    - Modify `public/sitemap.xml` (Phase 1 fallback stays as-is).
    - Add an XML library (hand-written template literal is sufficient — RESEARCH.md "Don't Hand-Roll" justification: "6 routes, fixed schema, no namespaces beyond default urlset").
    - Add hreflang annotations or `<xhtml:link>` elements (Phase 3 scope per CONTEXT.md `<deferred>`).

    - Emit `dist/sitemap.xml` if `routes.length === 0` (defensive: throws instead).
    - Reference any integration-contract constant.
