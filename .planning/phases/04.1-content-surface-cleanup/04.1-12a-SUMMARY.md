# Phase 4.1 Plan 12a: Extract AboutPage body to data file — Summary

**One-liner:** Extracted RU `/about` body into new `src/content/about.tsx` per-language data file, refactored `AboutPage.tsx` to consume `aboutContent[lang] ?? aboutContent.ru` — pure refactor, no visual or copy change, unblocks W10 / W11 / W12b.

## Goal

Prerequisite refactor for the rest of Wave 2-4:

- **W10** (free-tier purge) edits the "По цене" paragraph → now lives in `aboutContent.ru.sections[2]`.
- **W11** (Magnific insertion) edits two paragraphs in sections 1 + 2 → now lives in the same data file.
- **W12b** (EN sibling) needs a single per-language data source for the body → `aboutContent.en` is the
  null sentinel ready to be populated.

Without this extraction, W10/W11 would have to do JSX surgery in AboutPage.tsx, and W12b would either
double the AboutPage size by inlining EN JSX next to RU JSX, or duplicate-render both languages —
both rejected per CLAUDE.md "no new abstractions".

## Filename deviation from seed (`.ts` → `.tsx`)

The captured-decisions seed named the new file `src/content/about.ts`. Practical TypeScript constraint
forced a switch to `.tsx`: the data structure contains inline JSX (`<a>`, `<strong>`,
`<LocalizedLink>`) per the D-Wave-12a "no AST renderer" decision, and `.ts` files cannot contain JSX.
The import site in `AboutPage.tsx` is unaffected — `from "../../content/about"` (no extension) and
TS resolves `.tsx` automatically. Noted in file header comment for downstream awareness.

This deviation does NOT add abstraction (still no AST renderer; the JSX literals are the body) and
matches the gpt-image-2 pattern in spirit (per-language object literals consumed via `useLang()`)
even though that file is `.ts` because its content is pure strings without inline JSX.

## Structural delta vs pre-12a

The only structural difference in the DOM:

```diff
 <article class="about-content ...">
-  <h2>Откуда взялся Opten</h2>
-  <p>...</p>
-  <p>...</p>
-  <p>...</p>
-  <h2>Почему именно про промпты</h2>
-  ...
+  <div><h2>Откуда взялся Opten</h2><p>...</p><p>...</p><p>...</p></div>
+  <div><h2>Почему именно про промпты</h2>...</div>
+  <div><h2>Что вы получаете</h2>...</div>
+  <div><h2>Контакты</h2>...</div>
 </article>
```

The `<div>` is needed for React's keyed-list reconciliation across the `.map()` call. It's a
block-level wrapper inside a block-level `<article>` — zero visual impact, zero a11y impact
(no semantic role override), zero SEO impact (search engines flatten the structure for content
indexing). Acceptable per the wave's "byte-for-byte identical RU body" requirement which
explicitly allows "differ only in inert wrapper element types" per CONTEXT.
