# Phase 04.1 Plan 06: Pricing heading → i18n + trust section id rename — Summary

One-liner: Routed the landing Pricing heading through three new i18n keys (`pricing.heading.{lead,accent,tail}`) preserving the `<Accent>` design split, deleted the dead flat `pricing.heading` key from both dicts, and renamed the trust section `id="privacy"` → `id="trust"` to eliminate the slug collision with the `/privacy` legal route.

## What changed

**`src/i18n/ru.json`** (`+3 / -1`)
- Removed flat dead key `"pricing.heading": "Начни бесплатно. Перейди на Pro, когда будешь готов."` (no `t()` call referenced it).
- Added three composing keys matching the pre-existing inline JSX text:
  - `"pricing.heading.lead": "Скачай бесплатно. "`
  - `"pricing.heading.accent": "Перейди на Pro,"`
  - `"pricing.heading.tail": " когда будешь готов."`

**`src/i18n/en.json`** (`+3 / -1`)
- Removed flat dead key `"pricing.heading": "Start free. Upgrade to Pro when you're ready."`.
- Added three composing keys matching the pre-existing inline JSX text:
  - `"pricing.heading.lead": "Start Free. "`
  - `"pricing.heading.accent": "Upgrade To Pro"`
  - `"pricing.heading.tail": " When You're Ready."`

**`src/app/App.tsx`** (`+6 / -15`)
- `Pricing()` function (lines 418-449): replaced the `isRu ? (...) : (...)` block inside `<SectionTitle>` with a single i18n-driven composition that works for both languages. Kept the dual mobile/desktop variants — they differ only in `<br />` separators on the desktop variant. `<Accent>` wrap is preserved.
- `const isRu = lang === "ru"` retained — still needed for the `onetimePrice` / `proPrice` ternaries below.
- `Privacy()` function (line 393): `<section id="privacy" …>` → `<section id="trust" …>`. Comment block above updated to a one-liner referencing IN-04, with the Phase 4 D-08 history preserved underneath. ClassName, padding, content, and heading inside unchanged.
