Per CONTEXT D-Wave-12a: **keep inline JSX in the data structure rather than building an AST renderer** — simpler, matches CLAUDE.md "no new abstractions". This means the `paragraphs` field of each section is typed as `JSX.Element[]` or `ReactNode[]`, with each paragraph being either a string or a `<p>...</p>` JSX expression containing inline `<a>`, `<strong>`, `<LocalizedLink>` children.

Suggested shape (informed by gpt-image-2.ts but simpler — body doesn't have steps with before/after):
```ts

export interface AboutSection {
  heading: string;        // h2 text
  body: ReactNode;        // freeform JSX subtree (paragraphs, lists, links, strong)
}

export interface AboutContent {
  sections: AboutSection[];
}

const ru: AboutContent = {
  sections: [
    {
      heading: "Откуда взялся Opten",
      body: (
        <>
          <p className="mb-[14px]">
            Я Влад Воронежцев — снимаю YouTube Shorts...
            (канал <a href="https://youtube.com/@v.voronezhtsev" target="_blank" rel="noopener noreferrer" className="text-[#9cfb51] underline hover:text-white transition-colors">@v.voronezhtsev</a>...
          </p>
          <p className="mb-[14px]">...</p>
          <p className="mb-[14px]">...</p>
        </>
      ),
    },
    { heading: "Почему именно про промпты", body: (<>...</>) },
    { heading: "Что вы получаете", body: (<>...</>) },
```
