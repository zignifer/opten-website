## Outcome

## What changed in `scripts/prerender.mjs`

Diff vs pre-Wave-1 baseline (33 insertions, 9 deletions across 4 surgical edits):

1. **Extended import (line 23)** — added `DEFAULT_OG_IMAGE_EN` to the destructured import from the manifest bundle. Defensive: the helper itself uses the per-route `meta.ogImage` field, but the import list now mirrors the manifest's public surface.

2. **Added `escapeRegex(s)` helper (lines 33-36)** — module-scope helper paired with the existing `escapeAttr` / `escapeHtml` family. Used by the refactored font-preload guard to interpolate literal paths into `new RegExp(...)` safely.

3. **Added `applyOgImage(html, meta)` helper (lines 90-107)** — placed between `applyOgLocale` and `applyMeta`. Resolves `meta.ogImage ?? DEFAULT_OG_IMAGE`, then performs two regex replacements (`og:image` + `twitter:image`) with fail-fast `throw` if either anchor is missing. Mirrors the established `applyOgLocale` / `applyHtmlLang` fail-fast pattern.

4. **Wired into the route loop (line 266)** — `html = applyOgImage(html, meta);` inserted between `applyOgLocale` and `applyJsonLd`, matching the placement spec in the plan's `<interfaces>` block.

5. **Deleted stale comment + dead code (route loop)** — the obsolete `// (ogImage is read but not currently re-injected — index.html already has og-card-ru.png hardcoded; Phase 4+ may swap this per-route)` comment and the now-dead `const ogImage = meta.ogImage ?? DEFAULT_OG_IMAGE;` resolve at the loop top both removed. `applyOgImage` owns the fallback now.

6. **Refactored `FONT_PRELOAD_REGEXES` → `REQUIRED_FONT_PRELOADS` (lines 240-259)** — replaced the two-element regex literal array with a two-element string array of paths. `applyHeroPreloadGuard` now builds each regex per iteration via `new RegExp(...)` using the new `escapeRegex` helper. Error message updated from `missing font preload ${re}` (printed `/regex/` literal) to `missing font preload ${fontPath}` (prints `/fonts/PT-Root-UI_VF.woff2` — clearer diagnostics).

The route-loop `apply*` ordering for every other helper is unchanged. The `/pay` and `/en/pay` Paddle script gate (INTEGRATION-CONTRACT §6) is untouched.
