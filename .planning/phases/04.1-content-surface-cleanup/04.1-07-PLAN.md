<objective>
Close IN-06 (inline `<style>` block at AboutPage.tsx:156-192 pollutes global CSS scope) with one atomic commit on a single file. Pure refactor; no functional change; no copy edit.

Purpose: AboutPage currently uses an inline `<style>{\`...\`}</style>` JSX block to style `.about-content h2/p/ul/li/a/strong` selectors. Because React renders this as a real `<style>` tag in the document head, those selectors leak globally — any future page that uses an `about-content` className anywhere would inherit them. Per CLAUDE.md "Don't add features beyond what the task requires" + CONTEXT D-Wave-7, the fix is to lift the rules into Tailwind utility classes applied directly to the rendered JSX nodes. NO new Tailwind component / @apply / custom utility class — just longer `className` strings where rules repeat.

This plan is INDEPENDENT of every other wave EXCEPT that it makes Wave 12a easier (the data-extract refactor will pick up cleaner JSX). Per CONTEXT ordering preference: W7 → W12a → W10/W11 (about edits) — but only because consecutive AboutPage edits in one direction reduce merge friction. No code coupling.

Output: One commit modifying `src/app/pages/AboutPage.tsx` only.
</objective>

Tailwind equivalents (validate against `src/styles/theme.css` if any custom scale tokens are needed — none of these sizes should require custom tokens since they map to standard Tailwind utilities or arbitrary-value brackets):
