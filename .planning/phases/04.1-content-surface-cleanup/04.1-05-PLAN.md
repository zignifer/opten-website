<objective>
Close WR-09 (V-10 visible Q/A == JSON-LD mainEntity is convention, not gate) by adding a new build-time verifier that enforces FAQ parity as a hard fail. One new script + one chain step in package.json.

Purpose: D-08 / V-10 declared "visible FAQ Q/A == FAQPage mainEntity" as a phase invariant, but it's currently enforced only by the shared source file (`src/content/landingFaq.ts` feeds both `<FaqBlock items={landingFaq[lang]} />` and `faqPageBlock(landingFaq[lang], pageId)`). A future contributor could accidentally pass a different source to one side (or a future plan could fork the source) and the visible/schema drift would go undetected until Google's Rich Results Test catches it weeks later. A 60-line postbuild verifier that parses both sets out of the emitted HTML closes this gap. Failure mode: build fails with a clear `Route X: visible Q/A {…} differs from JSON-LD mainEntity {…}` diff.

Depends on Plan 04.1-04 — sequential because both modify `scripts/seo-routes.ts` and `src/app/components/FaqBlock.tsx`'s readonly types must be in before the verifier ships (no actual code coupling, but ordering keeps git history clean).

Output: New file `scripts/verify-faq-mainentity.mjs` + extended `package.json` build script.
</objective>

Routes carrying FAQPage blocks (from `scripts/seo-routes.ts` post-Phase 4):
- `/` → `faqPageBlock(landingFaq.ru, ...)` → 5 questions
- `/en/` → `faqPageBlock(landingFaq.en, ...)` → 5 questions
- `/guides/gpt-image-2` → `faqPageBlock(gptImage2Guide.ru.faq, ...)` → 5 questions
- `/en/guides/gpt-image-2` → `faqPageBlock(gptImage2Guide.en.faq, ...)` → 5 questions

How to identify routes with FAQPage from the manifest (preferred — avoids re-discovering by file scanning): import the routes manifest via `await import(pathToFileURL(resolve(ROOT, ".ssr-meta", "seo-routes.js")).href)` and filter `routes.filter(r => r.prerender !== "none" && r.schema?.some(b => b["@type"] === "FAQPage"))`.

How to extract JSON-LD mainEntity questions from an emitted HTML file:
- Read the file
- Regex-extract all `<script type="application/ld+json">([\s\S]*?)</script>` blocks
- For each block: `JSON.parse` it, check `@type === "FAQPage"`; if so, collect `mainEntity[*].name` strings
- IMPORTANT: prerender.mjs HTML-escapes JSON-LD payload via `escapeJsonLd` (lines 191-196), replacing `<`, `>`, `&` with `<`, `>`, `&`. `JSON.parse` accepts those unicode escapes natively (they decode to the original characters), so no unescape step is needed before parsing.

How to extract visible FAQ `<dt>` content from the same HTML:
- The `FaqBlock` markup is `<section id="faq" aria-labelledby="faq-heading">...<dl><div><dt>QUESTION</dt><dd>ANSWER</dd></div>...</dl></section>` (per `src/app/components/FaqBlock.tsx`).
- Regex: `/<section[^>]*id="faq"[\s\S]*?<\/section>/` to scope, then `/<dt[^>]*>([\s\S]*?)<\/dt>/g` within the scoped block. Trim + collapse whitespace on each match. For SSR'd React markup, the `<dt>` content may include HTML-escaped chars (`&amp;`, `&quot;`) — decode the minimal set (`&amp;` → `&`, `&quot;` → `"`, `&#39;` → `'`, `&lt;` → `<`, `&gt;` → `>`) before comparing.
- Edge case: guide pages have `id="faq"` for the FaqBlock section per `src/app/pages/GuidePage.tsx` convention (consistent with landing).

Comparison logic:
- Build two sets: `visibleSet = new Set(visible_dt_texts)` and `schemaSet = new Set(mainEntity_names)`.
- If `visibleSet.size !== schemaSet.size` OR any element differs: throw with a clear diff: list questions in visibleSet but not in schemaSet, list questions in schemaSet but not in visibleSet, include the route path.
- For symmetry, also assert `visibleSet.size === landingFaq[lang].length` (or `gptImage2Guide[lang].faq.length`) — though this is implied by the equality check if neither side dedupes.

Error message format:
`verify-faq-mainentity.mjs: route ${meta.path} — FAQ drift detected. visible ↔ schema parity broken.\\n  Visible-only questions:\\n${diffA.map(q => "    - " + q).join("\\n")}\\n  Schema-only questions:\\n${diffB.map(q => "    - " + q).join("\\n")}`

Success log format (mirror llms.mjs):
`✓ verify-faq-mainentity: N routes checked, all visible↔schema parity holds`
