# Phase 4.1: Content Surface Cleanup (captured decisions seed)

## Goal

Close all 15 deferred findings from `04-REVIEW.md` (9 warnings + 6 info items)
in surgical, file-scoped atomic commits. No new product surface, no schema
additions, no route changes — pure remediation of issues surfaced by code
review of Phase 4.

The 2 critical findings (CR-01 JSON-LD `<script>` escape, CR-02 RU /refund
description drift) were already resolved in commit `eee40f6` during Phase 4
and are NOT part of this phase.

## Why now

The site is on production. Each finding is either customer-facing (WR-01 EN
OG cards), reputation-affecting (WR-02 $0 Free tier in Rich Results), SEO-
damaging (WR-03 breadcrumb soft-404), a real bug (WR-04 detectExtension
race, IN-02 lang-boundary anchor), or a defensive guard whose cost to add
now is much lower than the cost to add after it has masked a regression.

## Hard constraints

- **No changes to `EXTENSION_IDS`, `SUPABASE_URL`, `SUPABASE_ANON_KEY`** —
  these are mirrored in the extension and are locked by INTEGRATION-CONTRACT.
- **No changes to locked routes** (`/welcome`, `/pay`, `/success`, `/account`,
  `/dashboard/download-skill`). `/pay` keeps its synchronous Paddle CDN
  `<script>` injection and the full prerender promotion from Phase 4 D-12.
- **No regressions to Phase 4 build-time gates** — `npm run build` must keep
  producing all 14 prerendered routes with the same titles/meta/canonical/
  hreflang/JSON-LD as today; sitemap and llms.txt must keep their current
  shape (except for additive fixes called out per wave).
- **Wave 9 (PayPage.tsx) requires explicit approval** before commit — touches
  the payment surface; falls under the "extra caution around extension-coupled
  surfaces" memory rule.

## Waves

Each wave is one file scope, one atomic commit, one verification step.
Waves are ordered low-risk-first so any rollback in late waves doesn't
require unwinding earlier ones.

### Wave 3: `scripts/llms.mjs` — traversal guard + HTML entity decoder

**Addresses:** WR-06 (path traversal), WR-07 (HTML entity decoder covers only 5 entities)

**Changes:**
1. After computing `outPath` from `meta.path`, assert
   `outPath.startsWith(DIST + sep) || outPath === resolve(DIST, "index.html")`
   and throw if it escapes. Plus reject any `meta.path` containing `..`,
   backslash, or absolute prefix at the input edge for defense in depth.
2. Extend `extractText` to decode `&#nnn;`, `&#xHHHH;`, and strip any
   remaining `&name;` named entity rather than leaking it.

**Verify:**
- `npm run build` succeeds; `dist/llms.txt` and `dist/llms-full.txt` still
  generate without errors.
- Spot-check `dist/llms-full.txt` for `&` followed by alphanumerics — should
  not appear (any future entity would be stripped, not leaked).

**Commit:** `fix(04.1): harden llms.mjs — path-traversal guard + broader HTML entity decoder (WR-06, WR-07)`

### Wave 8: `scripts/entry-server.tsx` — defense-in-depth Route assert

**Addresses:** IN-03 (no build-time check that full-tier routes are mounted in entry-server.tsx)

**Changes:**
1. In `scripts/prerender.mjs` (not entry-server.tsx itself — the assert
   belongs at the consumer, not the producer), after the SSR import is
   resolved, fail fast if any `routes.filter(r => r.prerender === "full").map(r => r.path)`
