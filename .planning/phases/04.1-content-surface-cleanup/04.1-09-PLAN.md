**Wave 9 may ONLY change two things:**

1. **`detectExtension` race fix.** Current (lines 193-223):
   ```js
   const detectExtension = () => {
     const chrome = (window as any).chrome;
     if (!chrome?.runtime?.sendMessage) {
       setExtStatus("not_installed");
       return;
     }
     let tried = 0;
     for (const id of EXTENSION_IDS) {
       try {
         chrome.runtime.sendMessage(id, { type: "GET_AUTH_TOKEN" }, (response: any) => {
           tried++;
           if (chrome.runtime.lastError || !response) {
             if (tried >= EXTENSION_IDS.length) setExtStatus("not_installed");
             return;
           }
           if (response.token) {
             setToken(response.token);
             if (response.email) setEmail(response.email);
             setExtStatus("ready");
             fetchSubscription(response.token);
           } else {
             setExtStatus("not_logged_in");
           }
         });
       } catch {
         tried++;
         if (tried >= EXTENSION_IDS.length) setExtStatus("not_installed");
       }
     }
   };
   ```
   Add a `let resolved = false;` in the function-scoped closure (before the `for (const id of EXTENSION_IDS)` loop). Inside the callback:
   - Add `if (resolved) return;` as the FIRST line of the callback (before `tried++`). Idempotency safety net: once any callback has set `resolved = true`, subsequent late-firing callbacks are full no-ops, so they cannot overwrite `extStatus`.
   - In the positive branches — `response.token` (success) and `else` (not_logged_in) — set `resolved = true;` BEFORE calling `setExtStatus(...)`. This locks in the positive outcome.
   - In the negative branch (`chrome.runtime.lastError || !response`), do NOT set `resolved = true` UNTIL `tried >= EXTENSION_IDS.length` (all IDs exhausted, no positive result observed) — that's the "all failed → not_installed" terminal state.
   - Apply the same `resolved` semantic to the `catch` branch's `tried >= EXTENSION_IDS.length` setExtStatus call: gate it on `!resolved` so a synchronous throw on one ID after another ID has already succeeded doesn't flip back to not_installed.

   The race fix should be ~6-8 net lines added; rest of function unchanged.
