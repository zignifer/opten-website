# Wave 7 ‚Äî Replace AboutPage inline `<style>` block with Tailwind utility classes (IN-06)

## Summary

One-liner: AboutPage now declares its body typography via inlined Tailwind utility classes on every JSX node (h2 / p / ul / li / a / strong) instead of an inline `<style>{` ... `}</style>` JSX block, removing global CSS-scope pollution while preserving visual rendering byte-for-byte.

## What changed

Single-file refactor on `src/app/pages/AboutPage.tsx`:

1. Deleted the 37-line inline `<style>{` ... `}</style>` JSX block (lines 156-192 in the pre-refactor file) that scoped 8 CSS rules to `.about-content h2/h2:first-child/p/ul/li/a/a:hover/strong`. Those rules were emitted into the document head as a real `<style>` tag at runtime and would have inherited globally onto any future page using an `about-content` className anywhere.

2. Inlined the equivalent Tailwind utility classes onto every typography-relevant JSX node inside the `<article className="about-content ...">` body:

| Element | className added |
|---|---|
| `<h2>` √ó 4 | `text-white text-[24px] font-medium mt-[48px] mb-[18px] leading-[1.3] tracking-[-0.4px] first:mt-0` |
| `<p>` √ó ~10 | `mb-[14px]` |
| `<ul>` √ó 1 | `mt-[8px] mb-[16px] pl-[24px] list-disc` |
| `<li>` √ó 7 | `mb-[8px]` |
| `<a>` (article body) √ó 3 + `<LocalizedLink>` √ó 2 | `text-[#9cfb51] underline hover:text-white transition-colors` |
| `<strong>` √ó ~11 | `text-white font-semibold` |

   The `first:mt-0` modifier cleanly replaces the `.about-content h2:first-child { margin-top: 0 }` rule ‚Äî Tailwind applies it only when the h2 is the first child of `<article>`, which is the case in the existing markup.

3. Preserved the `className="about-content"` on the `<article>` tag as a semantic hook (no CSS targets it any more, but removing it would be a separate concern per CONTEXT no-new-abstraction rule).

4. Left untouched: header / footer chrome, hero section, the Telegram CTA button outside `<article>`, the footer LocalizedLink rendering, body copy, component imports.
