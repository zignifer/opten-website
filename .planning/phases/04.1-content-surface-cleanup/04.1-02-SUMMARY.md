## WR-05 smoke test — non-finite price triggers build failure

Temporarily replaced `"199"` with `"199,99"` on the RU `/pay` productBlock entry, then ran `npm run build`. Result:

```
Error: productBlock(https://opten.space/pay): non-finite price "199,99" for plan "Pro Monthly (RU)" — expected a number-parseable string (e.g. "199" or "2.99")
    at file:///C:/Projects/opten-website/.ssr-meta/seo-routes.js:245:13
    at Array.map (<anonymous>)
    at productBlock (file:///C:/Projects/opten-website/.ssr-meta/seo-routes.js:242:24)
```

Build exited non-zero with the expected, file/line-anchored error message. Reverted the test edit; subsequent `npm run build` clean. The Number()-based validation does its job — a future translator typo (`"199,99"` European-locale comma, `"199 руб"` unit suffix, or any other non-finite input) will fail the build at prerender time with a clear, actionable message rather than silently propagating NaN into AggregateOffer math or shipping a broken Product schema to production.
