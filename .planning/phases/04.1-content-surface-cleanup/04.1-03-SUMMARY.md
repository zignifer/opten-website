# Phase 04.1 Plan 03: Harden scripts/llms.mjs Summary

One atomic commit hardening the postbuild llms.txt/llms-full.txt emitter
(`scripts/llms.mjs`) against two defensive concerns: path traversal at the
file-read boundary (WR-06) and silent HTML-entity leakage in the plaintext
output (WR-07). Pure defensive work — zero behavior change on the happy
path.

## What changed

`scripts/llms.mjs` only (39 insertions, 1 deletion). Three surgical edits
landed in the single atomic commit `b20af89`:

1. **Import** — added `sep` to the `node:path` import (line 8) for the
   cross-platform `startsWith(DIST + sep)` boundary check.

2. **WR-06 — path-traversal guards in `bodyForRoute`** (lines 107–137).
   - Input-edge rejection at the top of the function: throws if
     `meta.path` contains `..` or backslash (`\`). Error message:
     `llms.mjs: refusing suspicious meta.path "<value>" (contains '..' or backslash — possible path traversal)`.
   - Escape-boundary assertion after `outPath` computation, before the
     `readFile`: throws if `outPath` is neither `resolve(DIST, "index.html")`
     (the `/` special case) nor begins with `DIST + sep`. Error message:
     `llms.mjs: outPath "<path>" escapes DIST "<DIST>" for meta.path "<value>"`.
   - Both guards throw rather than warn — a malformed manifest entry
     should fail the build, not silently write outside `dist/`.

3. **WR-07 — extended entity decoder in `extractText`** (lines 80–102).
   Three new `.replace()` calls inserted between the existing `&#39;` →
   `'` line and the `\s+` whitespace collapse:
   - Decimal numeric entity decoder (`&#nnn;`) — decodes via
     `String.fromCodePoint`, drops invalid code points (out of range or
     non-finite) rather than leaking them.
   - Hex numeric entity decoder (`&#xHHHH;` / `&#XHHHH;`) — same shape,
     same defensive validation, case-insensitive.
   - Catch-all named-entity stripper (`&[a-zA-Z][a-zA-Z0-9]*;`) — runs
     LAST in the entity chain so the six explicit named-entity decoders
     (`&nbsp;`, `&amp;`, `&lt;`, `&gt;`, `&quot;`, `&#39;`) still win.
     Any remaining unknown named entity (e.g. `&hellip;`, `&mdash;`,
     `&ldquo;`) is stripped silently — preferable to leaking it as
     literal text in `dist/llms-full.txt`.

The existing `&#39;` → `'` line is intentionally kept (back-compat with
the historical chain); the new decimal decoder would also match it but
the explicit replace runs first. Harmless overlap.

Sections NOT touched: `SECTIONS` array, `SIZE_CAP_BYTES` legal-truncation
logic, prerendered-routes floor check (line 21), file-write/console
output. The runtime shape of `llms.txt` and `llms-full.txt` is identical
to the pre-04.1-03 baseline for valid inputs.
