## Milestone Summary

**Key Decisions:**
- **D-10..D-13** (Phase 2): no external API (Claude Code Agent tool only); meta locked in `_registry.ts`; atomic batch commits not monolith; regenerate only on ≥2 systematic errors.
- **EN meta via hand-maintained `metaEn.ts` overrides** — registry is auto-generated from RU-only skill MDs, so EN prose fields (name/platform/duration/resolution) get a regen-safe override layer rather than regenerating the registry.
- **Umbrella models kept-but-hidden** — `HUB_HIDDEN_SLUGS` hides 11 bare-brand pages from the hub grid/ItemList while keeping them live + sitemap'd for generic-query SEO (no 404, not orphaned).
- **`hydrated` gate for URL-derived filters** — prerendered hub/blog must render the full list on first client paint to match SSR; filters apply post-mount.

**Issues Resolved:**
- Russian leaking into EN pages (visible meta + head meta).
- React #418/#425 hydration mismatch on direct-load of filtered hub/blog URLs (was latent in both pages).
- Duplicate "Seedance 2.0" card title.

**Technical Debt / Deferred:**
- **MODELS-B-6** GSC indexing verification (monitoring window).
- **Codex MEDIUM** — wire `verify-models-content.mjs` into the build chain (needs Node ≥22.18 confirmed on Vercel).
- **Codex LOW** — verify script doesn't check all nested required fields (partly covered by `vite build` types).
- `/models/seedance` content `<title>`/`<h1>` still "Seedance 2.0" (hidden page, low priority).
