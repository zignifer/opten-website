# GEO / LLM Visibility Audit

Date: 2026-06-12
Site: https://opten.space
Scope: AI crawler access, llms files, schema/entity graph, public citations, Google AI feature eligibility, LLM citability, Learn/Blog/Models content surfaces.

## Executive summary

Opten has a solid technical GEO foundation, but weak external entity authority.

The site is doing many technical things right:

- `robots.txt` explicitly allows major search and AI crawlers.
- `Content-Signal: search=yes, ai-train=yes, ai-input=yes` is present.
- `llms.txt` and `llms-full.txt` are live.
- Pages are prerendered and textual.
- Rich JSON-LD exists: `Organization`, `Person`, `SoftwareApplication`, `Article`, `TechArticle`, `LearningResource`, `VideoObject`, `FAQPage`, `BreadcrumbList`, `ItemList`.
- This audit fixes the Google video warning source: `VideoObject.uploadDate` now emits ISO DateTime with `+05:00`.

The missing part is not another AI-specific file. The missing part is that LLMs do not yet have enough trusted, repeated, third-party evidence that Opten is an entity worth mentioning.

Google's own AI features guidance says there are no special extra technical requirements for AI Overviews/AI Mode beyond normal Search eligibility, helpful content, crawlability, internal links, textual content, page experience, and matching structured data. It also says pages need to be indexed to appear as supporting links:
https://developers.google.com/search/docs/appearance/ai-features

Since most new Opten URLs are not indexed in Google yet, they cannot feed Google AI surfaces. Since independent citations are sparse, ChatGPT/Perplexity/Copilot/Gemini have little reason to mention Opten unless the user asks about Opten directly.

## Why LLMs do not mention Opten yet

LLM visibility generally needs three things:

1. Crawl/index eligibility.
2. Clear entity understanding.
3. Repeated trusted citations from independent sources.

Opten has the first partly solved, but Google indexing is incomplete. It has an internal entity graph, but external validation is thin. It does not yet have enough independent citations on high-trust surfaces.

Current LLM likely behavior:

- Direct query `what is Opten` can work if the model browses or has recent access.
- Generic query `best AI prompt optimizer for image generation` will likely prefer older/larger tools or pages with more backlinks and mentions.
- Query about `GPT Image 2 prompts` or `Kling 3 prompts` will likely cite YouTube, Habr, official docs, Reddit, or high-authority blogs before Opten.
- Query about a third-party Learn Find video will likely cite the original YouTube video unless Opten's page adds unique, text-first analysis.

## Content citability audit

### Models

Status: good foundation.

Model pages are 1450-2500 words locally and have structured data. They are the strongest SEO/GEO assets.

Weakness:

- Many pages share a similar programmatic pattern.
- They need more first-hand proof:
  - before/after prompt examples
  - generated output notes
  - model-specific constraints
  - "tested in Opten" snippets
  - links from related blog/Learn pages

### Blog

Status: decent.

Blog posts are 937-1806 words, which is much stronger than Learn.

Weakness:

- Need more visible author expertise and original tests.
- Need external references/citations when discussing tools/models.
- Need internal links back to model pages and Opten product workflows.

### Learn and Learn Finds

Status: main GEO risk.

Learn lessons and Finds are too thin:

- Learn lessons: median about 333 words.
- Learn Finds: median about 462 words.

For LLM citability, a page should be extractable as an answer without watching the video. Current pages often require the user/model to treat the embedded video as the primary object.

Recommended structure for each Learn Find:

- H1 with exact task/query.
- 50-word answer block.
- "What this video teaches" with concrete claims.
- "Opten editorial notes" with original interpretation.
- Step-by-step workflow.
- Prompt/command blocks.
- Common mistakes.
- When not to use this workflow.
- Resources and source links.
- FAQ.
- Clear attribution to the third-party video creator.

## Platform-specific recommendations

### Google AI Overviews / AI Mode

Priority: indexability and helpful content.

Actions:

- Fix normal Google indexing first.
- Expand Learn text.
- Make important pages findable through internal links.
- Keep structured data aligned with visible text.
- Use GSC to monitor Web search traffic; Google counts AI feature traffic in Search Console Web data.

Reference: https://developers.google.com/search/docs/appearance/ai-features

### ChatGPT Search / OpenAI

Priority: crawl access plus external citations.

Current:

- `GPTBot`, `ChatGPT-User`, and `OAI-SearchBot` are explicitly allowed.

Actions:

- Add official social/profile URLs to schema once they are stable.
- Create a press/about page with clear product facts.
- Get mentions on sites likely to be indexed and cited.
- Ensure the strongest pages are indexed in Bing/Google, because browsing systems often rely on search indexes.

### Perplexity

Priority: citation-ready pages.

Actions:

- Add source-backed article sections.
- Include tables, definitions, and concise answer blocks.
- Add outbound citations to official model docs and original videos.
- Build comparison pages where Opten has a legitimate reason to be cited.

### Gemini

Priority: Google index.

Actions:

- Same as Google Search.
- Do not expect `llms.txt` to compensate for non-indexed pages.

### Microsoft Copilot

Priority: Bing index and external mentions.

Actions:

- Check Bing Webmaster separately.
- Make sure `Bingbot` stays allowed.
- Submit sitemap to Bing if not already done.
- Get citations from Microsoft/Bing-visible public sources.

## Entity and authority action plan

### P0 - Build a stronger Opten entity profile

Create or update a public entity/press surface:

- `/about` should include:
  - product one-liner
  - launch date
  - extension version
  - supported platforms
  - supported model count
  - Chrome Web Store link
  - founder profile
  - official social links
  - screenshots/logos
- Add `/press` or `/brand` if media/partner links are expected.

Schema changes:

- Add official Instagram to `sameAs` if it is a stable official identity.
- Add official YouTube channel if product videos are published there.
- Add LinkedIn Company/Product page when created.
- Do not add random directories to `sameAs`; use them as citations/backlinks, not identity mirrors.

### P0 - Grow Chrome Web Store trust

Current store signal is real but small:

- 298 users
- 2 ratings

Target:

- 10+ honest reviews.
- 500-1000 users.
- Product screenshots and copy synchronized with current positioning.

Only after enough honest reviews, consider restoring aggregate rating schema if it exactly matches the live store.

### P1 - Create independent citations

Target 10-20 external pages over 60-90 days:

- Product Hunt launch.
- AlternativeTo / SaaS / AI tool directories where relevant.
- Habr/VC-style founder post about building Opten.
- YouTube descriptions linking to exact site pages.
- Instagram/Telegram posts linking to exact canonical pages.
- Guest posts or interviews about AI prompt workflows.
- Partner pages from supported tool ecosystems if possible.
- Authentic Reddit participation where allowed, not spam links.

The goal is not raw backlink count. The goal is consistent entity triples:

- Opten
- AI prompt scorer/optimizer Chrome extension
- opten.space
- founder Vlad Voronezhtsev
- works with image/video generation prompts

### P1 - Turn Learn into original assets

Current Learn direction can work, but only if it stops being "short recap around someone else's video" and becomes "Opten's annotated, practical guide based on the video."

For every third-party video page, add:

- what Opten tested
- where the original author is right/wrong/incomplete
- exact prompt examples
- practical checklist
- risks/limitations
- related Opten model/blog links

This creates a reason for Google/LLMs to cite Opten instead of the source video alone.

### P1 - Add answer assets

Create pages/sections designed for extraction:

- "What is Opten?"
- "How Opten scores prompts"
- "Supported AI models"
- "Best GPT Image 2 prompt structure"
- "Best Kling 3 prompt structure"
- "Prompt scoring checklist"
- "AI video prompt checklist"

Each should have:

- one concise definition block
- a table or checklist
- author/update date
- internal links to product and deeper articles
- external citations when making factual claims

## Bottom line

The technical GEO layer is already better than most small product sites. The missing growth lever is not another schema trick. It is a combination of:

1. Google indexing more of the current pages.
2. Fewer thin pages and more original Learn/editorial depth.
3. More third-party mentions that establish Opten as a real product/entity.
4. More public product trust signals: users, reviews, launch pages, directories, interviews, and founder/product profiles.
