## Executive Summary

**Overall GEO Score: 63 / 100 — Fair**

Сайт находится в редкой для русскоязычных AI-стартапов позиции: **техническая часть почти идеальна** (prerender, hreflang reciprocity, robots.txt с AI-ботами, llms.txt, schema graph через @id) — это топ-10% индустрии. Но **AI-системы практически не цитируют Opten**, потому что:

1. В `Person` schema founder назван **«Виктор Воронежцев»**, тогда как visible text везде — «Влад Воронежцев» (это **machine-readable bug**, который рвёт entity resolution для всех 5 платформ).
2. **Brand Authority = 18/100** — ноль упоминаний на Wikipedia, Reddit, Habr, vc.ru, LinkedIn, Product Hunt, AI-каталогах. AI-моделям не на что опереться при entity recognition.
3. **Mobile LCP 3.5s + CLS 0.129 на RU** — два Core Web Vitals в красной зоне.

Если исправить только 5 пунктов из P0-блока (≈3-4 часа работы), композитный score реалистично поднимется до **75-78**, потому что технический фундамент уже готов.

## Critical Issues (исправить на этой неделе)

### CR-1. Person schema: имя founder противоречит visible content
**Файл:** [scripts/seo-routes.ts:110](../../scripts/seo-routes.ts)
**Проблема:** `name: "Виктор Воронежцев"` в `PERSON_FOUNDER_BLOCK`, но в description на той же странице (строка 411) и в `src/content/about.tsx` founder назван **«Влад Воронежцев»**. Legal-документы (`/privacy`, `/terms`) указывают «Воронежцев Владислав Павлович».
**Эффект:** AI-системы при entity resolution получают противоречие между schema (доверенный источник) и body text → понижают confidence по сущности «Opten founder». ChatGPT/Perplexity на запрос «who founded Opten» с высокой вероятностью назовут «Виктор Воронежцев» — некорректное имя.
**Подтверждено независимо:** geo-ai-visibility, geo-content, geo-schema, Codex CLI.
**Fix:**
```typescript
// scripts/seo-routes.ts строки 106-116
export const PERSON_FOUNDER_BLOCK: SchemaBlock = {
  // ...
  name: "Влад Воронежцев",
  alternateName: "Воронежцев Владислав Павлович",
  givenName: "Владислав",
  familyName: "Воронежцев",
  // ...
};
```
**Effort:** 5 минут + redeploy. **ROI:** очень высокий — фундаментальный entity fix.

### CR-2. Person.sameAs содержит только Telegram — теряется 80% authority signal
**Файл:** [scripts/seo-routes.ts:106-116](../../scripts/seo-routes.ts)
**Проблема:** founder ведёт YouTube канал `@v.voronezhtsev` (4500+ subs, 54 ролика, 7 млн просмотров) — это **самый сильный verifiable authority signal**, и он **не связан** с Person через `sameAs`. AI-системы (особенно Gemini, Perplexity) специально ищут `sameAs` для linking entity к проверяемой digital footprint.
**Fix:**
```typescript
sameAs: [
  FOUNDER_TELEGRAM_URL,
  "https://www.youtube.com/@v.voronezhtsev",
],
```
**Effort:** 2 минуты. **ROI:** очень высокий — +5-8 баллов по Gemini/ChatGPT/Perplexity сразу.

### CR-3. Organization.sameAs всего 2 ссылки
**Файл:** [scripts/seo-routes.ts:73-76](../../scripts/seo-routes.ts)
**Проблема:** только Chrome Web Store + Telegram. Для AI entity graph рекомендуется ≥4 платформ. Бренд «Opten» совпадает с «Opten GmbH» (немецкая legal-info компания) — **collision risk** для ChatGPT.
**Fix:** добавить YouTube канал founder в Organization.sameAs (как организационный авторитет), LinkedIn (когда заведётся), Wikidata Q-ID (после создания).

### CR-4. Mobile LCP 3.5s + CLS 0.129 на RU — failing Core Web Vitals
**Замер:** PageSpeed Insights mobile, оба landing-маршрута.
**Причина:** два variable шрифта (`Unbounded-VF.woff2`, `PT-Root-UI_VF.woff2`) preload-ятся одновременно, RU-текст с длинными словами вызывает font-swap CLS.
**Fix:**
- `src/styles/fonts.css` — `font-display: optional` для одного из шрифтов или sub-set неиспользуемых глифов.
- `index.html` — preload только критичный hero-шрифт (Unbounded для H1), второй lazy.
- Проверить, что hero image и SVG в `src/imports/` имеют explicit `width`/`height`.

**Effort:** 1-2 часа. **ROI:** Google AIO + Perplexity явно весят CWV.

### CR-5. Logo bar лендинга: бренды-партнёры без disclaimer
**Файл:** [src/app/App.tsx:215-228](../../src/app/App.tsx)
**Проблема:** Syntx, Freepik, Higgsfield, Canva отображаются как logo bar без явного разделения «это сайты, где работает Opten» vs «партнёры Opten». Юридический риск + AI могут интерпретировать как official partnership.
**Fix:** добавить заголовок «Opten работает на этих сайтах» (вместо «After install, use Opten on:») + disclaimer-строчку: «Opten — независимый продукт. Указанные бренды не аффилированы с Opten».

### HI-4. Article / TechArticle schema отсутствует на /guides/* и /about

### HI-5. Дата гайда `publishedAt: "2026-05-17"` не попадает ни в HTML, ни в JSON-LD
**Файлы:** `src/content/guides/gpt-image-2.ts` (date в TS), `src/app/pages/GuidePage.tsx` (render), `scripts/seo-routes.ts` (schema).
**Fix:** visible `<time datetime="2026-05-17">17 мая 2026</time>` после H1 + `datePublished`/`dateModified` в HowTo + Article schema (см. HI-4).

### HI-6. CSP отсутствует
**Файл:** [vercel.json](../../vercel.json)
**Fix:**
```
Content-Security-Policy: default-src 'self';
  script-src 'self' 'unsafe-inline' https://cdn.paddle.com https://*.paddle.com;
  connect-src 'self' https://*.supabase.co https://*.paddle.com;
  img-src 'self' data: https:;
  font-src 'self' data:;
  frame-src https://*.paddle.com;
```
`unsafe-inline` нужен для inline JSON-LD скриптов.

### HI-7. Bing Webmaster Tools + IndexNow не настроены
**Эффект:** Bing Copilot узнаёт о новых страницах с задержкой в дни/недели вместо часов. Это бесплатная конфигурация на 1 час работы.
**Fix:**
1. Зарегистрировать https://www.bing.com/webmasters → добавить `<meta name="msvalidate.01" content="...">` в `index.html`.
2. Сгенерировать IndexNow API key → положить `{key}.txt` в `public/` → добавить POST на `api.indexnow.org` в `scripts/sitemap.mjs` при build.

### HI-8. Robots.txt: 8 AI-ботов работают только под wildcard
**Файл:** [public/robots.txt](../../public/robots.txt)
**Не указаны явно:** `OAI-SearchBot`, `ChatGPT-User`, `CCBot`, `Bytespider`, `Meta-ExternalAgent`, `Amazonbot`, `YandexBot` (критично для RU!), `Bingbot`.
Под `User-agent: *` они работают, но эксплицитная запись страхует от регрессий и помогает crawler-логике некоторых ботов.

### HI-9. Brand Authority: внешние упоминания = 0
**Эффект:** даже после фиксов CR-1..3 AI-системы будут плохо знать Opten, пока нет external corroboration.
