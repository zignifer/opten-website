# Blog post SEO/GEO rules

Эти правила статичны и применяются к каждой статье, которую агент сайта пишет по
недельному брифу.

## Ключи

- Одна основная фраза на статью + 4-8 смежных фраз.
- RU и EN прорабатываются отдельно: не переводить ключи автоматически.
- Ключи должны звучать естественно. Бриф дает рекомендуемые места и варианты, а
  не финальные фразы, которые нужно вставлять дословно.

## Размещение

| Элемент | Что использовать |
|---------|------------------|
| `title` / H1 | primary keyword ближе к началу, естественно. |
| `excerpt` / description | primary + 1 secondary в формулировке пользы. |
| intro | короткий answer-block в первые 40-60 слов. |
| H2/H3 | secondary keywords, вопросы и how-to формулировки. |
| body | long-tail и синонимы без повторов в точную форму. |
| FAQ | реальные query-like вопросы из брифа. |
| image alt | описательный alt + 1 релевантная фраза, без keyword stuffing. |

## Рубрика блога

- Видимая рубрика статьи задается только полем `category`.
- Разрешены ровно два значения: `guide` и `news`.
- По умолчанию ставь `guide`. `news` используй только для настоящих новостей:
  запуск продукта, изменение тарифа, релиз Opten, важное объявление. Обзоры
  моделей, сравнения, how-to и release-note-похожие SEO-статьи остаются `guide`.
- `tags` — технические SEO-keywords для schema keywords, не публичные рубрики.

## GEO

- В начале статьи дать прямой ответ, который можно процитировать.
- Использовать конкретные модели, версии, сценарии и ошибки.
- Добавить 3-5 FAQ с прямыми ответами.
- Добавить named practical case: модель, проблема, исправление промпта, результат.

## Opten CTA

Opten упоминается 1-2 раза как ассистент для генерации, оценки и улучшения
промпта под конкретную модель. CTA мягкий: статья должна сначала решать задачу
читателя, а не продавать расширение в каждом блоке.

## Humanizer pass

После генерации статьи сделать отдельный редакторский проход:

- RU: `seo2/rules/humanizer-ru.md`
- EN: `seo2/rules/humanizer-en.md`

Humanizer не должен вырезать SEO-ключи. Он смягчает машинный ритм, убирает
штампы и делает текст живее, но сохраняет keyword lock: primary в title/H1-zone
и intro, primary + secondary в meta/excerpt, secondary в H2/body/FAQ/image alt.

## Чего избегать

- Free-seeker хвост как основная тема (`free`, `без регистрации`, `unlimited`),
  если статья не умеет честно конвертировать такой спрос.
- Карьера/курсы/prompt engineering jobs.
- Coding-сравнения нейросетей.
- Слишком строгий машинный текст: после генерации статью можно и нужно
  humanize, сохраняя semantic coverage из брифа.
