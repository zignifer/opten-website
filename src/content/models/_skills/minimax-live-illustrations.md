## Структура хорошего промпта

I2V-01-Live принимает 2D-иллюстрацию + промпт описывающий ДВИЖЕНИЕ. Промпт не должен описывать изображение — только анимацию.

```
[Subtle motion description] + [Direction/type of movement] + [Atmospheric changes]
```

### Принципы

- Описывать ТОЛЬКО движение и анимацию, не содержимое иллюстрации
- Субтильные движения: дыхание, моргание, развевающиеся волосы, колыхание ткани
- Модель СОХРАНЯЕТ стиль линий и арт-стиль оригинальной иллюстрации
- Короткие и конкретные промпты — 10–30 слов

### Типы входных изображений

- Аниме-иллюстрации
- Манга-панели
- Цифровые портреты
- Digital art / concept art
- Иллюстрации книг
- Рисованные персонажи

## Ключевые возможности

1. **Сохранение арт-стиля** — линии, цвета и стиль оригинальной иллюстрации остаются нетронутыми
2. **Live2D-стиль анимации** — субтильные, плавные движения характерные для Live2D
3. **Аниме/манга специализация** — лучшая модель для анимации рисованных персонажей
4. **Целостность линейного рисунка** — не ломает линии и контуры

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Описание движения** — конкретное описание анимации. «Hair gently flowing», «eyes blinking slowly», «fabric rippling in wind».
2. **Входное изображение** — I2V-01-Live ТРЕБУЕТ 2D-иллюстрацию. Если нет входного изображения — fail, score не выше 20.

### Бонусные элементы (улучшают результат)
3. **Тип движения** — субтильное vs динамичное. Для 2D-иллюстраций субтильные движения обычно лучше.
4. **Атмосферные изменения** — «light shifting», «leaves drifting past», «sparkles appearing».
5. **Направление движения** — «from left to right», «upward», «toward camera».

### Антипаттерны (ухудшают результат)
- **Описание содержимого иллюстрации** — модель видит изображение, описывать только ДВИЖЕНИЕ.
- **Слишком динамичные действия** — сложные экшн-сцены не подходят для 2D-анимации. Модель оптимизирована для субтильных движений.
- **Промпт для фотореалистичного контента** — модель специализирована на 2D-иллюстрациях, не фотореализме.
- **Описание стиля рисунка** — стиль определяется входным изображением, не промптом.
- **Слишком длинный промпт** — для I2V 2D-анимации 10–30 слов достаточно.
- **Попытка генерации без изображения** — T2V НЕ поддерживается.
- **Использование для фото/реалистичных изображений** — для этого лучше Hailuo 2.3 или 02.

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- I2V-01-Live = специализированная модель для 2D-анимации
- Короткий промпт (10–30 слов) — НОРМАЛЬНО и часто оптимально
- Субтильные движения — это фича, не ограничение
- Отсутствие описания сцены — правильно, модель видит иллюстрацию
- Не путать с Hailuo 2.3 или 02 — это ОТДЕЛЬНАЯ модель
- Для аниме/манги — I2V-01-Live лучший выбор, для реалистики — использовать другие модели
