## Структура хорошего промпта

### Для Soul 2.0 / Soul Cinema (Image)

Промпт как **креативный бриф**. Определить настроение, референсы, визуальное намерение.

```
[Mood/Aesthetic] + [Subject with visual cues] + [References/Style] + [Technical hints]
```

Модель нативно понимает:
- Фэшн-терминологию и субкультурный язык
- Онлайн-сленг и нишевые эстетические коды
- Микротренды: «phone flash», «disposable camera», «Y2K street»

Длина: короткие-средние промпты. Выбор пресета снижает нужду в сложном промпте.

### Для DoP (Video, I2V)

Короткие и прямые промпты. **Одно действие + одно движение камеры за раз.**

```
[Active verb action] + [Camera direction]
```

Принципы:
- Активные глаголы: «darts through», «leaps across», «crashes into»
- Разделение задач: освещение и кадрирование — в промпте изображения, движение — в видео-промпте
- Удалять «fillers» — командный стиль
- Камерное оборудование: «Shot on full-frame cinema camera» повышает реализм
- Motion preset выбирается ОТДЕЛЬНО от текстового промпта

## Продвинутые техники

### Soul ID — консистентность персонажей
Soul ID обеспечивает сохранение идентичности персонажа через несколько генераций. Использовать свежие, однотипные референсные фото.

### Soul HEX — извлечение цветовых палитр
Автоматическое извлечение и применение цветовых палитр из референсных изображений.

### Soul Reference — I2I
Использование референсного изображения для стилевого переноса и визуального якорения.

### Soul Cinema — эра-осведомлённость
Отдельная модель от Soul 2.0 с кинематографическим фокусом. Глубокие текстуры, плёночное зерно, стилизация под эпохи (70s, 90s). Результаты служат как ключевые кадры для видео-раскадровки.

### DoP Motion Presets
100+ пресетов: 360 Orbit, Dolly In/Out, Whip Pan, FPV Drone, Bullet Time и т.д.

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что в кадре. Конкретные визуальные детали.
2. **Визуальное намерение** — настроение, эстетика или стиль. Не абстрактное «красивое», а конкретный визуальный код.
3. **Для DoP: действие** — конкретный активный глагол движения. «Leaps across», не «moves».
4. **Для DoP: камера** — движение камеры или motion preset. Одно за раз.

### Бонусные элементы (улучшают результат)
5. **Субкультурные/нишевые отсылки** — модель понимает микротренды нативно.
6. **Техника съёмки** — «phone flash», «disposable camera», «shot on full-frame cinema camera».
7. **Разделение кадрирования и движения** — освещение в изображении, движение в видео-промпте.
8. **Soul Cinema: эпоха** — указание конкретной эры (70s, 90s) для стилизации.
9. **Освещение** — конкретные источники и тип света.

### Антипаттерны (ухудшают результат)
- **Чистый белый фон** на входном изображении — не хватает глубины для Soul
- **Смешивание старых и новых референсных фото** для Soul ID — вызывает drift идентичности
- **Переобъяснение** эстетических терминов, которые модель понимает нативно
- **Длинные сложные промпты для DoP** — короткие и прямые работают лучше
- **Смешивание камеры, персонажа и движения** в одном блоке для DoP — вызывает нестабильность
- **Освещение/объектив в видео-промпте** — вызывает мерцание (описывать в изображении)
- **Несколько лиц в кадре** — DoP приоритизирует ближайшее лицо
- **Промпт для Face Swap / Character Swap** — эти режимы не используют текстовые промпты

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Soul 2.0 промпт = креативный бриф, а не список тегов
- DoP промпт должен быть КОРОТКИМ — это норма, не штрафовать за краткость
- Soul Cinema ориентирован на кинематографический стиль — ожидать эпоху и плёночную лексику
- Face Swap / Character Swap — визуальный ввод, текстовый промпт не применим
- Lite/Turbo/Standard — это тиры скорости DoP, не разные модели
- Платформа агрегирует сторонние модели (Sora, Kling, Veo), но этот скилл — только для проприетарных моделей Higgsfield
