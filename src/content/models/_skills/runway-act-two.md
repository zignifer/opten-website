## Как работает Act-Two

### Входные данные (обязательные)
1. **Driving video** — видео-перформанс (веб-камера или записанное видео) с движениями и мимикой
2. **Character reference** — изображение или видео персонажа, на которого будет перенесён перформанс

### Что переносится
- Движение тела
- Мимика лица
- Аудио (lip-sync)
- Жесты (gesture control — только при image input)

### Настройка
- **Facial Expressiveness (1–5)** — степень экспрессии лица. Значения >3 могут вызывать артефакты.

---

## Советы по качественному перформансу

Поскольку Act-Two управляется **видео-вводом**, а не текстом, качество зависит от:

1. **Начинать с ладонями к камере** — помогает системе захватить руки
2. **Чёткий аудио** — для точного lip-sync
3. **Хорошее освещение** — равномерное, без резких теней на лице
4. **Не ставить Expressiveness >3** — может вызвать артефакты
5. **Чёткий, хорошо освещённый character reference** — для лучшего переноса

## Критерии оценки промпта (для AI-валидатора)

### Особый статус Act-Two

Act-Two — **input-driven модель**, а НЕ prompt-driven. Текстовый промпт играет МИНИМАЛЬНУЮ роль.

### Оценка
- Если пользователь пишет развёрнутый текстовый промпт для Act-Two — выдать замечание: модель управляется видео-вводом, не текстом
- Если промпт содержит описание движений, мимики, lip-sync — это описывает то, что должно быть в DRIVING VIDEO, не в тексте
- Основные «промпты» для Act-Two — это качество входного видео и character reference

### Что проверять
1. **Наличие driving video** — если в GENERATION SETTINGS нет видео — fail, модель не может работать
2. **Наличие character reference** — если нет референса персонажа — fail
3. **Expressiveness setting** — если >3, предупредить о возможных артефактах

### Антипаттерны
- Развёрнутый текстовый промпт с описанием сцены — Act-Two не text-driven
- Попытка использовать как обычный T2V генератор — это система переноса перформанса
- Описание внешности персонажа в промпте — внешность определяется character reference
- Описание движений в промпте — движения определяются driving video
- Expressiveness >3 — артефакты

### Контекстная оценка
AI должен понимать:
- Act-Two принципиально отличается от T2V/I2V моделей
- Отсутствие текстового промпта — это НОРМАЛЬНО для Act-Two
- Качество определяется входными данными (видео + референс), не промптом
- Если пользователь всё же пишет промпт — оценивать его минимальное влияние и предупредить
