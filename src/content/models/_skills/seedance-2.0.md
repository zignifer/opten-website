## ⚠️ КРИТИЧЕСКИЕ ПРАВИЛА ВЫВОДА (язык и формат)

Эти правила имеют **наивысший приоритет** — они перекрывают любые примеры ниже в этом документе. При конфликте между этими правилами и каноническими шаблонами — следовать этим правилам.

### Правило 1 — Язык заголовков ВСЕГДА совпадает с языком промпта

Если `RESPOND_IN: ru` — **все** имена блоков, актов и полей должны быть на русском. Если `RESPOND_IN: en` — на английском. **Запрещено смешивать**: русское тело промпта + английские лейблы (`LOCATION:`, `Action:`, `Camera:`) — это критическая ошибка.

**Словарь обязательных переводов (для `RESPOND_IN: ru`):**

| Английский (канонический) | Русский (обязательный) |
|---------------------------|------------------------|
| `[TITLE & ACT]` | `[НАЗВАНИЕ И АКТ]` |
| `LOCATION` | `ЛОКАЦИЯ` |
| `REFERENCE ASSIGNMENT` | `НАЗНАЧЕНИЕ РЕФЕРЕНСОВ` |
| `STYLE` | `СТИЛЬ` |
| `STORY` | `СЮЖЕТ` |
| `CHARACTERS` | `ПЕРСОНАЖИ` |
| `SHOT STRUCTURE` | `СТРУКТУРА КАДРОВ` |
| `Action` | `Действие` |
| `Emotional Acting` | `Эмоциональная игра` |
| `Camera` | `Камера` |
| `Lighting` | `Свет` |
| `VFX` | `VFX` (техническая аббревиатура, не переводим) |
| `Audio Rule` | `Звук` |
| `[THE ENTRY]` | `[ЗАВЯЗКА]` |
| `[THE REVELATION]` | `[ПОВОРОТ]` |
| `[ACTION RESPONSE]` | `[РАЗВЯЗКА]` |
| `[SETUP]` | `[ЗАВЯЗКА]` |
| `[CLIMAX]` | `[КУЛЬМИНАЦИЯ]` |
| `[PAYOFF]` | `[РАЗВЯЗКА]` |

**Технические якоря, которые НЕ переводятся** (это keywords для платформы, не текст): `@image1`/`@audio1`/`@video1`, `4K`, `8K`, `FPS`, `HDR`, `16:9`, `2.35:1`, `dolly-in`, `crash-zoom`, `handheld`, `rim light`, `diegetic`, `cinematic`, `film grain` — оставлять as-is в любом языке. Также имена стилистических якорей в STYLE-блоке (`Ultra-photorealistic 4K live-action cinema`, `Gritty film grain`, `No cartoonish look`) — оставлять на английском, они работают как технический preset.

### Правило 2 — Формат: обязательные пустые строки и отступы

- Каждый TRY CGI-блок (`ЛОКАЦИЯ`, `СТИЛЬ`, `СЮЖЕТ` и т.д.) начинается с новой строки.
- Между блоками — **ровно одна пустая строка**. Никогда не клей конец одного блока со следующим заголовком (`...света.СТИЛЬ:` — БАГ; должно быть `...света.\n\nСТИЛЬ:`).
- Внутри `СТРУКТУРА КАДРОВ` между актами (`0-4с`, `4-10с`, `10-15с`) — **ровно одна пустая строка**.
- Каждое поле акта (`Действие`, `Эмоциональная игра`, `Камера`, `Свет`, `VFX`, `Звук`) — с новой строки, отступ два пробела перед именем поля.
- После двоеточия после заголовка — **всегда пробел** (`СТИЛЬ: Ultra-photorealistic`, не `СТИЛЬ:Ultra-photorealistic`).
- Тайминги пишем по локали: для `RESPOND_IN: ru` — `0-4с`, `4-10с`, `10-15с` (русское «с» вместо `s`); для `RESPOND_IN: en` — `0-4s`, `4-10s`, `10-15s`.

### Правило 3 — Эталонный русский шаблон вывода

Когда `RESPOND_IN: ru` и используется TRY CGI-структура, итоговый промпт должен выглядеть **строго так**:

```
ЛОКАЦИЯ: Городская площадь днём. Бетонное покрытие, деревянные скамейки, столы с посудой, толпа людей в фоне. Яркое естественное освещение с контровым боковым лучом света.

СТИЛЬ: Ultra-photorealistic 4K cinematic action, грейнистая плёнка, реалистичные мускулы и мех собаки, практическое освещение. Gritty realism как в боевике Нолана. Никакого AI-пластика, никаких мультяшных эффектов.

СЮЖЕТ: Собака в чёрном ошейнике совершает стремительный побег через городскую площадь, демонстрируя атлетические прыжки и манёвры между людьми и препятствиями.

ПЕРСОНАЖИ: Собака, мускулистая и энергичная, в чёрном ошейнике. Глаза сосредоточены, зрачки расширены от адреналина. Мех развевается от скорости.

СТРУКТУРА КАДРОВ (15 СЕК ВСЕГО):

0-4с — [ЗАВЯЗКА]
  Действие: Собака стартует с места, низко приседая перед прыжком, мускулы напряжены и готовы к взрыву.
  Эмоциональная игра: Концентрация и готовность. Зрачки расширены, взгляд острый, ноздри раздуты.
  Камера: Низкий ракурс, динамичный трекинг следом за собакой.
  Свет: Контровой боковой луч подсвечивает мех.
  Звук: Diegetic only — стук когтей по бетону, дыхание собаки.

4-10с — [ПОВОРОТ]
  Действие: Собака закладывает резкий поворот, перепрыгивая скамейку.
  Эмоциональная игра: Чистая фокусировка на цели, без страха.
  Камера: Боковой dolly, motion blur на заднем плане.
  Свет: Резкие тени от полуденного солнца.

10-15с — [РАЗВЯЗКА]
  Действие: Финальный рывок, собака пролетает в кадре крупным планом и уходит за рамку.
  Эмоциональная игра: Адреналиновая концентрация.
  Камера: Handheld, shaky cam, лёгкий motion blur.
  Звук: Только diegetic — шаги, дыхание. No music.

No music. Diegetic sound design only. Grounded realism, gritty film grain, no cartoonish look, no AI aesthetic.
```

Обрати внимание: пустая строка между каждым блоком и между каждым актом. Поля акта с отступом два пробела. После каждого `:` — пробел. Технические якоря (`Ultra-photorealistic 4K`, `diegetic`, `dolly`, `handheld`, `No music`) — на английском внутри русского текста, это нормально.

## Структура хорошего промпта

Оптимальная структура промпта для Seedance 2.0:

```
[Subject/Character] + [Scene/Environment] + [Action/Motion] + [Camera Movement] + [Timing Breakdown] + [Audio/Sound] + [Style/Mood]
```

Не обязательно использовать все элементы — состав зависит от типа видео. Но чем больше конкретики, тем лучше результат.

### 6-шаговая формула (рекомендуемая)

1. **Субъект** — кто/что в кадре (внешность, одежда, поза)
2. **Действие** — конкретные физические движения
3. **Сцена** — окружение, время суток, погода
4. **Камера** — тип плана, движение камеры, угол
5. **Стиль** — визуальный стиль, цветовая гамма, настроение
6. **Звук** — музыка, SFX, диалоги (если применимо)

## Обязательная структура блоков (TRY CGI master template)

Для cinematic-результата Seedance 2.0 ожидает промпт, разбитый на **именованные блоки в строгом порядке**. Это сильнее, чем просто 6-шаговая формула, потому что даёт модели «брифинг» как режиссёру.

**Канонический порядок:**

```
[TITLE & ACT]: Название сцены и номер акта (опционально, но помогает в длинных сюжетах)

LOCATION: Описание окружения. Освещение, погода, ключевые детали фона.
Если локаций несколько — описание перехода.

REFERENCE ASSIGNMENT: Назначение @-референсов (см. следующий раздел).

STYLE: Визуальный стиль (например: Ultra-photorealistic 4K live-action,
gritty superhero realism). Никакого «cartoonish» или «AI look».

STORY: Что происходит в этой конкретной генерации (1–2 предложения).

CHARACTERS: Участники, их текущее настроение, внешние особенности
в данный момент.

SHOT STRUCTURE: см. раздел «Структура 0–15s — канонический шаблон».
```

**Почему это работает лучше, чем «полотно текста»:** Seedance внутренне маршрутизирует разные блоки в разные слои генерации (location → environment model, characters → identity model, shot structure → temporal planner). Когда промпт смешан в один параграф, маршрутизация шумит и качество падает.

Если оцениваешь промпт и видишь «сплошной текст без названных блоков» — это валидное замечание: **«предложи разбить промпт на TRY CGI-блоки [TITLE & ACT] / LOCATION / REFERENCE ASSIGNMENT / STYLE / STORY / CHARACTERS / SHOT STRUCTURE»**.

## @-синтаксис референсов

Seedance 2.0 ожидает референсы через `@`-префикс, привязанный к роли:

- **@image1, @image2, @image3** — изображения (персонаж, локация, стиль).
- **@audio1, @audio2, @audio3** — аудио (голос, диегетические звуки, ритм).
- **@video1, @video2, @video3** — видео (для extension, copy camera, beat sync).

**Шаблон REFERENCE ASSIGNMENT:**

```
- Protagonist (@image1): Strict identity preservation. Use this image
  for exact facial features and wardrobe. No morphing or style changes.
- Lighting/Style (@image2): Reference for cinematic high-contrast
  lighting and color grading.
- Audio (@audio1): Reference for realistic electrical buzzing and
  low machinery hum.
```

**Критически важная фраза:** для персонажа всегда добавлять
**`Strict identity preservation. No morphing or style changes.`**

Без этой фразы модель будет «улучшать» лицо между секундами видео, и persistent character ломается. Это #1 совет TRY CGI для consistency.

Если в промпте есть @image-референс на персонажа, но нет фразы про identity preservation — это валидная рекомендация добавить её.

## Эмоциональный реализм — секрет «живого лица»

Главная причина, почему AI-видео выглядит «мёртвым» — отсутствие микромимики. Модель по умолчанию рендерит «маску» с правильной общей эмоцией, но без живых мелких движений. TRY CGI рекомендует **в каждом тайминге** прописывать минимум 2–3 микро-сигнала из трёх групп:

**Группа 1 — Взгляд:**
- `eyes darting` (глаза бегают)
- `pupils dilating` (зрачки расширяются)
- `focused on target / piercing gaze` (фокус на цели)
- `micro-tremor in the eyelids` (мелкое дрожание век)
- `screen light reflects in the pupils` (отражение экрана в зрачках)

**Группа 2 — Мышцы лица:**
- `jaw clenching` (сжатие челюсти)
- `nostrils flaring` (раздувающиеся ноздри)
- `brow tension / furrowed brow` (напряжение в бровях)
- `lips tightening / trembling` (сжатые/дрожащие губы)
- `cheek twitch` (нервный тик щеки)

**Группа 3 — Дыхание и тело:**
- `visible heavy breathing` (видимое тяжёлое дыхание)
- `shoulder movement on inhale` (движение плеч при вдохе)
- `visible sweat on the brow` (пот на лбу)
- `hand tremor` (дрожь руки)

**Пример хорошего таймингового описания** (4–10s блок из примера TRY CGI):

> Action: Character looks directly at the terminal screen.
> Emotional Acting: Shock. **Jaw clenches, nostrils flare, eyes widen as pupils dilate. Micro-tremor in the eyelids.**
> Camera: Slow Dolly-in to an extreme close-up of the eyes.
> Lighting: The screen light reflects realistically in the character's pupils.

Если в промпте есть таймповые блоки, но эмоциональная игра описана общими словами («он испуган», «она счастлива») — это самая высокоценная рекомендация: предложить добавить конкретную микромимику из этого лексикона.

## Структура 0–15s — канонический шаблон с именованными актами

Для 10–15-секундных видео TRY CGI рекомендует **3 именованных акта** с фиксированным каркасом полей. Это сильнее, чем абстрактная «timestamp storyboarding», потому что задаёт **драматургию**, а не просто хронометраж.

**Канонический 15s шаблон:**

```
SHOT STRUCTURE (15 SEC TOTAL):

0-4s — [THE ENTRY]
  Action: Что делает персонаж — конкретное физическое движение.
  Emotional Acting: Микромимика (см. раздел выше).
  Camera: Тип плана, движение, угол, объектив.
  Lighting: Источник, направление, тени.
  VFX: Дым, пыль, свет, частицы (если есть).
  Audio Rule: Какие звуки слышны (см. раздел Audio Rule).

4-10s — [THE REVELATION]
  Action: Точка разворота — субъект что-то видит/понимает.
  Emotional Acting: Эмоциональный сдвиг — shock, realization, fear.
  Camera: Часто dolly-in или crash-zoom на close-up.
  Lighting: Контрастная смена — screen reflection, rim light.

10-15s — [ACTION RESPONSE]
  Action: Решающее действие — побег, удар, бросок.
  Emotional Acting: Чистая адреналиновая концентрация.
  Camera: Handheld, shaky cam, motion blur.
  Audio Rule: Только diegetic — шаги, удары, дыхание. No music.
```

**Адаптация под другую длительность:**
- **8s:** 2 акта — [ENTRY] (0–4s) → [PAYOFF] (4–8s).
- **10s:** 2 акта — [SETUP] (0–4s) → [CLIMAX] (4–10s).
- **15s:** канонический 3-актовый шаблон выше.
- **>15s:** сегментировать (см. «Многосегментная генерация»), внутри каждого сегмента — свой 3-актовый шаблон.

Если промпт на 10–15s без названных актов и без полей Action/Emotional/Camera/Lighting/Audio — это валидная рекомендация: предложить TRY CGI-структуру с именованными актами.

## Audio Rule по умолчанию

Seedance 2.0 хорошо умеет генерировать аудио, но **дефолт должен быть тишина + diegetic**, не музыка.

**Правило:** если пользователь явно не попросил background music — её **не должно быть** в промпте. Музыка часто ломает погружение в cinematic-контексте (как в реальном кино без саундтрека в hard-сценах).

**Что писать в Audio Rule блока:**
- ✅ `Only the sound of heavy boots on metal and electrical crackling.`
- ✅ `Diegetic only: footsteps, distant traffic, jacket rustling.`
- ✅ `Realistic electrical buzzing and low machinery hum. No music.`
- ❌ `Epic orchestral soundtrack` (если пользователь не попросил)
- ❌ `Dramatic background score swelling`

**Универсальный финальный якорь для cinematic-промптов:**
`No music. Diegetic sound design only.`

Если промпт упоминает «epic music», «soundtrack», «soaring score» без явного запроса от пользователя — это валидное замечание: убрать в пользу diegetic, чтобы получить cinematic-realism, а не trailer-look.

## Анти-AI-look фразы (защита от «мультяшности»)

Seedance 2.0 без явных указаний склонна выдавать «AI-эстетику»: пластиковая кожа, чрезмерно тёплый свет, hyperreal-stylization. TRY CGI рекомендует **финальную строку** с эксплицитными анти-AI-якорями:

**Готовый стек для photorealism:**
- `Ultra-photorealistic 4K live-action cinema.`
- `Grounded realism.`
- `Gritty film grain.`
- `No cartoonish look, no AI aesthetic, no over-stylization.`
- `Practical lighting, no glossy AI render look.`
- `Natural skin texture with visible pores and imperfections.`

**Для superhero / action жанра:**
- `Gritty superhero realism, the look of Nolan's Dark Knight, not animation.`

**Для documentary:**
- `Documentary cinema verité, handheld realism, the look of HBO drama, not Netflix gloss.`

Эти фразы лучше всего помещать **в STYLE-блок** или в самом конце промпта как финальный якорь.

Если промпт под Seedance 2.0 описывает реалистичную сцену, но нет анти-AI-фраз и в STYLE написано просто «cinematic» — это валидная рекомендация добавить эти якоря, потому что без них модель почти гарантированно уйдёт в plastic AI-look.

## 10 типов генерации Seedance 2.0

### 1. Чистый текст (Text-to-Video)
Генерация только из текстового описания, без референсов.
- Формула: `(Субъект) + (Действия) + (Среда/Свет) + (Камера) + (Стиль)`
- Пример: «Камера следует за мужчиной в чёрном, который убегает по переулку. Боковой трекинг, он в панике сбивает фруктовый прилавок, встаёт и бежит дальше. Звуки толпы и паники.»

### 2. Consistency Control (одинаковый персонаж/продукт)
Через @-референсы изображений сохраняем внешность персонажа, продукта, или сцены через несколько генераций.
- Формула: `[Персонаж]@Image + [Действие] + [Сцена]@Image + [Камера]`

### 3. Копирование камеры и движений
Загружаем референсное видео и копируем его камерные движения, хореографию, ритм.
- Формула: `Ссылка на @Video (камера/действие/ритм) + [Субъект]@Image + [Сцена]`

### 4. Копирование спецэффектов и шаблонов
Воспроизведение переходов, рекламных шаблонов, VFX.
- Формула: `Ссылка на @Video (эффекты/переходы) + замена элементов на @Image + [Описание]`

### 5. Дополнение сюжета (Story Completion)
Модель достраивает сюжет из раскадровки или изображений.
- Формула: `[Описание раскадровки/изображения] + [Способ показа] + [Звук/Диалоги]`

### 6. Продление видео (Video Extension)
Плавное продолжение существующего видео вперёд или назад.
- Формула: `Продлить @Video на X секунд + [Описание нового контента]`
- Важно: выбранная длительность = длительность НОВОГО контента

### 7. Звуковой контроль
Клонирование голоса, генерация диалогов, звуковой дизайн.
- Формула: `[Визуал] + голос/BGM ссылка на @Video или @Audio + [Диалоги в кавычках]`

### 8. Одним дублем (One-Take Long Shot)
Непрерывный план без монтажных склеек.
- Формула: `Одним дублем + @Image1 @Image2... + [Последовательное описание сцен] + "без монтажных склеек"`

### 9. Редактирование видео
Замена персонажей, изменение сюжета, добавление/удаление элементов.
- Формула: `Заменить [A] в @Video на @Image + [Дополнительные изменения]`

### 10. Музыкальная синхронизация (Beat Sync)
Визуальный ритм синхронизируется с музыкой.
- Формула: `@Image1 @Image2... + ссылка на ритм @Video + [Стиль]`

## Продвинутые техники

### Тайминг-раскадровка (Timestamp Storyboarding)
Для 13-15 секундных видео — разбивка по секундам даёт точный контроль:

```
0-3с: [сцена + камера + звук]
4-8с: [сцена + камера + звук]
9-12с: [сцена + камера + звук]
13-15с: [сцена + камера + звук]
```

Это самая мощная техника для длинных генераций.

### Технические параметры в начале промпта
```
[Ориентация] + [Соотношение сторон: 2.35:1/16:9/9:16] + [FPS: 24fps] + [Длительность] + [Стиль/тон]
```

### Негативные инструкции
В конце промпта можно указать что НЕ нужно:
```
Запрещено: текст, субтитры, логотипы, водяные знаки
```
Примечание: на syntx.ai этот формат может работать иначе чем на нативной платформе 即梦.

## Рекомендации по длительности

| Длительность | Подход |
|-------------|--------|
| 4-8 сек | Простая сцена. 1-2 ключевых момента. Тайминг не обязателен. |
| 9-12 сек | Полная сцена. 2-3 этапа. Тайминг рекомендуется. |
| 13-15 сек | Полный нарратив. Тайминг-раскадровка обязательна для хорошего результата. |
| >15 сек | Разбивать на сегменты по 15 сек. Каждый следующий = продление предыдущего через @Video. |

### Многосегментная генерация (>15 сек)

1. Сегмент 1: нормальная генерация (до 15с). Закончить на «чистом» кадре.
2. Сегмент 2+: загрузить предыдущее видео как @Video1, написать «Продлить @Video1 на Xс» + описание.
3. Между сегментами — описание точки склейки (что на последнем кадре).

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что в кадре. Конкретно, не абстрактно.
2. **Действие** — физические действия, не абстрактные концепции. «Идёт, поворачивается, берёт» вместо «что-то происходит».
3. **Камера** — хотя бы одно указание на крупность или движение камеры.
4. **Сцена/среда** — где происходит действие.

### Бонусные элементы (улучшают результат)
5. **Тайминг** — раскадровка по секундам (особенно важно для >10 сек).
6. **Стиль/настроение** — визуальный стиль, цветовая палитра, атмосфера.
7. **Освещение** — конкретные источники света и их характер.
8. **Звук** — звуковой дизайн, диалоги, музыка, SFX.
9. **Язык** — английский даёт лучшие результаты на syntx.ai.

### Бонус для Seedance 2.0
10. **Сложные цепочки действий** — модель лучше справляется с multi-step actions, поэтому их наличие — плюс.
11. **Consistency-референсы** — использование @Image/@Video для сохранения стиля между генерациями.

### Антипаттерны (ухудшают результат)
- Слишком короткий промпт (<15 слов) — модель будет додумывать слишком много
- Слишком длинный промпт (>2000 символов) — перегрузка деталями
- Конфликтующие камерные движения одновременно (zoom + pan + orbit)
- Абстрактные/размытые формулировки («что-то красивое происходит»)
- Одиночный промпт на >15 секунд (лимит платформы)
- Негативные формулировки без позитивной альтернативы
- Противоречие между текстовым описанием и загруженным референсом

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Мультяшное видео не требует реалистичного освещения
- Простой продуктовый шот не требует сложной раскадровки
- Абстрактное арт-видео может не иметь конкретного субъекта
- Оценка должна учитывать тип задачи, а не слепо проверять наличие всех элементов
- Seedance 2.0 поддерживает самые сложные промпты — можно ожидать высокого уровня детализации
