## Структура хорошего промпта

Оптимальная иерархия промпта для MidJourney (слова в начале имеют больший вес):

```
[Субъект] + [Детали субъекта] + [Контекст/Среда] + [Стиль/Настроение] + [Техника/Камера/Освещение] + [Параметры]
```

Пример:
```
An elderly fisherman with a weathered face and silver beard, standing on a wooden dock at dawn, documentary photography style, contemplative mood, shot on Leica M11 with natural morning light --ar 3:2 --s 100 --v 7
```

### Ключевой принцип: описывай как для кинооператора

V7+ понимает естественный язык. Пиши связные описания, а не списки ключевых слов через запятую. Вместо «mountain, fog, sunrise, epic, cinematic» пиши «A snow-capped mountain emerging from fog at sunrise, dramatic warm light on the peaks, wide-angle composition».

## Параметры MidJourney

### Основные параметры

| Параметр | Диапазон | По умолчанию | Описание |
|----------|----------|-------------|----------|
| `--ar` | Любое соотношение | 1:1 | Соотношение сторон (например, --ar 16:9) |
| `--v` | 5, 5.1, 5.2, 6, 7, 8 | 7 | Версия модели |
| `--s` / `--stylize` | 0–1000 | 100 | Степень художественной интерпретации |
| `--c` / `--chaos` | 0–100 | 0 | Разнообразие между 4 вариациями |
| `--w` / `--weird` | 0–3000 | 0 | Необычность/экспериментальность результата |
| `--no` | текст | — | Негативный промпт (что исключить) |
| `--seed` | 0–4294967295 | случайный | Фиксирует начальный шум для воспроизводимости |
| `--q` / `--quality` | 1–4 | 1 | Качество/детализация (4 = макс. когерентность в V8) |
| `--tile` | — | выкл | Генерирует бесшовные паттерны |
| `--repeat` | 1–40 | 1 | Повторяет генерацию N раз |

### Параметры стиля и персонализации

| Параметр | Описание |
|----------|----------|
| `--style raw` | Минимальная художественная обработка, максимально буквальная интерпретация |
| `--p` / `--personalize` | Применяет профиль персональных предпочтений пользователя |
| `--sref [URL]` | Style Reference — переносит визуальный стиль из изображения |
| `--sw` | Style Weight — сила влияния style reference (0–1000) |
| `--cref [URL]` | Character Reference — сохраняет внешность персонажа (не в V7/V8, есть в Niji 6) |
| `--cw` | Character Weight — сила влияния character reference |
| `--oref` | Omni Reference — универсальная ссылка (V7) |
| `--iw` | Image Weight — вес изображения-референса (0–3) |
| `--niji` | Переключение на аниме-модель Niji |
| `--exp` | Experimental — усиленная детализация, динамика (0–100) |

### Стилизация: когда какую использовать

| Диапазон --s | Эффект | Подходит для |
|-------------|--------|-------------|
| 0–50 | Минимальная интерпретация | Продуктовые фото, техническая точность |
| 50–150 | Сбалансированный (по умолчанию) | Общее использование, портреты |
| 150–300 | Заметный стиль | Художественные фото, атмосферные работы |
| 300–500 | Сильный стиль | Иллюстрации, концепт-арт |
| 500–1000 | Очень стилизованно | Абстракция, экспериментальные работы |

## Мульти-промпты и веса

Разделение концепций через `::` с весами:

```
cat::2 spaceship::1
```

Вес 2 у «cat» означает, что кот будет доминировать в изображении. Полезно для контроля баланса между элементами.

## Негативный промпт (--no)

Указывает, что исключить из генерации:

```
portrait of a woman --no earrings, necklace, hat, makeup
landscape --no text, watermark, people, cars
```

В V8 параметр `--no` работает значительно надёжнее, чем в предыдущих версиях.

## Типичные сценарии использования

### Фотореалистичные портреты
- Указывай освещение, объектив, настроение
- `--s 50-150` + `--style raw` для максимального реализма
- Пример: `Portrait of a young woman, soft window light from the left, 85mm lens, shallow depth of field, editorial photography --ar 4:5 --s 75 --v 7`

### Иллюстрации и концепт-арт
- Указывай стиль, художественное направление, палитру
- `--s 300-600` для выразительности
- Пример: `A floating city in the clouds, steampunk architecture, warm sunset palette, concept art, detailed environment design --ar 16:9 --s 400 --v 7`

### Продуктовая фотография
- Минимальная стилизация, точное описание
- `--s 0-50` + `--style raw`
- Пример: `Product photograph of a luxury watch on black marble surface, studio lighting, clean white background, sharp focus --ar 1:1 --s 25 --style raw --v 7`

### Абстракция и экспериментальные работы
- Высокая стилизация + weird
- Пример: `Abstract flowing energy, bioluminescent colors, organic forms --s 750 --weird 1500 --chaos 50 --v 7`

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что изображено. Должен быть конкретным: «elderly fisherman with a silver beard» вместо «a man».
2. **Контекст/Среда** — где находится субъект. «On a wooden dock at dawn» вместо «outside».
3. **Стиль или медиум** — тип изображения: фотография, иллюстрация, 3D-рендер и т.д.

### Бонусные элементы (улучшают результат)
4. **Освещение** — конкретные источники и характер света. Самый мощный фактор качества.
5. **Камера/Оптика** — объектив, глубина резкости, ракурс.
6. **Настроение/Атмосфера** — эмоциональный тон сцены.
7. **Цветовая палитра** — конкретные цвета или тональное направление.
8. **Параметры** — осмысленное использование --ar, --s, --style raw и других.
9. **Материалы/Текстуры** — конкретные описания материалов поверхностей.

### Антипаттерны (ухудшают результат)
- **Спам качества** — «beautiful, stunning, 8k, detailed, masterpiece, best quality» — V7+ это не улучшает, а засоряет промпт
- **Списки ключевых слов через запятую** вместо связных описаний — MidJourney V7+ понимает естественный язык
- **Слишком короткий промпт** (<5 слов) — модель додумает слишком много
- **Слишком длинный промпт** (>150 токенов) — детали начинают конфликтовать
- **Противоречивые инструкции** — «dark, bright, moody, cheerful» одновременно
- **Стиль в начале промпта** — «beautiful cinematic photo of...» вместо описания субъекта первым
- **Конфликт параметров** — одновременно высокий --s и --style raw работают друг против друга
- **Забытый --ar** — соотношение сторон критически влияет на композицию, менять его после итераций = начинать заново
- **Абстрактные формулировки** — «something cool» или «nice scene» ничего не дают модели
- **Использование устаревших версий** без причины — V5/V6 значительно уступают V7+

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Продуктовое фото не требует художественной стилизации
- Абстрактное искусство может не иметь конкретного субъекта — это нормально
- Промпт для аниме (без --niji) стоит предложить перенести на Niji
- Очень короткие промпты могут быть намеренно минималистичными — если указан стиль, это допустимо
- Оценка должна учитывать версию модели: советы для V8 отличаются от V7
