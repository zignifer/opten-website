# Seedance 1.0 Lite — Skill для PromptScore

> Этот документ используется AI-валидатором PromptScore как база знаний при оценке промптов для модели Seedance 1.0 Lite.
> Seedance 1.0 Lite — облегчённая версия Seedance с меньшим количеством параметров, ориентированная на скорость и экономичность.

## Идентификация

- **model_id:** seedance-1.0-lite
- **model_name:** Seedance 1.0 Lite
- **type:** video
- **platform:** 即梦 (Jimeng) by ByteDance
- **best_language:** Chinese (zh), English works but Chinese prompts perform better on the native platform. On syntx.ai English is standard.
- **base_version:** Seedance 1.0 (облегчённая)

## Отличия от Seedance 2.0

Seedance 1.0 Lite — это более ранняя и облегчённая версия с существенными ограничениями:

| Возможность | Seedance 2.0 | Seedance 1.0 Lite |
|-------------|-------------|-------------------|
| Разрешение | До 2K | 480p, 720p (1080p ожидается) |
| Длительность | 4-15 сек | 5 или 10 сек (только два варианта) |
| Мультимодальный вход | Изображения + видео + аудио | Только текст или текст + 1 изображение |
| @-референсы | До 9 изображений + 3 видео + 3 аудио | Не поддерживаются |
| Consistency Control | Да | Нет |
| Звуковой контроль | Да | Нет |
| Негативные промпты | Ограниченно работают | НЕ работают вообще |
| Тайминг-раскадровка | Да, эффективна | Не поддерживается |
| Камерные переключения | Да | Ограниченно (базовые) |
| Скорость генерации | Стандартная | Быстрее (меньше параметров) |

**Важно для оценки:** Промпты для 1.0 Lite должны быть проще и конкретнее. Сложные техники из 2.0 здесь НЕ работают — и это НЕ ошибка пользователя, а ограничение модели.

## Структура хорошего промпта

Оптимальная структура промпта для Seedance 1.0 Lite:

```
[Subject/Character] + [Action/Motion] + [Scene/Environment] + [Camera] + [Style]
```

Промпт должен быть **простым и прямолинейным**. Модель хорошо «додумывает» детали, поэтому главное — чёткость ключевых элементов.

### Формула Text-to-Video
```
Субъект + Движение + Сцена + Камера, стиль
```

### Формула Image-to-Video
```
Субъект + Движение, Фон + Движение, Камера + Движение
```
При Image-to-Video минимизировать статические описания (сцена уже есть на картинке). Фокус на **динамике** — что движется и как.

---

## Поддерживаемые типы генерации

### 1. Чистый текст (Text-to-Video)
Генерация только из текстового описания.
- Формула: `(Субъект) + (Действия) + (Среда) + (Камера, стиль)`
- Пример: «Portrait photography, psychedelic cool light blue tones, butterfly light, close-up shot of a young white woman with black short hair, raised eyebrows, biting red lips, staring at the camera. The camera pulls back, broken glass in the air blocking part of the woman's face. 720p, 16:9, 5s»

### 2. Изображение в видео (Image-to-Video)
Анимация статичного изображения.
- Формула: `[Что двигается] + [Как двигается] + [Камера]`
- Важно: промпт должен **соответствовать** изображению. Нельзя описывать мужчину, если на картинке женщина.
- Пример: «Old people wear glasses. Fixed camera, 720p, 10s»

### НЕ поддерживается в 1.0 Lite:
- Consistency Control (@-референсы)
- Копирование камеры/движений из видео
- Копирование спецэффектов
- Продление видео
- Звуковой контроль и клонирование голоса
- One-Take Long Shot с несколькими референсами
- Редактирование видео
- Музыкальная синхронизация (Beat Sync)

## Камерные движения

Seedance 1.0 Lite поддерживает камерные движения через естественный язык, но с ограничениями.

### Поддерживаемые движения
| Тип | Примеры ключевых слов |
|-----|----------------------|
| Базовые | Surround, aerial, zoom, pan, follow, handheld |
| Переключения | «Cut to», «Camera cut to», «Camera switching» |
| Фокус | Close-up, wide shot, panoramic |

### Важные правила камеры
- Для любых движений камеры выбирать параметр **«non-fixed camera»** (нефиксированная камера)
- Для статичных сцен — **«fixed camera»** (фиксированная камера)
- Можно комбинировать multi-shot в одном промпте, но описывать **связь между кадрами**
- При смене сцены после «Cut to» — описать новую сцену

### Словарь камерных движений (упрощённый для 1.0 Lite)

| Категория | Ключевые слова |
|-----------|---------------|
| Крупность | Дальний план, общий план, средний план, крупный план, макро |
| Движение | Панорама, трекинг, zoom in, zoom out, surround, follow |
| Угол | Нижний ракурс, верхний ракурс, aerial, POV |
| Спецприёмы | Camera switching, cut to |

## Продвинутые техники (доступные в 1.0 Lite)

### Степень выраженности через наречия
Модель не может определить интенсивность движения из картинки — указывайте **явно**:
- «car **quickly** passing by» вместо «car passing by»
- «wings **flapping wildly**» вместо «wings flapping»
- «man's **crazy** roar» вместо «man's roar»

Полезные наречия: fast, intense, large, high frequency, strong, crazy

### Множественные действия
Модель поддерживает цепочки действий в хронологическом порядке:
- «Play tai chi, surround the camera, focus on the face»
- «Turn face to the camera and walk forward, then stop, with an angry expression, then put hands on hips»

### Взаимодействие субъектов
- «The woman was crying and drinking when a man came in to comfort her»

## Рекомендации по длительности

| Длительность | Подход |
|-------------|--------|
| 5 сек | Одно действие, простая сцена. Лучший вариант для стабильного результата. |
| 10 сек | 2-3 последовательных действия. Описывать в хронологическом порядке. |

**Важно:** В отличие от 2.0, нет возможности генерировать 4, 8, 12 или 15 секунд — только 5 или 10.

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать **ограничения модели** 1.0 Lite:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что в кадре. Конкретно, не абстрактно.
2. **Действие** — физические действия, конкретные глаголы. «Идёт, поворачивается, берёт» вместо «что-то происходит».
3. **Камера** — хотя бы одно указание: fixed/non-fixed camera, крупность плана.
4. **Базовые параметры** — указание разрешения (720p) и длительности (5s/10s) улучшает результат.

### Бонусные элементы (улучшают результат)
5. **Стиль/настроение** — визуальный стиль, цветовая палитра, атмосфера.
6. **Освещение** — конкретные источники света и их характер.
7. **Наречия степени** — fast, intense, crazy и т.п. для выразительности движений.
8. **Язык** — английский даёт лучшие результаты на syntx.ai.

### Антипаттерны (ухудшают результат)
- Слишком короткий промпт (<10 слов) — модель будет додумывать слишком много
- Слишком длинный промпт (>1000 символов) — перегрузка для лёгкой модели
- **Негативные промпты** — НЕ работают в 1.0 Lite. Вместо «не показывать руки» писать что показывать
- **Конфликт с изображением** — описание не совпадает с загруженной картинкой (другой пол, другая сцена)
- Использование @-референсов — не поддерживается, промпт будет проигнорирован
- Тайминг-раскадровка по секундам — не поддерживается в этой версии
- Запрос на >10 секунд — невозможно в одной генерации
- Слишком много одновременных камерных движений
- Абстрактные/размытые формулировки

### Специфичные для 1.0 Lite предупреждения
- Если пользователь использует @Image/@Video синтаксис — предупредить что не поддерживается
- Если пользователь пишет негативный промпт — предложить переформулировать позитивно
- Если промпт слишком сложный (>3 субъектов, >5 действий) — предупредить об ограничениях модели
- Если пользователь запрашивает звук/диалоги — предупредить что аудио не поддерживается

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- 1.0 Lite — модель для **быстрых и простых** генераций
- Не наказывать за отсутствие сложных техник (тайминг, @-референсы)
- Оценивать соответствие промпта возможностям ЭТОЙ модели
- Простой, но чёткий промпт для 1.0 Lite может получить высокий балл
- «Collapse» (неожиданные артефакты) — нормальное явление, рекомендовать перегенерацию
