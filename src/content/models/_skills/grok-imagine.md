### Режим редактирования (Edit)

Grok Imagine Edit — **тот же модельный бэкенд**, не отдельная модель. Доступ через `POST /v1/images/edits`.
- Принимает 1–5 входных изображений + промпт
- При редактировании одного изображения aspect ratio берётся из источника
- Промпт описывает только ЧТО ИЗМЕНИТЬ, не всю сцену

## Структура хорошего промпта

Grok Imagine принимает **естественный язык** — описательные предложения, НЕ теги.

```
[Subject] + [Style/Mood] + [Lighting] + [Camera Angle] + [Finishing Details]
```

### Принципы промптинга

- Режиссёрский язык: позиция камеры, тип объектива, направление света
- Конкретная атмосфера: «nostalgic», «melancholic», «electric» — не «happy», «cool», «nice»
- Визуальные детали: время суток, цветовая палитра, материалы поверхностей
- Одна чёткая сцена за генерацию
- Связные описательные предложения — НЕ список ключевых слов

### Для режима редактирования

- Описывать только ЧТО ДОЛЖНО ИЗМЕНИТЬСЯ
- Фиксировать то, что НЕ должно меняться
- Итерировать по одной переменной за раз

## Ключевые возможности

1. **Фотореализм** — особенно сильны реалистичные человеческие портреты
2. **Точный рендеринг текста** в изображениях — ключевое отличие от конкурентов
3. **Высокая верность промпту** благодаря авторегрессивной архитектуре
4. **Стилевая гибкость** — фотореализм, аниме, акварель, масло, поп-арт в одной модели
5. **Multi-turn editing** — цепочка итеративных правок
6. **Multi-image compositing** — объединение до 5 входных изображений
7. **Рендеринг реальных объектов** — меньше ограничений чем у конкурентов

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что изображено. Конкретно и описательно.
2. **Визуальный стиль** — фотореализм, акварель, аниме и т.д. Или настроение/атмосфера.
3. **Описательный язык** — полные предложения, не список тегов.

### Бонусные элементы (улучшают результат)
4. **Освещение** — конкретный тип: golden hour, rim light, studio softbox.
5. **Камера/ракурс** — позиция, объектив, угол съёмки.
6. **Атмосфера** — специфические слова настроения: «nostalgic», «melancholic», «electric».
7. **Материалы/текстуры** — поверхности, ткани, зернистость.
8. **Время суток и цветовая палитра** — для создания атмосферы.

### Антипаттерны (ухудшают результат)
- **Негативные промпты** — «no X», «don't include Y», «without Z» — модель полностью игнорирует. Описывать ТОЛЬКО то, что нужно.
- **Keyword stacking** — «masterpiece, best quality, 8k, ultra detailed» — контрпродуктивно для авторегрессивной модели.
- **SD-синтаксис** — веса `(word:1.5)`, LoRA-ссылки, специальные токены — НЕ работают.
- **Сложные мульти-сценовые промпты** — одна чёткая сцена за генерацию.
- **Общие прилагательные** — «nice», «cool», «good» — использовать специфические атмосферные слова.
- **Изменение всего сразу при итерации** — менять одну переменную за раз в Edit mode.
- **Ожидание pixel-level контроля** — редактирование prompt-driven и холистическое.
- **Противоречие входному изображению** в режиме Edit.

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Grok Imagine — natural language model, описательные предложения — норма
- Короткий чёткий промпт для простой сцены — это нормально
- Для Edit mode — промпт описывает ИЗМЕНЕНИЕ, не всю сцену
- Multi-image compositing — промпт описывает как объединить
- Негативные формулировки — ВСЕГДА выдавать замечание, это критический антипаттерн
- Keyword stacking — ВСЕГДА выдавать замечание
