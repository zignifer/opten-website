## Структура хорошего промпта

Оптимальная структура промпта для Veo 3:

```
[Subject + Appearance] + [Context/Scene] + [Action] + [Camera Movement] + [Style/Mood] + [Lighting] + [Dialogue/Audio]
```

Veo 3 — первая модель семейства Veo с нативной генерацией аудио. Это значит, что промпт ДОЛЖЕН включать описание звуковой составляющей, иначе модель будет додумывать (часто неудачно).

---

## Ключевая особенность: Генерация аудио и диалогов

### Veo 3 генерирует аудио автоматически
Каждое видео создаётся со звуком. Аудио включает:
- **Диалоги персонажей** — модель может озвучивать речь
- **Фоновые звуки** — ambience, окружение
- **Звуковые эффекты** — SFX привязанные к действиям
- **Музыка** — фоновая музыка под настроение

### Промптинг диалогов
Два подхода:

**Явный (Explicit):** Точный текст после двоеточия
```
A guy says: My name is Ben
```

**Неявный (Implicit):** Описание того, что персонаж говорит
```
A guy tells us his name
```

Явный подход — для точного контроля слов. Неявный — когда модель может сама придумать текст.

### Правила диалогов
- **Диалог должен умещаться в ~8 секунд** произнесения. Слишком длинный текст = неестественно быстрая речь.
- **Слишком короткий диалог** = AI-мусор и бессмысленное бормотание в оставшееся время.
- **Произношение:** для правильного произношения сложных слов используйте фонетическую запись: «foh-fur» вместо «fofr».
- **Кто говорит что:** при нескольких персонажах чётко указывайте кто говорит: «The woman wearing pink says: ... The man with glasses replies: ...»
- **Речь через двоеточие, НЕ кавычки:** `says: My name is Ben` работает лучше чем `says "My name is Ben"` (кавычки провоцируют субтитры).

### Избежание субтитров
Veo 3 часто генерирует вшитые субтитры (бывают с ошибками). Как избежать:
- Добавьте `(no subtitles)` или `(no subtitles!)` в промпт
- Используйте двоеточие для диалогов вместо кавычек
- При необходимости повторите: `No subtitles. No subtitles!`

### Промптинг фоновых звуков
Если НЕ указать фоновый звук, модель додумает сама — часто неудачно. Типичная проблема: **unwanted studio audience laughter** (смех аудитории как в ситкоме).

**Решение:** Всегда явно описывайте фоновые звуки:
```
sounds of distant bands, noisy crowd, ambient background of a busy festival field
```

### Промптинг музыки
Для музыки в сцене — опишите жанр, настроение, стиль:
```
a tense cinematic score plays in the background
a cheerful upbeat pop melody
```

## Основные типы генерации

### 1. Text-to-Video (основной режим)
Генерация видео с аудио из текстового описания.
- Формула: `(Субъект + Внешность) + (Действие + Диалог) + (Среда) + (Камера) + (Стиль) + (Звуки/Музыка)`

### 2. Selfie-видео
Veo 3 отлично генерирует selfie-стиль.
- Начинайте с «A selfie video of…»
- Укажите видимую руку: «holds the camera at arm's length, arm clearly visible in frame»
- Добавьте естественные движения глаз: «occasionally looking into the camera»
- Добавьте текстуру: «slightly grainy, film-like image» — убирает «AI-чистоту»

## Продвинутые техники

### Консистентность персонажей
Veo 3 при одинаковом промпте генерирует почти идентичных персонажей. Используйте это:
- Создайте детальное описание персонажа и повторяйте его дословно в разных сценах
- Пример: «John, a man in his 40s with short brown hair, wearing a blue jacket and glasses, looking thoughtful»
- Чем уникальнее описание — тем стабильнее консистентность между генерациями

### Стилистические модификаторы
Задайте стиль через «In the style of [style]:»

| Стиль | Описание |
|-------|----------|
| LEGO | Персонажи и сцена из LEGO |
| Claymation | Пластилиновая анимация |
| Pixar animation | 3D мультипликация в стиле Pixar |
| Anime | Японская анимация |
| 8-bit retro | Пиксельная ретро-графика |
| Graphic novel | Стиль графического романа/комикса |
| South Park | Стиль мультфильма South Park |
| Origami | Стиль оригами |
| Blueprint | Чертёжный стиль |
| Marble | Мраморная скульптура |
| Simpsons | Стиль мультфильма Симпсоны |

### Физика
Veo 3 хорошо симулирует реалистичную физику: падение, отскок, жидкости, движение объектов. Физика сохраняется даже при смене стиля.

### Изменение промпта при каждой генерации
Veo 3 очень консистентен — один и тот же промпт даёт почти одинаковый результат при разных seed. Если нужны вариации — **меняйте промпт**, а не перегенерируйте.

## Типичные сценарии использования

### Реклама / Коммерческое видео
- Продуктовые демо, промо-ролики
- Стилизация под коммерческую съёмку
- Закадровый голос + визуальный ряд

### Короткие истории / Диалоговые сцены
- Персонажи с детальным описанием внешности
- Явные диалоги через двоеточие
- Фоновые звуки и атмосфера прописаны явно
- `(no subtitles!)` в конце

### Подкасты / Talking Head
- «A podcast show, [описание ведущего], talks into a mic saying: [текст]»
- Указание «no subtitles»

### Арт / Стилизация
- Стилистический модификатор: «In the style of LEGO:»
- Физика сохраняется при смене стиля

### Селфи-контент
- «A selfie video of…» + видимая рука + естественные движения
- Добавление зернистости: «slightly grainy, film-like»

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что в кадре. Конкретное описание с деталями внешности: одежда, возраст, причёска, выражение лица. «A man in his 40s with short brown hair, wearing a blue jacket» вместо «a man».
2. **Действие** — что происходит в кадре. Физические действия, глаголы: «picks up the phone, turns, walks, laughs» вместо «something happens».
3. **Контекст/Среда** — где происходит действие: «in a quiet forest clearing», «on a busy Tokyo street market».
4. **Камера** — хотя бы одно указание: крупность плана, движение камеры или угол съёмки.

### Бонусные элементы (улучшают результат)
5. **Аудио/Диалоги** — КРИТИЧЕСКИ ВАЖНО для Veo 3. Описание звуковой составляющей: что говорят персонажи, фоновые звуки, музыка. Без этого модель додумает сама (часто плохо).
6. **Указание «no subtitles»** — если есть диалоги, добавить `(no subtitles!)` во избежание вшитых субтитров.
7. **Стиль/Настроение** — визуальный стиль, цветовая палитра, атмосфера.
8. **Освещение** — конкретные источники света и их характер.
9. **Детали для консистентности** — при серии генераций: подробное повторяемое описание персонажей.

### Антипаттерны (ухудшают результат)
- Слишком короткий промпт (<10 слов) — модель будет додумывать слишком много, результат непредсказуемый
- **Отсутствие описания аудио** — модель добавит случайный звук, часто неуместный «studio audience laughter»
- **Слишком длинный диалог** для 8-секундного видео — персонаж будет говорить неестественно быстро
- **Слишком короткий диалог** (1-2 слова) — модель заполнит оставшееся время AI-бормотанием
- **Диалоги в кавычках** вместо двоеточия — провоцирует генерацию вшитых субтитров
- Конфликтующие камерные движения (zoom in + zoom out, static + tracking)
- Абстрактные формулировки без конкретики («something beautiful», «cool video»)
- Перегенерация одного промпта вместо его изменения (Veo 3 даёт одинаковый результат при одинаковом промпте)
- Попытка вертикального видео (нативно не поддерживается — только постобработка)
- Негативные формулировки без позитивной альтернативы
- Отсутствие описания фоновых звуков при наличии диалогов

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Для talking head / подкаста — камерная работа не так критична, но аудио обязательно
- Для арт-видео — стиль важнее реалистичного освещения
- Для selfie-контента — специфический формат промпта
- Для диалоговых сцен — аудио-элементы (диалоги, фон, «no subtitles») критически важны
- Оценка должна учитывать тип задачи, а не слепо проверять наличие всех элементов
- Если в промпте есть диалоги, но нет `(no subtitles)` — это важное замечание
- Если в промпте есть персонажи, но нет описания фоновых звуков — предупредить о возможном «studio audience»
