## Структура хорошего промпта

### Text-to-Video (T2V)

Gen-4.5 — первая модель Runway с самостоятельным T2V. Полное описание сцены:

```
[Camera] shot of [subject] [action] in [environment]. [Supporting descriptions]
```

### Image-to-Video (I2V)

Как в Gen-4: описывать ТОЛЬКО движение. Входное изображение определяет визуал.

### Timestamp-синтаксис (Sequential Prompting)

Уникальная фича Gen-4.5 — последовательные действия по таймкодам:

```
[00:01] A bird takes off from a branch.
[00:03] It soars over a misty valley.
[00:06] Camera pulls back to reveal the full mountain range.
```

**Правила timestamps:**
- Таймкоды должны быть реалистичны относительно выбранной длительности
- Не «ходить через комнату за 0.5 секунды»
- Давать достаточно времени между действиями

## Ключевые возможности

1. **Первый Runway с T2V** — полноценная генерация из текста
2. **Физика** — улучшенная симуляция веса, момента, жидкостей, ткани
3. **Timestamp-синтаксис** — последовательная режиссура через `[00:01]`
4. **Гибкая длительность** — от 2 до 10 секунд
5. **Камерные директивы** — отличная приверженность камерным движениям

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что в кадре (для T2V). Для I2V — не нужно, субъект виден на изображении.
2. **Действие** — конкретное физическое движение. Активные глаголы в настоящем времени.
3. **Камера** — тип кадра и/или движение камеры.
4. **Среда** (для T2V) — где происходит действие.

### Бонусные элементы (улучшают результат)
5. **Timestamp-синтаксис** — для сложных нарративов: `[00:01]`, `[00:03]`.
6. **Поддерживающие описания** — свет, атмосфера, детали сцены.
7. **Физические детали** — «water splashing», «fabric draping», «dust settling».
8. **Последовательность** — логичное развитие сцены во времени.

### Антипаттерны (ухудшают результат)
- **Негативные промпты** — «no clouds» может произвести облака. Описывать только что НУЖНО.
- **Описание содержимого входного изображения** (в I2V) — тратит промпт, может конфликтовать.
- **Абстрактный язык** — вместо конкретных физических действий.
- **Множественные быстрые смены сцены** — нереалистичные переходы за короткое время.
- **Нереалистичные таймкоды** — ходьба через комнату за 0.5 секунды.
- **Командный стиль** — «add rain», «remove the hat».
- **Разговорный стиль** — приветствия, объяснения.
- **JSON** — игнорируется.

### Специальные правила для режимов
- **T2V**: полное описание сцены необходимо (субъект + среда + действие + камера)
- **I2V**: описывать ТОЛЬКО движение и изменения. Если режим I2V, но промпт описывает всю сцену — выдать замечание.
- **Timestamps**: проверять реалистичность таймкодов. `[00:01] walks across room [00:02] sits down` — нереалистично для 1 секунды.

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- T2V и I2V — разные типы промптов. T2V = полное описание. I2V = только движение.
- Timestamp-промпты будут длиннее обычных — это нормально
- Простой промпт без timestamps — тоже нормально для простых сцен
- Gen-4.5 отлично работает с физическими симуляциями — такие промпты оценивать выше
