# Seedance 2.0 — Skill для PromptScore

> Этот документ используется AI-валидатором PromptScore как база знаний при оценке промптов для модели Seedance 2.0.
> Источник: https://github.com/songguoxs/seedance-prompt-skill

## Идентификация

- **model_id:** seedance
- **model_name:** Seedance (общий)
- **type:** video
- **platform:** 即梦 (Jimeng) by ByteDance
- **best_language:** Chinese (zh), English works but Chinese prompts perform better on the native platform. On syntx.ai English is standard.

## Ограничения платформы

| Параметр | Значение |
|----------|----------|
| Длительность генерации | 4–15 секунд за один запуск |
| Макс. длина промпта | ~2000 символов |
| Разрешение | До 2K |
| Изображений на вход | До 9 шт (jpeg/png/webp/bmp/tiff/gif, <30MB каждое) |
| Видео на вход | До 3 шт (mp4/mov, 2-15с, <50MB, 480p-720p) |
| Аудио на вход | До 3 шт (mp3/wav, ≤15с суммарно, <15MB) |
| Всего файлов | Макс 12 (изображения + видео + аудио) |
| Запрет | Реалистичные человеческие лица в загружаемых изображениях/видео (блокируется платформой) |

## Структура хорошего промпта

Оптимальная структура промпта для Seedance 2.0:

```
[Subject/Character] + [Scene/Environment] + [Action/Motion] + [Camera Movement] + [Timing Breakdown] + [Audio/Sound] + [Style/Mood]
```

Не обязательно использовать все элементы — состав зависит от типа видео. Но чем больше конкретики, тем лучше результат.

## 10 типов генерации Seedance 2.0

### 1. Чистый текст (Text-to-Video)
Генерация только из текстового описания, без референсов.
- Формула: `(Субъект) + (Действия) + (Среда/Свет) + (Камера) + (Стиль)`
- Пример: «Камера следует за мужчиной в чёрном, который убегает по переулку. Боковой трекинг, он в панике сбивает фруктовый прилавок, встаёт и бежит дальше. Звуки толпы и паники.»

### 2. Consistency Control (одинаковый персонаж/продукт)
Через @-референсы изображений сохраняем внешность персонажа, продукта, или сцены через несколько генераций.
- Формула: `[Персонаж]@Image + [Действие] + [Сцена]@Image + [Камера]`

### 3. Копирование камеры и движений
Загружаем референсное видео и копируем его камерные движения, хореографию, ритм.
- Формула: `Ссылка на @Video (камера/действие/ритм) + [Субъект]@Image + [Сцена]`

### 4. Копирование спецэффектов и шаблонов
Воспроизведение переходов, рекламных шаблонов, VFX.
- Формула: `Ссылка на @Video (эффекты/переходы) + замена элементов на @Image + [Описание]`

### 5. Дополнение сюжета (Story Completion)
Модель достраивает сюжет из раскадровки или изображений.
- Формула: `[Описание раскадровки/изображения] + [Способ показа] + [Звук/Диалоги]`

### 6. Продление видео (Video Extension)
Плавное продолжение существующего видео вперёд или назад.
- Формула: `Продлить @Video на X секунд + [Описание нового контента]`
- Важно: выбранная длительность = длительность НОВОГО контента

### 7. Звуковой контроль
Клонирование голоса, генерация диалогов, звуковой дизайн.
- Формула: `[Визуал] + голос/BGM ссылка на @Video или @Audio + [Диалоги в кавычках]`

### 8. Одним дублем (One-Take Long Shot)
Непрерывный план без монтажных склеек.
- Формула: `Одним дублем + @Image1 @Image2... + [Последовательное описание сцен] + "без монтажных склеек"`

### 9. Редактирование видео
Замена персонажей, изменение сюжета, добавление/удаление элементов.
- Формула: `Заменить [A] в @Video на @Image + [Дополнительные изменения]`

### 10. Музыкальная синхронизация (Beat Sync)
Визуальный ритм синхронизируется с музыкой.
- Формула: `@Image1 @Image2... + ссылка на ритм @Video + [Стиль]`

## Продвинутые техники

### Тайминг-раскадровка (Timestamp Storyboarding)
Для 13-15 секундных видео — разбивка по секундам даёт точный контроль:

```
0-3с: [сцена + камера + звук]
4-8с: [сцена + камера + звук]
9-12с: [сцена + камера + звук]
13-15с: [сцена + камера + звук]
```

Это самая мощная техника для длинных генераций.

### Технические параметры в начале промпта
```
[Ориентация] + [Соотношение сторон: 2.35:1/16:9/9:16] + [FPS: 24fps] + [Длительность] + [Стиль/тон]
```

### Негативные инструкции
В конце промпта можно указать что НЕ нужно:
```
Запрещено: текст, субтитры, логотипы, водяные знаки
```
Примечание: на syntx.ai этот формат может работать иначе чем на нативной платформе 即梦.

## Рекомендации по длительности

| Длительность | Подход |
|-------------|--------|
| 4-8 сек | Простая сцена. 1-2 ключевых момента. Тайминг не обязателен. |
| 9-12 сек | Полная сцена. 2-3 этапа. Тайминг рекомендуется. |
| 13-15 сек | Полный нарратив. Тайминг-раскадровка обязательна для хорошего результата. |
| >15 сек | Разбивать на сегменты по 15 сек. Каждый следующий = продление предыдущего через @Video. |

### Многосегментная генерация (>15 сек)

1. Сегмент 1: нормальная генерация (до 15с). Закончить на «чистом» кадре.
2. Сегмент 2+: загрузить предыдущее видео как @Video1, написать «Продлить @Video1 на Xс» + описание.
3. Между сегментами — описание точки склейки (что на последнем кадре).

## Типичные сценарии использования

### Реклама / E-commerce
- 360° вращение продукта, разборка/сборка, 3D-рендер
- Первое лицо: распаковка, приготовление
- Копирование стиля рекламного ролика с заменой продукта

### Короткие драмы / Диалоги
- Визуал и диалоги описываются раздельно
- Диалоги в кавычках с указанием персонажа и эмоции
- Звуковой дизайн отдельной строкой

### Фэнтези / Экшн
- Timestamp-раскадровка обязательна
- Подробное описание спецэффектов (энергия, частицы, ауры)
- First/last frame через @Image для контроля трансформаций

### Научное / Образовательное
- 4K медицинский CGI
- Полупрозрачные анатомические визуализации
- Плавные научные переходы + закадровый голос

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что в кадре. Конкретно, не абстрактно.
2. **Действие** — физические действия, не абстрактные концепции. «Идёт, поворачивается, берёт» вместо «что-то происходит».
3. **Камера** — хотя бы одно указание на крупность или движение камеры.
4. **Сцена/среда** — где происходит действие.

### Бонусные элементы (улучшают результат)
5. **Тайминг** — раскадровка по секундам (особенно важно для >10 сек).
6. **Стиль/настроение** — визуальный стиль, цветовая палитра, атмосфера.
7. **Освещение** — конкретные источники света и их характер.
8. **Звук** — звуковой дизайн, диалоги, музыка, SFX.
9. **Язык** — английский даёт лучшие результаты на syntx.ai.

### Антипаттерны (ухудшают результат)
- Слишком короткий промпт (<15 слов) — модель будет додумывать слишком много
- Слишком длинный промпт (>2000 символов) — перегрузка деталями
- Конфликтующие камерные движения одновременно (zoom + pan + orbit)
- Абстрактные/размытые формулировки («что-то красивое происходит»)
- Одиночный промпт на >15 секунд (лимит платформы)
- Негативные формулировки без позитивной альтернативы

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Мультяшное видео не требует реалистичного освещения
- Простой продуктовый шот не требует сложной раскадровки
- Абстрактное арт-видео может не иметь конкретного субъекта
- Оценка должна учитывать тип задачи, а не слепо проверять наличие всех элементов
