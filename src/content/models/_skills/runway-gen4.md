## Структура хорошего промпта

Промпт для Gen-4 описывает **ТОЛЬКО ДВИЖЕНИЕ** — входное изображение определяет визуальную составляющую.

```
[Camera movement]: [subject] [action]. [additional motion details]
```

### Принципы

- **НЕ описывать** то, что видно на изображении — только движение и изменения
- Начинать просто: 10–15 слов, затем итерировать
- Естественный язык предпочтительнее — полные предложения дают больше контроля
- «Clarity matters more than structure»
- Без приветствий, объяснений, команд

### Камерные движения (хорошо работают)

Dolly, truck, pan, tilt, crane, handheld, whip pan, crash zoom, push-in, pull-out, arc shots.

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Действие/движение** — конкретное физическое движение субъекта или камеры. Активные глаголы.
2. **Камера** — тип камерного движения. Dolly, pan, tilt, crane и т.д.
3. **Краткость** — 10–30 слов оптимально. Промпт описывает только ДВИЖЕНИЕ.

### Бонусные элементы (улучшают результат)
4. **Конкретные физические детали** — «hair blowing in wind», «fabric rippling».
5. **Одно движение камеры** — не перегружать несколькими одновременно.
6. **Направление** — «slowly», «suddenly», «gradually» для контроля темпа.

### Антипаттерны (ухудшают результат)
- **Описание визуальной сцены** — «a woman in a red dress standing in a park» — это уже определено входным изображением. Описывать только ДВИЖЕНИЕ.
- **Негативные промпты** — «no X» может произвести X. Описывать только что НУЖНО.
- **Абстрактный язык** — «something emotional happens» — без конкретных физических действий.
- **Множество быстрых смен сцены** — в 5–10 секундах нет места для сложной раскадровки.
- **Командные промпты** — «add rain», «remove the hat» — не команды, а описания.
- **JSON-форматирование** — игнорируется генеративными моделями.
- **Разговорный стиль** — приветствия, объяснения, «пожалуйста сделай».
- **Отсутствие входного изображения** — Gen-4 ТРЕБУЕТ изображение. Без него — fail.

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Gen-4 — ТОЛЬКО I2V. Промпт = описание движения.
- Короткий промпт 10–15 слов — это НОРМАЛЬНО и часто оптимально
- Длинные описательные промпты о внешности сцены — антипаттерн для I2V
- Turbo — для прототипирования, допустим менее детальный промпт
- Если в GENERATION SETTINGS нет входного изображения — КРИТИЧЕСКАЯ ошибка, score не выше 20
