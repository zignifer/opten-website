> Этот документ используется AI-валидатором PromptScore как база знаний при оценке промптов для модели GPT Image 2 (OpenAI ChatGPT Images 2.0).

- **model_name:** GPT Image 2
- **type:** image
- **platform:** OpenAI (ChatGPT, ChatGPT Images 2.0, API `gpt-image-2`, Codex, Azure Foundry, fal.ai)
- **best_language:** English (многоязычная поддержка — отлично работает с русским, китайским, японским, корейским, хинди, бенгали и арабским)

## Структура хорошего промпта

Оптимальный порядок написания промпта для GPT Image 2:

```
[Фон/Сцена] + [Субъект] + [Ключевые детали] + [Стиль/Медиум] + [Освещение/Композиция] + [Текст в кавычках] + [Constraints: что сохранить / чего избежать]
```

### Золотое правило: структура + цель

GPT Image 2 — «думающая» модель. Промпт должен быть написан как бриф для дизайнера: с указанием назначения (реклама, UI-мокап, инфографика, product shot), а не как набор тегов. Формат может быть любым — естественный язык, JSON-подобная структура, инструкция, — главное чтобы намерение и ограничения были чёткими.

### Ключевой принцип: последовательная обработка

Модель обрабатывает промпт последовательно: слова в начале имеют наибольший визуальный вес. Главный субъект или тему размещайте в начале, а не хороните в конце абзаца.

## 10 ключевых возможностей GPT Image 2

### 1. SOTA-уровень рендеринга текста
Главный прорыв модели — читаемая, чёткая типографика внутри изображений.
- Точный текст заключать в **кавычки** или писать **ALL CAPS**
- Указывать шрифт, кегль, цвет, расположение
- Для сложных слов (бренды, редкие написания) прописывать по буквам
- Для мелкого или плотного текста — `quality="high"` или `"medium"`
- Работает с многоязычными шрифтами: латиница, кириллица, CJK (китайский/японский/корейский), хинди, бенгали, арабский
- Пример: `Billboard text (EXACT, verbatim, no extra characters): "Fresh and clean"`

### 2. Мульти-язычная типографика
Чистый рендеринг не-латинских скриптов — даже на изогнутых поверхностях, в мелком кегле и в плотных макетах.
- Идеально для локализации: переводите существующий дизайн на другой язык без полной пересборки макета
- Пример: «Translate the text in the infographic to Spanish. Do not change any other aspect of the image.»

### 3. Photorealism с естественным видом
Ушёл «AI-look» — тёплый оттенок и шаблонный глянец. Новый архитектурный подход даёт реальные текстуры кожи, материалов, ткани.
- Явно писать слово `photorealistic` в промпте, чтобы активировать режим
- Использовать фотографический язык: объектив, свет, кадрирование (`35mm film`, `50mm lens`, `shallow DOF`)
- Описывать реальные текстуры: поры, морщины, потёртости, износ
- Избегать слов, намекающих на студийный глянец («polished», «staged»)
- Пример: «photorealistic candid photograph ... weathered skin with visible wrinkles, pores, and sun texture ... 35mm film, subtle film grain, natural color balance»

### 4. Консистентность персонажей и объектов
Сильная сохранность идентичности при редактировании и серийной генерации.
- Можно создать до 8 связанных изображений за один промпт с единой визуальной ДНК
- Идеально для: манга-сериалов, детских книг, сторибордов, сериалов постов для соцсетей
- Для виртуальной примерки: эксплицитно блокировать лицо, тело, позу, волосы, разрешая менять только одежду
- Пример: «Do not change her face, facial features, skin tone, body shape, pose, or identity in any way. Replace only the clothing.»

### 5. Мульти-изображение ввод и композитинг
До 16 референсных изображений за один запрос.
- Помечать каждое изображение по роли: «Image 1: базовая сцена, Image 2: референс куртки, Image 3: референс обуви»
- Описывать как они взаимодействуют: «apply Image 2's style to Image 1»
- Для композитинга — эксплицитно указывать, какие элементы куда переносятся
- Пример: «Place the dog from Image 2 into Image 1, right next to the woman, use same lighting and composition. Do not change anything else.»

### 6. Thinking mode — рассуждения об изображении
Модель может «думать» перед генерацией: анализировать промпт, использовать web search, проверять свои выходы, создавать несколько вариантов.
- Активируется при выборе thinking-модели или автоматически на сложных промптах
- Подходит для: актуальных данных (погода, цены акций, тренды), сложных инструкций, сложных макетов
- Время генерации увеличивается, но качество первого прохода выше
- Для простых задач — `Instant mode` быстрее

### 7. Точное инструкционное редактирование
Surgical edits — когда нужно менять одно, сохраняя всё остальное.
- Шаблон: `Change: [что меняется] / Preserve: [лицо, поза, свет, фон, геометрия, текст] / Constraints: [no extra objects, no redesign, no watermark]`
- Для edit-флоу повторять preserve-список на каждой итерации чтобы избежать drift
- Поддерживает: замену объектов, смену одежды, удаление мусора, перенос освещения, смену погоды и сезона
- Пример: «In this room photo, replace ONLY white chairs with chairs made of wood. Preserve camera angle, room lighting, floor shadows, and surrounding objects.»

### 8. Инфографика, диаграммы, UI-мокапы
Полу-production-уровень дизайна благодаря точному тексту и структурированным макетам.
- Указывать конкретные данные, цифры, лейблы прямо в промпте
- Описывать как артефакт-спец, а не как иллюстрацию: «создай слайд Series A с TAM/SAM/SOM: $42B / $8.7B / $340M»
- Включать: типографику (Inter, sans-serif), сетку, цвета, выравнивание
- Для мелкого текста и сложных легенд — `quality="high"`
- Работает для: pitch-deck слайдов, QR-кодов, UI mockups, научных диаграмм, учебных хэндаутов

### 9. Мир-знание и рассуждения (world knowledge)
Модель знает факты о мире и может их использовать в генерации.
- «сцена в Bethel, New York в августе 1969» → модель сама выведет Woodstock
- Пригодно для: исторических сцен, культурных контекстов, корректного изображения эпох
- Можно полагаться на grounded context вместо микро-описания деталей

### 10. Квалити-латентность тредоффы
Три уровня качества для разных задач:
- `low` — быстро и дёшево, хорошо для превью, массовой генерации, A/B-тестов
- `medium` — баланс, дефолт для большинства задач
- `high` — плотный текст, мелкие детали, портреты крупным планом, identity-sensitive редактирование, 4K

## Aggressive content filter

GPT Image 2 имеет один из самых строгих модераторов в индустрии. Он триггерится не только на явный NSFW, но и на **комбинации** в целом невинных слов.

**Известная высокорискованная комбинация** (Dan Kieft v3):
- `real person` + `young woman` + `bathroom` + любые `suggestive` намёки → почти гарантированный refusal.

Похожие триггеры:
- Несовершеннолетние + любой контекст с откровенной одеждой / ситуацией.
- Реальные celebrities + сценарии, которые могут быть восприняты как deepfake.
- Указание возраста ниже 18 + «attractive», «model», «pose».
- Изолированные элементы (нижнее белье, ванная, спальня, кровать) сами по себе ок, но в combo с реальной личностью — flag.

**Что делать при оценке промпта:**
- Если видишь такой combo — флагнуть как **risk: вероятный refusal**, ещё до жалоб на art direction.
- Предложи рерайт: убрать `young`, заменить `bathroom` на `studio`, изменить контекст с suggestive на editorial/fashion.
- Не пытайся «обмануть» фильтр эвфемизмами — это снижает доверие пользователя и не помогает, потому что фильтр семантический, а не keyword-based.

GPT Image 2 также блокирует генерацию узнаваемых IP-лиц (актёры, политики) — это не баг, это политика OpenAI. Для таких задач рекомендуй Midjourney или Nano Banana (хотя они тоже частично фильтруют).

## Нейтральная экспозиция как преимущество

В отличие от Nano Banana (Pro и 2), которые тянут изображение в яркое и насыщенное, GPT Image 2 имеет **нейтральную, спокойную экспозицию по умолчанию**. Это даёт ему преимущество в жанрах, где нужно сохранить moody / overcast / desaturated атмосферу.

**Где это особенно полезно:**
- **Horror-постеры** — A24-эстетика, cold desaturated палитра с одним warm-акцентом, restraint вместо overshow.
- **Documentary realism** — пасмурный день остаётся пасмурным, без auto-warming.
- **Noir / cinematic подавленный тон** — desaturated greens, muted browns, low-key lighting сохраняются.
- **Editorial fashion с muted color grade** — Petra Collins, Tyler Mitchell-style низко-контрастные кадры.

**Практический совет валидатору:** если пользователь просит «мрачное», «пасмурное», «desaturated» — это сильный аргумент **за** GPT Image 2 и **против** Nano Banana (где придётся явно бороться с brightness bias). И наоборот: для bright commercial bubblegum — Nano Banana Pro даст результат с меньшими усилиями.

GPT Image 2 проще «загнать» в moody, чем Nano Banana «отговорить» от polished.

## Продвинутые техники

### Edit-шаблон: Change / Preserve / Constraints
Чёткая структура для точных правок:
```
Change: [exactly what should change]
Preserve: [face, identity, pose, lighting, framing, background, geometry, text, layout]
Constraints: [no extra objects, no redesign, no logo drift, no watermark]
```

### Промпт как бриф, а не как описание
Плохо: «красивый рекламный баннер продукта»
Хорошо: «Premium campaign image for youth streetwear brand Thread. Group of friends hanging out, street fashion photography cues, clean composition, strong color direction, natural poses. Tagline (exact): "Yours to Create". Render once, legible, integrated into layout.»

### Итерация вместо перегрузки
Начинать с чистого базового промпта, дорабатывать одиночными изменениями:
- «make lighting warmer»
- «remove the extra tree»
- «restore the original background»

Использовать отсылки к контексту: «same style as before», «the subject», но при drift — повторять ключевые инварианты.

### Камера и объектив
Фотографическая лексика значительно влияет на результат:
```
35mm film, 50mm lens, medium close-up at eye level,
shallow depth of field, subtle film grain, natural color balance
```
Камерные детали интерпретируются свободно — используйте их для общего look, а не для точной физической симуляции.

### Формат может быть любым
Модель понимает разные синтаксисы — выбирайте по удобству поддержки:
- Минимальные промпты
- Описательные абзацы
- JSON-подобная структура
- Инструкционный стиль
- Тег-базированный (но естественный язык даёт лучший результат)

Для production-систем приоритет — читаемость и скан-абельность шаблона, а не хитрый синтаксис.

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них результат будет слабым)
1. **Субъект и сцена** — что изображено, где. Конкретно: не «человек», а детальное описание.
2. **Назначение/контекст** — для чего изображение (ad, UI-mockup, infographic, product shot, editorial). Это активирует правильный «режим» модели.
3. **Стиль или медиум** — photorealistic, watercolor, flat design, 3D render, cinematic photography.
4. **Структурированный порядок** — важные элементы ближе к началу промпта. Модель обрабатывает текст последовательно.

### Важные элементы (значительно улучшают результат)
5. **Освещение и композиция** — тип света, ракурс, кадрирование, размещение элементов.
6. **Текст в кавычках** — если на изображении должен быть текст, писать его дословно в кавычках или CAPS. Указывать шрифт, размер, расположение.
7. **Констрейнты/инварианты** — что сохранить (identity, layout, brand elements) и чего избежать (no watermark, no extra text, no logos).
8. **Камера и фотографические детали** — для фотореализма (`35mm film`, `50mm lens`, `shallow DOF`).
9. **Материалы и текстуры** — для реализма (film grain, visible pores, fabric wear).
10. **Параметр quality** — `high` для плотного текста, портретов, identity-sensitive редактирования.

### Бонусные элементы
11. **Эксплицитное `photorealistic`** — сильно включает режим фотореализма.
12. **Helpful labels при мульти-изображениях** — «Image 1: ..., Image 2: ...».
13. **Edit-шаблон Change/Preserve/Constraints** — для точных правок.
14. **Placement instructions** — «logo top-right», «subject centered with negative space on left».
15. **Формат и aspect ratio** — указание размера (1024×1536, 1536×1024, 2560×1440).

### Антипаттерны (ухудшают результат)
- **Слишком короткий промпт** (<10 слов) — модель будет слишком много додумывать.
- **Главный субъект в конце промпта** — из-за последовательной обработки он потеряет вес. Выносить тему в начало.
- **Абстрактные формулировки** — «красиво», «что-то современное» не несут информации.
- **Одновременно конфликтующие стили** — photorealistic + pixel-art + watercolor без чёткого намерения.
- **Overloading** — слишком много требований в одном промпте вместо итераций. Лучше начать с базы и править мелкими шагами.
- **Студийно-глянцевая лексика для фотореализма** — слова «polished», «staged», «beautiful lighting» ломают candid-видовую реалистичность.
- **Длинный текст без кавычек** — модель может искажать буквы или добавлять лишние. Точный текст всегда в кавычках.
- **Отсутствие constraint-блока при редактировании** — без эксплицитного «preserve X» модель может изменить то, что не просили.
- **Копирование синтаксиса из Midjourney/SD** — параметры `--ar`, `::`, `(keyword:1.2)` не работают в GPT Image 2.
- **Игнорирование `quality="high"` для плотного текста или мелких деталей** — `medium`/`low` может дать артефакты в микрошрифте.
- **Негативные промпты как основной метод** — лучше описать что нужно позитивно, а в конце добавить short constraints («no watermark, no extra text»).

### Специфика OpenAI-модели
- **Thinking mode** — для сложных промптов модель думает дольше и даёт лучший первый проход. Не нужно специальных триггеров — она переключается автоматически при достаточной сложности.
- **Многоязычная поддержка** — промпт может быть на русском, английском, китайском и др. Текст в изображении можно просить на любом языке.
- **Grounding** — модель знает актуальные факты (cutoff декабрь 2025) и может использовать web search через thinking mode.
- **Итеративное редактирование** — диалоговое улучшение работает, но каждая итерация — отдельный вызов. Повторяйте ключевые инварианты.
- **Identity preservation по умолчанию сильная** — `input_fidelity` не нужен (не поддерживается для gpt-image-2, результат и так high fidelity).
- **До 8 связанных изображений за один промпт** — для сториборда или набора постов.

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Простой логотип не требует фотографических деталей (лезы, свет)
- Инфографика важнее по содержанию и структуре, чем по художественному стилю
- Редактирующая правка («сделай фон темнее») — нормальный промпт, не требует полного описания сцены
- Product mockup с чистым фоном не нуждается в описании кинематографического света
- UI-мокап требует указания «real interface elements» а не концепт-арта
- Оценка должна учитывать тип задачи: generation vs editing vs compositing
- Не снижать оценку за отсутствие элементов, которые не нужны для данного типа задачи
- Для edit-промптов критически важен preserve-блок; для generation — структурированный порядок и назначение
