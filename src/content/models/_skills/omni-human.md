**КРИТИЧЕСКИ ВАЖНО:** OmniHuman — это **специализированная модель анимации людей**, НЕ генератор видео общего назначения. Обязательный вход: изображение человека. Основной драйвер — аудио, текстовый промпт — дополнительный.

---

## Как работает OmniHuman

### Три входа (работают вместе)
1. **Image (обязательно)** — референсное изображение человека (портрет, полуфигура, полная фигура)
2. **Audio** — аудио для lip-sync и body language (речь, музыка, пение)
3. **Text prompt (дополнительный)** — описание сцены, камеры, действий, эмоций

Качество зависит от **согласованности** всех трёх входов.

### Типы субъектов
- Реальные люди
- Животные
- Стилизованные / анимированные персонажи
- 3D-модели

## Структура хорошего промпта

Текстовый промпт = **дополнение к аудио**. Функция: описание сцены, камеры, действий.

```
[Character description + pose] + [Action/movement] + [Camera] + [Emotional tone]
```

Писать как короткие естественные сценарии, НЕ список ключевых слов.

### Пример
«A male DJ performing live on stage, wearing headphones and mixing music on a DJ controller, focused expression, subtle head movement following the beat»

---

## Ключевые возможности

1. **Audio-driven lip-sync** с эмоционально отзывчивым языком тела
2. **Single reference image** — одно изображение достаточно (портрет, полуфигура, полная фигура)
3. **Разнообразие субъектов** — люди, животные, стилизованные персонажи
4. **Multi-character scenes** — назначение спикера и фоновые реакции
5. **Микро-выражения** — распознаёт нюансы эмоций из аудио просодии

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы
1. **Входное изображение человека** — если нет — fail, score не выше 20. OmniHuman ТРЕБУЕТ изображение.
2. **Описание действия/позы** — что делает персонаж: «performing», «speaking», «dancing».
3. **Согласованность с аудио** — промпт + изображение + аудио должны быть логически совместимы.

### Бонусные элементы (улучшают результат)
4. **Эмоциональный тон** — «focused expression», «joyful», «serious».
5. **Камера** — тип кадра и ракурс.
6. **Описание движения** — «subtle head movement following the beat», «hands gesturing expressively».
7. **Multi-character назначение** — кто говорит, кто фоновый.

### Антипаттерны (ухудшают результат)
- **Использование как T2V** — OmniHuman ВСЕГДА требует изображение человека. Это не general video generator.
- **Использование как image generator** — OmniHuman не генерирует изображения.
- **Отсутствие аудио** — без аудио не будет lip-sync (главная фича).
- **Рассогласование входов** — DJ в промпте, классическая музыка в аудио, портрет бизнесмена — конфликт.
- **Ожидание высокого разрешения** — макс 1024×1024, не для продакшн 4K.
- **Keyword списки вместо описаний** — писать естественные предложения.
- **Описание внешности** — внешность определена входным изображением.

### Контекстная оценка
AI должен понимать:
- OmniHuman = specialized human animation, не universal video model
- Текстовый промпт — ДОПОЛНЕНИЕ к аудио, не основной вход
- Короткий промпт (15–40 слов) — нормально
- Качество = согласованность image + audio + prompt
- Без аудио — резко деградирует качество (нет lip-sync)
- Multi-character — нужно явное назначение спикеров
