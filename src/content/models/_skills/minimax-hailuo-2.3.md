## Структура хорошего промпта

Идеальная длина: **40–60 слов**. Макс: 2,000 символов.

Стиль: **режиссёрские заметки** на естественном языке, НЕ теги/ключевые слова.

```
[Camera shot + motion] + [Subject + description] + [Action (present tense)] + [Style & atmosphere] + [Emotional cues]
```

### Bracket Camera Syntax — `[command]`

15 поддерживаемых камерных команд:

| Команда | Описание |
|---------|----------|
| `[Truck left]` | Горизонтальное перемещение влево |
| `[Truck right]` | Горизонтальное перемещение вправо |
| `[Pan left]` | Панорама влево |
| `[Pan right]` | Панорама вправо |
| `[Push in]` | Движение к субъекту |
| `[Pull out]` | Движение от субъекта |
| `[Pedestal up]` | Подъём камеры |
| `[Pedestal down]` | Опускание камеры |
| `[Tilt up]` | Наклон вверх |
| `[Tilt down]` | Наклон вниз |
| `[Zoom in]` | Зум приближение |
| `[Zoom out]` | Зум отдаление |
| `[Shake]` | Тряска камеры |
| `[Tracking shot]` | Следящий кадр |
| `[Static shot]` | Статичный кадр |

**Комбинирование:** `[Pan left,Pedestal up]` — максимум 3 одновременных команды.
**Последовательность:** `"...[Push in], then...[Pull out]"` — через связующие слова.

### Prompt Optimizer

Параметр `prompt_optimizer` (default: `true`) — LLM MiniMax переписывает/улучшает промпт.
Установить `false` для точного контроля — модель будет следовать промпту буквально.

## Критерии оценки промпта (для AI-валидатора)

При оценке промпта AI должен учитывать:

### Обязательные элементы (без них генерация будет слабой)
1. **Субъект** — кто или что в кадре. Конкретные визуальные детали (внешность, одежда).
2. **Действие** — конкретные глаголы в настоящем времени. «Spinning», «leaping», не «moves around».
3. **Камера** — bracket command `[Push in]` или текстовое описание крупности и движения.
4. **Длина 40–60 слов** — оптимальный диапазон. Слишком коротко — генерик результат, слишком длинно — перегрузка.

### Бонусные элементы (улучшают результат)
5. **Стиль и атмосфера** — освещение, цветовая палитра, настроение.
6. **Эмоциональные маркеры** — «serene», «tense», «dreamlike» — модель чувствительна к эмоциям.
7. **Bracket camera commands** — точный контроль камеры.
8. **Комбинированные камерные движения** — `[Pan left,Pedestal up]`.

### Антипаттерны (ухудшают результат)
- **Tag-based промпты** — «cyberpunk, rain, neon, 4k» — использовать нарративные предложения.
- **Quality boosters** — «ultra-detailed, 8k, masterpiece» — вызывает чрезмерную насыщенность и контраст.
- **Перегрузка конфликтующими инструкциями** — вызывает деформацию лиц и объектов.
- **Описание изображения в I2V** — описывать только ДВИЖЕНИЕ/ИЗМЕНЕНИЯ.
- **Hailuo 2.3 Fast для T2V** — Fast версия поддерживает ТОЛЬКО I2V.
- **Размытые промпты** — «a video of a beach» — слишком обще, результат генерик.
- **Более 3 bracket commands одновременно** — перегрузка камерных инструкций.

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- 40–60 слов — оптимально, не штрафовать за краткость в этом диапазоне
- Bracket syntax `[command]` — это фича модели, не ошибка форматирования
- Hailuo 2.3 ≠ Hailuo 02 — это БОЛЕЕ НОВАЯ модель (02 — старая)
- I2V промпт должен быть короче и описывать только движение
- prompt_optimizer=true может компенсировать короткие промпты
- Мультиязычность поддерживается, но английский/китайский — лучше всего
