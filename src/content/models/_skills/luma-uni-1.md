## Ограничения платформы

| Параметр | Значение |
|----------|----------|
| Тип генерации | Текст → Изображение (Generate), Изображение → Изображение (Modify), Multi-Reference Fusion |
| Архитектура | Decoder-only autoregressive transformer (НЕ диффузия) — генерирует пиксели последовательно |
| Макс. разрешение | Нативно 2K (без апскейла), на API доступно несколько ступеней |

| Storyboard mode | Генерация многокадровых раскадровок и комиксов с консистентностью персонажей |
| Стоимость | Каждое изображение (вход или выход) ≈ 2000 billing tokens, на ~30% дешевле конкурентов на 2K |

## Структура хорошего промпта

### Универсальный «Fast Start» шаблон (закрывает 80% задач)

```
A [субъект], in [стиль], with [освещение], [камера/композиция], [окружение/фон], mood: [эмоция], details: [ключевые специфики]
```

Пример:
> A ceramic artist shaping a lopsided bowl, documentary photography style, soft window lighting, close-up shot, cluttered home studio background, mood: focused and quiet, details: clay-covered hands, imperfect texture, tools scattered on wooden table

### Золотое правило: Start → Direct → Refine → Finish

Uni-1 — не про идеальный промпт с первого раза. Это про **управляемый итеративный процесс**: сгенерировать → направить → уточнить → закрепить (lock seed). Модель «думает» через интент пользователя, а не угадывает по ключевым словам.

### Ключевой принцип: режим важнее тегов

Перед промптом всегда выбирается режим:
- **Generate** — создание нового изображения с нуля
- **Modify** — точечная правка существующего (с сохранением неизменных частей)
- **Multi-Reference** — фьюжн нескольких референсов с явными ролями

Указание режима и ролей референсов важнее, чем перегруженный текст.

## 8 шаблонов промптов (Uni-1 Field Guide)

### 1. Fast Start (для большинства задач)
```
A [subject], in [style], with [lighting], [camera/composition], [environment/background], mood: [emotion], details: [key specifics]
```
Когда использовать: исследование, первые идеи, быстрые выходы.

### 2. Cinematic Control (структурированный кинематографический брифинг)
Раздельные блоки: Subject / Style / Scene (Environment, Time of day, Mood) / Camera (Shot, Lens, Composition) / Details (Textures, Motion, Extra). Подходит для editorial-визуала и истории.

### 3. Direct Edit (точечное редактирование в режиме Modify)
```
Edit instructions:
- Change: [что меняется]
- Keep: [что должно остаться неизменным]
- Style shift: [опционально]
- Lighting: [опционально]
- Details: [специфические правки]
```
**Pro tip:** всегда явно прописывать **Keep** — это главный приём против drift.

### 4. Multi-Reference Fusion (комбинирование 2–9 референсов с ролями)
```
Combine the following:
IMAGE 1: [роль и что даёт] (например, use as CHARACTER reference)
IMAGE 2: [роль и что даёт] (например, use for STYLE)
IMAGE 3: [роль и что даёт] (например, use as LIGHTING reference)

Output:
- Subject: [финальная цель]
- Style: [доминантный стиль]
- Composition: [гайд по компоновке]
- Details: [специфичные слияния]
```
Каждый референс имеет авторитет только над своим слоем. Не позволяйте им конфликтовать.

### 5. Layout Control (управление компоновкой)
Прямое указание зон: LEFT / CENTER / RIGHT / BACKGROUND с описанием объектов в каждой. Работает как режиссёрская расстановка.

### 6. Storyboard Generator (раскадровка/комикс)
```
Create a storyboard with [X] frames.
Story: [короткий нарратив]
Frame details: 1. [сцена] 2. [сцена] 3. [сцена]
Style: [стиль]
Consistency: same characters and setting
```
Uni-1 особенно силён в многокадровых раскадровках с сохранением персонажа.

### 7. Loose / Creative Mode (фрагменты-настроение)
```
[вайбы, фрагменты, идеи, эмоции]
```
Для ранней идеации и неожиданных результатов. Полные предложения не обязательны.

### 8. Structured JSON
```json
{
  "subject": "...",
  "style": "...",
  "environment": "...",
  "lighting": "...",
  "camera": {"shot": "...", "lens": "...", "composition": "..."},
  "details": [...]
}
```
JSON работает, но обычная проза с структурой так же хороша.

## Продвинутые техники

### Reasoning + intent_weight
Uni-1 — «думающая» модель: сначала строит внутреннюю карту визуального намерения, потом рендерит. На API параметр `intent_weight` управляет глубиной этого рассуждения. Можно даже посмотреть «логи мыслей» через `/v1/reasoning-logs`. Для сложных композиционных промптов оставляйте reasoning активным.

### Назначение ролей референсам
```
Use IMAGE1 as a CHARACTER reference. Preserve their exact facial features,
bone structure, skin tone, and age.
```
Роли: CHARACTER, STYLE, LIGHTING, COMPOSITION, OUTFIT, BACKGROUND, POSE. Каждой можно дать одну роль — это снижает конфликты между референсами.

### Seed как «recipe»
Сохраняйте `prompt + seed` вместе — это переиспользуемый рецепт. При итеративной правке (Workflow «Exploration → Lock → Iterate») зафиксируйте seed после понравившегося результата и меняйте только текст.

### Workflow для точной правки
1. Сгенерировать базу
2. Идентифицировать проблемные зоны
3. Modify-промпт с эксплицитным `Keep` (что не трогать)
4. Повторить узкими шагами

### Композиционно сложные промпты
Uni-1 заметно сильнее диффузий на промптах с пространственными отношениями: «собака на красном диване, спиной к зрителю, в левой трети кадра, через окно виден дождливый город». Диффузия начинает терять элементы — Uni-1 удерживает.

### Многоязычный текст в изображении
Модель умеет рендерить читаемый текст на разных языках, включая кириллицу и украинский. Текст указывайте в кавычках, желательно короткими фразами.

## Типичные сценарии использования

### Editorial и нарративная фотография
- Документальные сцены (мастер за работой, уличный портной)
- Выдуманный или архивный editorial с явным освещением и линзой
- Преимущество Uni-1: сильное удержание спатиальных деталей

### Multi-Reference композиции
- Портрет + стиль + текстура одновременно (3 референса с разными ролями)
- Фьюжн винтажной позы и современного fashion-стиля
- Идеально для рекламы и брендового контента (Adidas, Mazda кейсы)

### Комиксы, манга, раскадровки
- Многостраничные комиксы с диалоговыми облачками и панельным макетом
- Серии кадров с одним персонажем через эпохи (мальчик → подросток → взрослый → старик)
- Self-correcting iteration внутри Storyboard режима

### Точное редактирование (Modify)
- Замена фона с сохранением субъекта и позы
- Перенос освещения без изменения геометрии
- Локализация текста на изображении на другой язык

### Сложные композиционные брифы
- «X в левой трети, Y в центре, Z на фоне» — сильная сторона модели
- Инфографика и социальные карточки с точно расставленным текстом
- Layout Control шаблон для дизайна

### Дизайн/реклама с требованием control
- Pitch-deck иллюстрации
- Магазинные обложки с верба́тимом
- Кампании с консистентным персонажем через серию

## Критерии оценки промпта (для AI-валидатора)

### Обязательные элементы (без них результат будет слабым)
1. **Субъект** — кто или что изображено. Конкретно: не «человек», а описание внешности/действия.
2. **Намерение/цель** — что должно получиться (editorial-портрет, story-board, edit, fusion). Это активирует правильный режим reasoning.
3. **Стиль или медиум** — photorealism, documentary, anime, oil painting, vector.
4. **Структурированность** — Uni-1 хорошо понимает шаблоны (Fast Start, Cinematic Control, Direct Edit). Промпт с явной структурой даёт лучший результат, чем сырая проза.

### Важные элементы (значительно улучшают результат)
5. **Освещение** — soft window light, golden hour, hard top-down key и т.д.
6. **Композиция/камера** — close-up / wide / off-center / 35mm / shallow DOF.
7. **Mood** — эмоциональный тон сцены (focused, intimate, dramatic).
8. **Details** — конкретные текстуры, мелкие визуальные акценты.
9. **Roles при референсах** — явное назначение ролей IMAGE1/IMAGE2 (CHARACTER, STYLE, LIGHTING) исключает конфликты между референсами.
10. **Keep-блок при редактировании** — что должно остаться неизменным в Modify-режиме.

### Бонусные элементы
11. **Seed** — для воспроизводимости и итераций.
12. **Структурированный формат** (Markdown-секции или JSON) для сложных промптов.
13. **Layout-инструкции** (LEFT / CENTER / RIGHT / BACKGROUND) для точной компоновки.
14. **Storyboard-указания** (X frames, consistency: same character) для серий.
15. **Контекст назначения** — для чего изображение (обложка, реклама, ad-кампания, иллюстрация).

### Антипаттерны (ухудшают результат)
- **Слишком короткий промпт** (<8 слов) — Uni-1 потратит reasoning впустую и придумает контекст за пользователя.
- **Tag soup без структуры** — модель не диффузия, она ждёт связных предложений или явных секций. Простые запятые-теги работают хуже.
- **Конфликт ролей у референсов** — два IMAGE с одинаковой ролью (например, оба CHARACTER) дают drift. Назначайте уникальные роли.
- **Отсутствие Keep-блока в Modify-режиме** — модель может изменить то, что не просили.
- **Параметры синтаксиса других моделей** — Midjourney `--ar`, `::`, weighted parentheses `(text:1.2)`, BREAK от SD не работают.
- **Слишком много стилей одновременно** — «photorealistic + anime + watercolor» без явного намерения ломает результат.
- **Игнорирование reasoning при сложных промптах** — для композиционно богатых сцен НЕ отключайте thinking.
- **Чрезмерный мульти-референс без ролей** — 5–9 референсов без явного назначения ролей превращаются в кашу.
- **Запросы, требующие speed-prioritization** — Uni-1 медленнее диффузий на 2K. Для high-volume A/B тестов лучше использовать другие модели.
- **Упоминание текстов нелатинскими алфавитами без кавычек** — текст всегда в кавычках, модель должна понять что это рендеринг.

### Специфика Luma-модели
- **Reasoning по умолчанию** — модель «думает» перед генерацией; для сложных промптов это плюс, для простых — медленнее, чем диффузия.
- **Seed = recipe** — сохраняйте seed после удачного результата для итераций.
- **Reference roles** — главная фича для контроля. Использовать обязательно при ≥2 референсах.
- **Generate vs Modify** — два разных режима с разной логикой; Modify требует эксплицитного Keep.
- **Multilingual** — промпт может быть на любом языке, текст в изображении тоже.
- **Storyboard режим** — отдельный мощный workflow с консистентностью персонажа через кадры.
- **API: intent_weight, /v1/reasoning-logs** — продвинутые контроли для разработчиков.

### Контекстная оценка
AI должен понимать НАМЕРЕНИЕ пользователя:
- Loose / Creative промпт («туман, пыль, утро, тишина») — это валидный режим, не нужно требовать структуры
- Modify-промпт с фокусом на одном изменении не требует полного описания сцены — главное Keep-блок
- Storyboard-запрос оценивается по нарративу + consistency, а не по фотореалистичным деталям
- Multi-Reference fusion оценивается по корректности назначения ролей, а не по перечислению свойств
- Простой Fast Start шаблон уже даёт хороший результат — не штрафовать за отсутствие Cinematic-уровня деталей
- Uni-1 — модель управления и итераций; перфекционизм в первом промпте не нужен
- Не сравнивать с диффузионными моделями: некоторые «правила» (короткие теги, weighted prompts) сюда не переносятся
- Оценка должна учитывать тип задачи и выбранный шаблон промпта, а не слепо проверять все элементы
